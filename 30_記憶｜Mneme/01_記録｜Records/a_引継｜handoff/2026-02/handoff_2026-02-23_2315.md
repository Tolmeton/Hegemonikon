### 1. LS は extension_server_port なしでも認証済み状態を返す

`--extension_server_port` を指定せずに standalone LS を起動しても、`GetUserStatus` は **200 OK + 完全なユーザー情報** (Pro プラン、Ultra tier 等) を返した。

**推定メカニズム**: LS バイナリは `~/.config/Antigravity/` 配下の `state.vscdb` から直接 OAuth トークンを読み込んでいる可能性が高い (API サーバーと直接通信)。

### 2. extension_server なしでの SendUserCascadeMessage は SIGSEGV

extension_server_port を指定しない standalone LS に対して `SendUserCascadeMessage` を呼ぶと、LS が **SIGSEGV (segmentation violation)** でクラッシュする。

```

goroutine 268:

```

**原因**: extension_server クライアントが nil であるため、Cascade 処理中に nil pointer dereference が発生。

## LS ポートの注意事項

LS は **HTTPS ポート** と **HTTP ポート** の2つをリッスンする:

```
Language server listening on random port at 38545 for HTTPS
Language server listening on random port at 38527 for HTTP
```

- ConnectRPC (JSON over HTTP/1.1) は **HTTP ポート** を使う
- HTTPS ポートに HTTP リクエストを送ると `Client sent an HTTP request to an HTTPS server` エラー
- `grep -oP '...for HTTP$'` でログから正確な HTTP ポートを取得すること
