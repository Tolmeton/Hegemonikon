## 🔬 Phase 0 抽出結果 (核心知見)

### 特定されたプロトコル体系

| パッケージ | サービスパス | 役割 |
|:-----------|:-------------|:-----|
| `extension_server_pb` | `/exa.extension_server_pb.ExtensionServerService/<Method>` | LS↔IDE Extension 間の本体通信 |
| `seat_management_pb` | `/exa.seat_management_pb.SeatManagementService/<Method>` | API Key → SessionToken の認証フロー |
| `api_server_pb` | `/exa.api_server_pb.ApiServerService/<Method>` | LS → API Server への推論リクエスト |

### 認証フロー (推定)

```
IDE Extension Server (port 36989)
  ↓ FetchMCPAuthToken / GetToken
  ↓ Token (proto field 1)
LS
  ↓ AuthToken (proto field 1 or 2)
API Server (cloudapi-dot-devrel-web-testing.appspot.com)
  ↓ StartCascade / SendUserCascadeMessage
  ↓ Claude/Gemini 推論結果
```

### 重要メソッド名

- `FetchMCPAuthToken` — MCP認証トークン取得
- `GetToken` / `UpdateAuth` / `SetCredential` — 認証関連
- `Heartbeat` — Extension Server の生存確認
- `LogEvent` — ログイベント送信
