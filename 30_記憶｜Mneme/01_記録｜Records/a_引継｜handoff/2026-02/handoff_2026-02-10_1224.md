## Value Pitch (R^π)

**Angle: できる (Flow)**

3時間前、Creator がテキストをコピーして Linux にペーストしようとするたびに、手が止まっていた。xrdp の `cliprdr` は `unknown atom` とだけ吐いて、何も運ばない。クリップボードという最も基本的な橋が壊れていた。

RustDesk をインストールした。だが Tailscale の IP を入れても繋がらない。RustDesk はポートを開けていなかった — デフォルトでは公開リレーサーバーを介して接続する設計で、TCP で待ち受ける口がなかった。ニューヨークのリレーを経由すると、画面がスライドショーのように遅い。

`direct-server = 'Y'` と `allow-direct-server = 'Y'` — この2つのオプションを両方設定してサービスを再起動したとき、21118番ポートが開いた。Tailscale が透過するその穴を通して、Creator の Windows と Debian が直接握手した。

**リレーなし。遅延なし。クリップボードが通る。**

外出先から、Creator は今まで通りの速度で Linux を操作でき、テキストは片方で Ctrl+C を押したら、もう片方で Ctrl+V すれば出てくる。当たり前のことが、当たり前に動く。

> **Conclusion**: 壊れた橋を直すのではなく、新しい橋を架けた。そしてその橋は、トンネル（Tailscale）の中を一直線に通っている。

2. **De-Contextualization**: ツール X の機能 F が壊れている → ツール X を修復するか、機能 F を持つ代替ツール Y に移行するか
3. **The Principle**: 修復コストが代替コストを上回る場合、代替が正解。ただし代替の「接続方式」が既存インフラ (Tailscale) と互換するかが成否を分ける

**法則化 (Laws Learned)**:
