### quota.py の変更詳細

**2段フォールバック戦略**:

```
1. agq-check.sh 実行 → 失敗時 ↓
2. Python 直接 LS API 呼び出し (_fetch_quota_direct)
   a. /proc/{pid}/cmdline で LS 検出 + CSRF 抽出
   b. ss -tlnp で PID ベースの全リスニングポート取得
   c. HTTPS POST で GetUserStatus を呼び出し
   d. レスポンスを QuotaResponse に変換
```

### 発見したバグ (3つ)

| # | バグ | 原因 | 修正 |
|:--|:-----|:-----|:-----|
| 1 | `ps aux` の行切り詰め | daemon プロセスからの実行でターミナル幅がなく、長いコマンドラインが切り詰められる | `/proc/{pid}/cmdline` 直読みに変更 |
| 2 | コマンドライン引数のポート番号が不信頼 | `server_port` と `extension_server_port` が同一値になるケースあり。gRPC 専用ポートに HTTPS POST すると SSL エラー | `ss -tlnp` で PID ベースの全リスニングポートを取得 |
| 3 | LS の頻繁な再起動 | IDE 操作に伴い PID・ポート・CSRF が都度変化 | 毎リクエストで動的検出（キャッシュしない） |
