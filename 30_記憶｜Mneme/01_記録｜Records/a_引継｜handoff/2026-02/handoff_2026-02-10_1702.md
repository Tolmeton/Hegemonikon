## 法則化 — 今日学んだこと

### 法則 1: Tauri v2 の capabilities は exact match

`http:scope` のように「ありそうだが存在しない」パーミッション識別子を使うと、
build.rs が exit 1 で失敗するが、エラーメッセージが capabilities 列挙の大量出力に埋もれて見つけにくい。
**法則**: capabilities の identifier は公式ドキュメントで確認してから使え。grep で `error` を探すより `cargo build --release 2>&1 | grep -v "Compiling"` でノイズを除去せよ。

### 法則 2: kalon は実装まで降りてこそ完成する

`/noe` で発見した「知識は問いとして走ってくる」は、`.nr-question` CSS クラスの紫グラデーション + 左ボーダーという UI デザインに具体化されて初めて「美」として体験できるものになった。
**法則**: 圏論的洞察は、最終的に pixels にならなければ Creator の行為可能性を増やさない。
