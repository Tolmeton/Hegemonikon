## 完了事項

### 1. 根本原因の特定

- **原因**: MCP SDK v1.26.0 が `stdio_server()` 内部で `sys.stdout.buffer` を参照するようになった
- **影響**: 全 MCP サーバーの `main()` が `sys.stdout = sys.stderr` でリダイレクトしていたため、レスポンスが stderr に送られ IDE が EOF で切断
- **診断方法**: SDK ソース (`~/.venv/lib/python3.13/site-packages/mcp/server/stdio.py`) を直接確認

### 2. `mcp_base.py` 修正 (6サーバーに影響)

- **ファイル**: `mekhane/mcp/mcp_base.py` (L200-217)
- **修正内容**: `_original_stdout` を保存し、`stdio_server(stdin=real_stdin, stdout=real_stdout)` に明示渡し
- **影響サーバー**: typos, digestor, sympatheia, mneme, periskope, ochema

### 3. `hermeneus/src/mcp_server.py` 修正 (独自実装)

- **ファイル**: `hermeneus/src/mcp_server.py` (L598-608)
- **修正内容**: 同上のパターン — `_original_stdout` 保存 → `anyio.wrap_file(TextIOWrapper(...))` → `stdio_server()` に渡す

### 4. `mneme_server.py` の `_project_root` 未定義修正

- **ファイル**: `mekhane/mcp/mneme_server.py` (L55)
- **修正**: `_project_root` → `_base.project_root` に変更
- **原因**: リファクタリング時の変数名変更漏れ

1. **Sync ディレクトリの競合**: `~/oikos` → `~/Sync/oikos` のシンボリックリンク構造により、ファイル編集が Sync サービスで巻き戻される可能性がある。mneme_server.py の修正が一度巻き戻された

3. **MCP SDK v1.26.0 の変更**: 今後のアップデートで `stdio_server()` の API が変わる可能性。`sys.stdout` リダイレクトパターン自体を見直すことを検討

## 技術的知見

- MCP SDK v1.26.0 の `stdio_server()` は `stdin`/`stdout` パラメータを受け付ける
- `anyio.wrap_file(TextIOWrapper(original_stdout.buffer, encoding="utf-8"))` でラップすることで、リダイレクト後も元の stdout に書き込める
- IDE の Manage MCPs の Refresh ボタンはサーバーの再起動を行う — IDE 全体の再起動は不要な場合がある
