### 2. メモリ消費量の最適化 (-6.7GB 達成)

**根本原因**: 複数のプロセスが `DigestorPipeline` (BGE-M3 等の ML モデル) をプロセス内にロードしていた。

| プロセス | Before | After | 対策 |
|:---------|:------:|:-----:|:-----|
| Digestor scheduler | 6.7 GB | **43 MB** | API クライアント化 + subprocess フォールバック |
| HGK Gateway | 4.6 GB | ~60 MB | `hgk_digest_run` / `hgk_digest_list` を API 呼び出しに変更 |
| Periskopē MCP (孤児) | 2.7 GB | 消滅 | kill 済 |

**変更ファイル**:

- `mekhane/ergasterion/digestor/scheduler.py` — `run_digestor()` を API クライアント化。`DigestorPipeline` の global import 削除。
- `mekhane/mcp/hgk_gateway.py` — `hgk_digest_list` と `hgk_digest_run` を `127.0.0.1:9696/api/digestor/run` 経由に変更。

**systemd 変更**:

- `digestor-scheduler@makaron8426.service` を `systemctl disable` で自動起動を無効化。手動で修正版を再起動済み。
