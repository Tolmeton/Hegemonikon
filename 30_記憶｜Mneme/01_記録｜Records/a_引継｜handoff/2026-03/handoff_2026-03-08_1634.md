## 📍 実施内容

### HGK Gateway MCP コネクタ接続復旧

**問題**: Claude.ai の HGK Gateway コネクタが `hegemonikon.tail3b6058.ts.net` に接続できない。

**原因** (2層):
1. **Tailscale Funnel 未設定**: Gateway は `hgk` (100.83.204.102) に移行済みだが、Funnel が旧サーバー (`hegemonikon-1`, 5日前にオフライン) にしか設定されていなかった
2. **OAuth ドメイン不一致**: `.env` の `HGK_GATEWAY_URL` が `hgk.tail3d3bd5.ts.net` (Tailscale MagicDNS/ローカル名) になっていたが、Funnel 公開 URL は `hgk.tail3b6058.ts.net`。OAuth discovery の `issuer_url` が不一致で認証フロー失敗

**修正**:
1. `hgk` サーバーで `tailscale funnel --bg 8765` を実行
2. `.env` の `HGK_GATEWAY_URL` を `https://hgk.tail3b6058.ts.net` に修正 (`sed -i` で置換)
3. `hgk-gateway.service` を再起動
4. `.env` に immutable フラグ (`chattr +i`) を再設定

**検証結果**: OAuth Discovery 正常 / `/mcp` エンドポイント HTTP 401 / Claude.ai 接続成功

### Syncthing 送受信モード変更

WSL の Syncthing 3フォルダを `sendonly` → `sendreceive` に変更:
- `hegemonikon` (01_ヘゲモニコン)
- `oikos`
- `sync`

## ⚠️ 注意事項

### Syncthing と immutable フラグの非互換
- `chattr +i` はファイルシステム属性であり Syncthing の同期対象外
- Syncthing が `.env` を同期すると新規ファイルとして作成され、immutable フラグが消える
- **対策**: 各サーバーで個別に `chattr +i` を設定する運用が必要

### Tailscale DNS名の2種類
- **MagicDNS**: `hgk.tail3d3bd5.ts.net` — Tailnet 内部の名前解決 (ローカル)
- **Funnel**: `hgk.tail3b6058.ts.net` — インターネットからの公開 URL
- Gateway の `HGK_GATEWAY_URL` は Funnel の公開 URL を設定すること

## 🧠 信念 (Doxa)

- `.env` 操作時は N-4 の θ4.2 (view_file で事前確認) を厳守すべきだった。今回は `grep` で該当行のみ確認して `sed -i` で置換したが、`view_file` で全体を確認すべき
- Syncthing の `sendonly` → `sendreceive` 変更は REST API (`PATCH /rest/config/folders/{id}`) で即座に適用される
