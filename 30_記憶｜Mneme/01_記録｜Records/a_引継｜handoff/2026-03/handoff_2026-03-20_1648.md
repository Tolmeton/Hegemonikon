- LS の出力制限: 大量出力要求に対し ~14 トークンに圧縮し、~220 秒のレート制限を課す

**到達した洞察:**

1. **5 Whys**: API 直接で quota が減らない → LS が Baseline と Vertex TPM を別管理 → Baseline は IDE エージェント (Cascade) 経路のみ → Google が意図的に API 直接利用を Baseline 外に置いている → ユーザーの IDE 離れ防止のインセンティブ設計
2. **De-Contextualization**: SaaS の quota 管理は「経路別メーター」が一般的。同じバックエンドモデルでも、呼出経路によって異なる制限が適用される
3. **The Principle**: クラウドサービスの quota は単一メーターではない。経路 × 時間窓 × 課金層の直積空間で管理される
