# Handoff: MCP Repair & Google OAuth Setup

## 🎯 結論・実績
* **MCP サーバー修復**: `mcp_config.json` の `PYTHONPATH` を全9サーバー分修正（`_src`対応）。全サーバーが正常に起動・導通（ping）するようになった。
* **Google OAuth / ADC 復旧**:
  - `15_` のリモートパスから `.secrets/` の全トークンを `01_` に復旧（SA JSON 2つ、ADC 1つ、Token Vault 8つ）。
  - Tolmeton 用の Vertex Search SA キーを新たに作成。
  - WSL 上に gcloud CLI (v559.0.0) をネイティブインストールし、全6アカウントの `auth login` を完了。
  - デフォルトアカウントを `Tolmetes@hegemonikon.org` に設定。

## 💡 Value Pitch
* **わかる / FEP**: なぜ MCP が死んでいたのか？ → 単なるパス変更の追従漏れ（`PYTHONPATH` の環境非継承）という因果が見えた。
* **できる / Flow**: WSL から Windows 側の gcloud を呼ぶとハングする問題を、WSL ネイティブの apt インストールで回避。安定した CLI 操作が可能になった。
* **確か / Precision**: `gcloud auth list` や Python による ADC 疎通テストを通じて、全アカウントの認証状態を「推定」ではなく「確信」レベルまで引き上げた。

## 🧠 信念 (Doxa)
* ツール（gcloud）の実行環境（Windows vs WSL）の不一致は、見えないハングアップやタイムアウトを生む。ネイティブ環境にインストールすることが一番の解決策。
* `application_default_credentials.json` は1ファイル=1アカウントだが、Hegemonikón の Token Vault（`tokens/*.json`）がラウンドロビンを担うため、ADC はデフォルトアカウント1つで良いというアーキテクチャの再確認。
