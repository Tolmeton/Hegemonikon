### mneme TaskGroup エラー 根本原因分析

| 項目 | 詳細 |
|:-----|:-----|
| `_handle_search` | **同期関数** (async def ではない) |
| Hub dispatch | `await session.call_tool()` で正しく async |
| 問題箇所 | `asyncio.to_thread(_handle_search, ...)` から投げられた例外が MCP SDK 内部の TaskGroup に伝播 → `ExceptionGroup` |
| 修正 | `_handle_search_facade` で try/except し、例外を TextContent として返却。TaskGroup 伝播を阻止 |
| 効果 | Hub 経由呼出時に根本エラーの **スタックトレースが読める** ようになる |
