## ⚠️ 重大な学び: Antigravity の MCP 設定3層構造

```
Layer 1: ~/.config/Antigravity/User/mcp.json
         → Antigravity 内部プロキシが読む。stdio サーバーの起動定義。
         → ここを壊すと内部プロキシ (9443) が停止する

Layer 2: ~/.gemini/antigravity/mcp_config.json  
         → Antigravity AI agent が読む。接続先 URL の定義。
         → mcpServers + serverUrl 形式
         → 元の形式: http://localhost:970X/mcp (socat で HGK バックエンドにトンネル)

Layer 3: VS Code の mcp.json (layer 1 と同一ファイルだが role が異なる)
         → VS Code 拡張機能が読む。type: stdio / type: http をサポート
```

**Layer 1 と Layer 2 は全く別のファイル・別の形式・別の role。混同が3回の設定事故を引き起こした。**
