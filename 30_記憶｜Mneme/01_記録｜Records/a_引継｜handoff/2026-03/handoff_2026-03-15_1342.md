### 設計原理
- `cos(input, simple_anchor) - cos(input, complex_anchor)` = **意味的複雑度の直接測定**

- 負の値 → 複雑テキスト → explore (外部検索を注入)
- 閾値 ±0.02 は 11サンプルの実データから決定

## 🧠 信念 (Doxa)

- **Matryoshka の次元打ち切りは、API が normalized スライスを返す場合に意味をなさない** — cos sim は常に 1.0 になる。これは embedding API の仕様であり、モデルの性質ではない
- **意味的アンカーとの cos sim 差は、テキスト複雑度の良い代理指標になる** — FEP の epistemic value (explore) と pragmatic value (exploit) の二項分解に直接対応する
