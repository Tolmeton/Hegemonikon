## A: 行動と成果

### 1. Ξ_search アーティファクト判別

| 正規化 | ジニ | 判定 |
|:---|:---|:---|
| 生データ Var(σ²) | 0.953 | ❌ スケール差のアーティファクト |
| z-score | 0.019 | ❌ 定義上全次元 σ²=1 (検証不能) |
| **CV ベース** | **0.549** | **✅ 本物の不均一性** |

- 生データの高ジニ値は dim[0] (seq 長, 値域 0~数百) と dim[8] (比率, 値域 0~1) のスケール差が主因
- CV (変動係数 = std/mean) は**スケール非依存**であり、CV ジニ 0.549 は真の識別力不均一
- VISION §19.4 の Ξ_search 定義を `Var(σ²)` → **CV ジニ**に修正

### 3. ast_supplement → ccl_supplement 移行

**核心**: 旧 `ast_supplement` は Python AST から直接 16d を取得 = U (忘却関手) の外から情報を持ち込む設計上の不整合。

**分析**: 16d のうち 11d は CCL テキストから即時計算可能、残り 5d は既存 CCL 演算子で表現可能:
- assert → `V:{}` (ガード) — **既に `_stmt_to_ccl` で実装済み**
- raise → `!err()` (例外射) — **既に `_stmt_to_ccl` で実装済み**
- decorator → `^{}` (メタ/上昇) — **新規追加** (`_stmt_to_ccl` L215)
- starred → `*expr` (展開) — **既に `_expr_to_ccl` で実装済み**

**実装**: `ccl_supplement()` を作成し、`ccl_feature_vector()` から `ast_supplement()` への依存を除去。`ast_supplement` は `[非推奨]` マークで後方互換のため残存。
