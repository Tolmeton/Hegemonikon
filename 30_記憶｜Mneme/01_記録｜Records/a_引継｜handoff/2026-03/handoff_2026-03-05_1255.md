### 3. hermeneus `[Errno 2]` デバッグ・修正

**ファイル**: `hermeneus/src/mcp_server.py` L515-585
**根本原因**: `use_llm=False` 時に旧 `WorkflowExecutor` (LMQL パイプライン) に委譲されていた。LMQL ランタイムの subprocess 起動で `[Errno 2]`
**修正**: `MacroExecutor + CognitiveStepHandler` に統一。`use_llm=True/False` で同じ `MacroExecutor` を使い、step_handler だけが異なる対称設計に

**呼び出しチェーン（修正前）**:

```
mcp_server.py _handle_execute(use_llm=False)
  → executor.py WorkflowExecutor
    → _phase_execute() → runtime.py LMQLExecutor
      → import lmql → subprocess → [Errno 2]
```

**呼び出しチェーン（修正後）**:

```
mcp_server.py _handle_execute(use_llm=False)
  → macro_executor.py MacroExecutor(CognitiveStepHandler.handle)
    → テンプレート応答（subprocess 不要）
```
