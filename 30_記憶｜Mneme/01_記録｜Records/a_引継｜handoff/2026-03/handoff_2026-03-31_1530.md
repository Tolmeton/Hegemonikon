### 設計判断

| DECISION | 内容 | 棄却肢 | 理由 |
|:---------|:-----|:-------|:-----|
| D-4 キーワード競合 | ochema の「ブラウザ」→「デスクトップ」に変更し、opsis に「ブラウザ」を移管 | 両方に残す | KEYWORD_MAP の排他性原則 (同一キーワードで複数バックエンドが競合すると推奨精度が下がる) |
| D-5 日本語境界対策 | 「webページ」「ウェブ」を CJK キーワードとして追加 | 英語「web」のみ | `\bweb\b` が「webページ」にマッチしない (英語の単語境界が CJK 文字との接合部で成立しない) |

### 未検証

- Claude Code → `hub_execute` → opsis: Claude Code の MCP セッションが旧 Hub (opsis 追加前) をキャッシュしている。Claude Code 再起動で解決する
- Hub 本体の Session terminated 問題: 全リクエストで即座にセッション切断。opsis 無関係の既存 issue (Python MCP クライアントから `/mcp` へ直接接続すると `McpError: Session terminated`)。ただし `/mcp/hub` および `/mcp/opsis` 等の SessionManager プロキシ経由は正常動作
- systemd 有効化 (`systemctl --user enable --now hgk-mcp@opsis`) 未実施

## R — Recommendation

### 次回アクション

1. **Claude Code 再起動** — MCP セッション更新で `hub_execute` → opsis が動くようになる
2. **`systemctl --user enable --now hgk-mcp@opsis`** — systemd で永続化。テンプレート (`hgk-mcp@.service`) は対応済み
3. **Hub Session terminated 調査** — `/mcp` エンドポイントが全セッションを即切断する問題。`hub_mcp_server.py` のセッション管理ロジックか MCP SDK バージョンの問題
4. **フォーム入力 E2E テスト** — `opsis_act` の fill + submit を実サイトで検証
5. **ROLE_VERB_MAP 拡充** — 実サイトで新しい aria role を発見したら Layer 3 マッピングを追加

### 不確実性

- Hub の Session terminated は MCP SDK (mcp パッケージ) のバージョン不整合の可能性。`.venv/lib/python3.13/site-packages/mcp/` のバージョンを確認し、サーバー側とクライアント側で不一致がないか調べる
- agent-browser のセッション永続性は headless ブラウザプロセスに依存。長時間 idle 後のタイムアウト挙動は未検証
