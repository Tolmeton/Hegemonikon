## S: 状況
Paper I「力とは忘却である」の核心実験 — Ξ (忘却不均一度) と P (タスク持続能力) の相関を測定する `xi_experiment.py` の v1 に致命的な設計欠陥が4つ見つかった。全てを修正し v2 を完成させた。

## A: 評価 — 発見した4つの致命的欠陥と修正

| # | 欠陥 | 影響 | v2 修正 |
|---|------|------|---------|
| **D1** | P を eviction **前**の出力で evaluate | 全条件 P=1.00。Ξ と P が独立変数 | eviction 後の cache を `past_key_values` として generate に渡す |
| **D2** | Ξ_attn を eviction 前の attention で計測 | 全条件同じ値 | Ξ_attn を削除し Ξ_SVD のみ (SVD は eviction 後 cache に対して計測済み) |
| **D3** | `output_attentions=True` (全層) | O(n²) メモリ。長コンテキストで OOM | H2O ポリシーのみ最終層 attention hook で取得 |
| **D4** | `--device cpu` 指定 | TPU VM なのに CPU | `auto` デバイス検出 (TPU > CUDA > CPU) |

**D1 が核心**: eviction 後の cache で generate しないと、「忘却の仕方 (Ξ) がタスク持続能力 (P) に影響する」という因果を測定できない。

### 検証ポイント
- 軽量テストで P < 1.0 の条件が出ること (D1 修正の確認)
- `none` ポリシー (eviction なし) が最高 P であること (baseline)
- `uniform` ポリシー (Ξ≈低) < `h2o` ポリシー (Ξ=高) の P 差があること (定理の核心)

### 潜在リスク
- `model.generate(past_key_values=evicted_cache)` が DynamicCache の再構成で失敗する可能性 → `_build_evicted_cache()` のフォールバック (タプル形式) で対応済みだが、モデルによっては追加対応が必要
- H2O の attention hook がモデルアーキテクチャによっては動かない可能性 → フォールバック (ランダム heavy hitter) 実装済み
- TPU 上で `torch.linalg.svdvals` が動くか未検証 → CPU にフォールバックする可能性
