## 📍 現在地

### 達成したこと

1. **hub.py async 化** (`20_機構｜Mekhane/_src｜ソースコード/mekhane/mcp/gateway_tools/hub.py`):
   - `_call_hub_mcp_sync()` (L42 `asyncio.run()`) → `async def _call_hub_mcp()` (`await` ベース) に変更
   - 全5ツール関数 (`hub_stats`, `hub_recommend`, `hub_execute`, `hub_gate`, `hub_shadow_status`) を `async def` に変更
   - 不要な `import asyncio` 除去
   - **根本原因**: Gateway は既に asyncio ループ内で動作 → `asyncio.run()` は「実行中ループ内で新ループを起動」エラーを引き起こしていた

2. **`_traced` デコレータ async 対応** (`20_機構｜Mekhane/_src｜ソースコード/mekhane/mcp/gateway_tools/_utils.py`):
   - `inspect.iscoroutinefunction(fn)` で判定し、async 関数には `async_wrapper`、sync 関数には従来の `wrapper` を適用
   - 既存の sync ツール群（ochema, sympatheia 等）に影響なし

3. **Hub MCP 直接接続の実現性確認**:
   - `.mcp.json` に `"hub": {"url": "http://localhost:9700/mcp/hub"}` が定義済み
   - `~/.claude/settings.json` に `"mcp__hub__*"` allow 済み
   - localhost:9700 は稼働中 (pid=1603)、`tools/list` で5ツール応答確認済み
   - HGK Gateway (`mcp-proxy.anthropic.com`) は 502 Bad Gateway — バックエンドダウン

### 発見

- HGK Gateway 経由の Hub ツールは **Gateway 自体がダウンしている** (502) ため全滅
- Hub MCP サーバはローカルで正常稼働しているが、Claude Code がこのセッションで `mcp__hub__*` 接続を確立していない → **セッション再起動が必要**
