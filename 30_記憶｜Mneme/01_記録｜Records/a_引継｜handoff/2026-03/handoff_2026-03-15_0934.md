## 🔧 技術メモ

- **BLOB ヘッダ形式**: `[dim0:f64][dim1:f64]...[data:f64 array]` — 先頭に shape の各次元を float64 で格納、残りがデータ
- **旧形式フォールバック**: ヘッダなし BLOB は `len // 8` で要素数を推定し1次元配列として復元
- **SQLite BLOB vs np.save**: SQLite BLOB は shape/dtype を失う → ヘッダ必須。np.save (.npy) は NumPy 標準形式で shape/dtype を内包 → 追加ヘッダ不要
- **マイグレーション戦略**: レコード0件の場合は `DROP TABLE + CREATE TABLE IF NOT EXISTS` が最も安全。>0件の場合はリネーム→INSERT-SELECT→DROP で epoch→ISO8601 変換
