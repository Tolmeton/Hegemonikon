### 設計上の決定

| 決定 | 理由 | 却下案 |
|:-----|:-----|:-------|
| subprocess で gws CLI をラップ | gws が MCP を内蔵していないため CLI ラッパーが最も確実 | Node.js で直接 gws コードを呼ぶ → Python MCP サーバーとの不整合 |
| 環境変数 `GOOGLE_WORKSPACE_CLI_CONFIG_DIR` でアカウント切替 | gws が単一 config dir = 1 アカウントの設計 | 複数の gws プロセスを起動 → リソース浪費 |
| systemd --user サービス | 他の MCP サーバーと同じパターン | sudo systemd → テンプレートが /home にあるため --user が適切 |
