## 🔑 決定事項

| 決定 | 理由 | 却下肢 |
|:-----|:-----|:-------|
| 境界マッチを言語別に分離 | 英語の `\b` は CJK で無効、CCL は `/` プレフィックス | 統一的な部分文字列一致 → 誤マッチ多発 |
| Hub を systemd 管理 | SPOF リスク軽減 (`Restart=always`) | 手動起動 → 再起動時に復帰しない |
| Hybrid config (直接+Hub) | IDE の config ホットリロードが URL を更新しない | 全経由一括 → IDE が接続不能 |

---

## 💡 学んだこと

1. **IDE の MCP config は起動時に接続確立**: config ファイルの変更はツール一覧更新に反映されるが、接続先 URL は起動時のまま
2. **Hub のカウンタ (`total_calls`) はプロキシ通信のみ計上**: IDE が直接接続の場合は Hub を経由しないためカウントされない
3. **systemd テンプレート `hgk-mcp@.service` は Hub を含むポートマッピング済み**: 手動起動が不要になった
