## 1. 証拠: /exe+ → /ele+ 二段階で証明された座標限界

### 実験

`EXPERIMENT_gemma4_oblivion_profile.md` に対して:

1. **/exe+** (S×Negative, 知覚的批判) を実行 → 🔴3件検出
2. **/dio+** で修正
3. **/ele+** (I×Negative, 推論的批判) を実行 → 🟠5件追加検出

### /exe+ が検出したもの (運用的脆弱性)

| # | 問題 | 性質 |
|:--|:-----|:-----|
| 🔴1 | N<d 共分散のランク不足 | コード実装 (`np.cov`) の数値安定性 |
| 🔴2 | Cross-model比較のプロンプト統制欠如 | 実験手順の不備 |
| 🔴3 | VRAM見積もりにKV cache未算入 | 環境リソースの見積もり |

**共通点**: 全て **SOURCE に直接見える問題** (コード、文書の記述、HW仕様)

### /ele+ が追加検出したもの (理論的矛盾)

| # | 問題 | 性質 |
|:--|:-----|:-----|
| L-1 | sliding-window→global 遷移で統計多様体が不連続になりうる | 情報幾何の定義域の前提 |
| L-2 | D_KL の有限サンプルバイアスが層タイプ依存 | 推定量の統計的性質 |
| L-3 | Mistral-7B は sliding-window attention — 「均一対照」は誤分類 | アーキテクチャの事実確認 + 実験設計の論理 |
| L-4 | PCA d'=64 が最適である根拠なし | 次元削減の理論的妥当性 |
| L-5 | 「完全成功」基準が L-1〜L-4 の解決を暗黙に前提 | 成功基準の論理的整合性 |

**共通点**: 全て **推論による構造分解** で発見。SOURCE を「見る」だけでは到達不能

## 2. 根本原因分析 (5 Whys)

Why1: /exe+ が理論的前提を検出できなかった
Why2: /exe は S象限 (知覚的) — 「目の前にあるもの」の弱点しか知覚できない
Why3: S象限の定義 = Afferent:Yes, Efferent:No — 入力処理であり推論ではない
Why4: 推論的批判 (I象限) は /ele の責務であり、/exe は設計上そこに到達しない
**Why5 (構造的原因): /exe の Phase 3 (品質門) に「推論的走査の未実行」を検知する仕組みがない**

→ /exe は自分の限界を「知って」おらず、/ele への射を推奨できない

---

## 3. 設計方針 (Creator 承認済)

### 採用案: /exe → /ele の射を明示化

**/exe の Phase 3 (品質門) に「理論的前提チェック未実行 → /ele 射推奨」を追加**

```
Phase 3 品質門:
  既存チェック (RQS 6項目)
  + 新規: 理論的前提走査チェック
    判定条件:
      - 対象に数学的定義・公式・推定量が含まれるか → YES なら走査推奨
      - 対象に「対照群」「比較」「統制」が含まれるか → YES なら走査推奨
      - 対象に暗黙の前提 (「均一」「不変」等の無条件的主張) が含まれるか → YES なら走査推奨
    出力:
      ⚠️ 理論的前提の推論的走査が未実行。/ele への射を推奨
      → 推奨射: /exe >> /ele (知覚的批判 → 推論的批判のパイプライン)
```

### 却下案: /exe 自体に理論的推論を内包する

却下理由: S象限の座標を逸脱し、事実上 /ele を内包する → 座標系の純度が崩壊

### 設計原則

> /exe が自分の限界を知って「ここから先は /ele に渡すべき」と言えること。
> これが認知操作を分けている意味であり、座標系の存在理由。
> 各動詞は「自分にできないこと」を知っていることが、システム全体の堅牢性を保証する。

## 4. 具体的な証拠パターン (射推奨ロジックのテストケース)

### パターンA: 数学的定義域の前提

**入力**: 「Φ(l) = D_KL(p_l ‖ p_L) を計算する」
**/exe が見つけたもの**: N<d で共分散が数値不安定 (コードレベル)
**/exe が見つけられなかったもの**: D_KL のガウス近似が有限サンプルバイアスを持つ (統計理論レベル)
**射推奨トリガー**: 「数学的定義・公式・推定量が含まれる」→ /ele 推奨

### パターンB: 暗黙の無条件的主張

**入力**: 「対照群: Mistral-7B — 均一 self-attention」
**/exe が見つけたもの**: (なし — 知覚的には問題なく見える)
**/exe が見つけられなかったもの**: Mistral は sliding-window attention → 「均一」は事実誤認
**射推奨トリガー**: 「暗黙の前提 (均一/不変) が含まれる」→ /ele 推奨

### パターンC: 測定の理論的同等性

**入力**: 「パイプラインは model-agnostic 設計」
**/exe が見つけたもの**: E4B → 31B の互換推論が非自明 (API レベル)
**/exe が見つけられなかったもの**: model-agnostic の意味が「APIレベル」と「情報幾何レベル」で異なる
**射推奨トリガー**: 「model-agnostic/不変/汎用 等の無条件主張」→ /ele 推奨
