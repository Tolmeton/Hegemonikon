# ROM: Opsis バックエンド設計・実装記録
date: 2026-03-30
session: Opsis Web 視覚基盤 初期実装

## 設計判断

### DECISION-1: エンジン選択
- **Vercel Agent Browser** (Rust CLI, OSS, npm agent-browser@0.23.0)
- 棄却: Tandem (MCP非対応), Notte (SSPL), Browserbase SaaS (非OSS), Chrome DevTools MCP (重い)
- 理由: トークン効率 (200-400 tok/snapshot), .claude-plugin 統合, Rust CLI (軽量)
- 撤回条件: Vercel Agent Browser が非推奨/アーカイブされた場合

### DECISION-2: アーキテクチャ
- H5 (Claude Code に乗る) を前提に H1+H2+H3 を統合
- opsis = HUB MCP 12番目のバックエンド (port 9712)
- パイプライン: Agent Browser → opsis (DOM prune) → digestor/periskope (知識化) → mneme (永続化)

### DECISION-3: Layer 3 — HGK 認知動詞 Perception Layer
- Notte の perception layer (DOM→NL action API) に**随伴する独自実装** (インポートではない)
- Creator の洞察: 「インポートではなく随伴するのはありなのでは」→ 正しい
- VERB_DEPTH_MAP: 認知動詞ごとに snapshot パラメータを自動調整 (/ski=depth1, /the=depth5)
- ROLE_VERB_MAP: DOM 要素の role から HGK 認知動詞の可能操作をアノテーション

### DECISION-4: Crawl4AI Fit Markdown (H2 — 知識化レイヤー)
- digestor に `action='crawl'` を追加
- Crawl4AI 0.8.6 + Playwright chromium + PruningContentFilter
- opsis (Agent Browser) = 対話的 DOM 操作、Crawl4AI = 知識抽出 — 役割分離
- Windows cp932 + Rich ロガー問題: `verbose=False` + `PYTHONIOENCODING=utf-8` で解決
- 出力: `incoming/eat_web_{ts}_{slug}.md` (YAML frontmatter + Fit Markdown)

### DECISION-5: opsis → mneme 永続化 (H3)
- mneme に `store(action='web')` ツールを追加 — Sophia インデックスに Web コンテンツを永続化
- digestor crawl に `index=true` パラメータ追加 — crawl と同時に Sophia に自動 ingest
- Sophia の既存 ingest API (`SophiaIndex.ingest()`) を直接利用 — pkl 永続化
- doc_id: `web-{sha256(url)[:12]}` — URL ベースの冪等キー
- 検索テスト: score 0.884 でトップヒット確認

### DECISION-6: ローカル起動方針 (リモート集約を棄却)
- **棄却**: リモートサーバー (100.80.253.2) への opsis 集約
- **理由**: DOM はローカルブラウザで取るべき。リモートの Chrome が見る DOM ≠ Creator のデバイスの DOM
- **採用**: 各デバイスでローカル起動。Hub は常に localhost:9712 に接続
- mcp_ports.sh + run_mcp_service.sh + MODULES に opsis 追加
- systemd: `systemctl --user start hgk-mcp@opsis` で起動可能
- 各デバイスに必要: agent-browser (npm), crawl4ai + playwright chromium (pip)
