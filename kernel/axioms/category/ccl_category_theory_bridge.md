## §2 WF パイプライン = 射の連鎖の圏論的表現

### 2.1 基本構造: CCL パイプラインは enriched bicategory の 1-cell

CCL 式 `/noe+ >> /ele+ >> /ene+` は、圏 Cog 内の**1-cell の合成**:

```
f: /noe → /ele     (認識 → 反駁)
g: /ele → /ene     (反駁 → 実行)
g ∘ f: /noe → /ene (認識 → 実行)
```

> **核心**: この合成は **非結合的** (weak bicategorical)。
> `(f >> g) >> h ≅ f >> (g >> h)` — 同型だが等しくない。
> 認知的に: 「(理解→反駁)→実行」≠「理解→(反駁→実行)」。前者は慎重、後者は一気通貫。
> 📖 参照: weak_2_category.md §1 *(not yet published)* L36-46

### 2.2 Dokimasia パラメータの射への付随

各 1-cell は Dokimasia パラメータを伴う:

```
f = /noe[Pr:U, Vl:+]+  →  射 f: Noe → Ele  with  param(f) = {Pr:U, Vl:+, depth:L3}
```

これが Lawvere metric (Drift ∈ [0,1]) として Hom 豊穣を与える:
- **Drift が小さい** = 近い認知操作 (精度が高い変換)
- **Drift が大きい** = 遠い認知操作 (精度が低い変換)

### 2.3 プロセス演算子は余代数的射

`~` 系演算子は**余帰納的** (coinductive) であり、代数的射 (`>>`, `*`, `_`) とは質的に異なる:

| 性質 | 代数的 (`>>`, `*`) | 余代数的 (`~`, `~*`, `~!`) |
|:-----|:-------------------|:---------------------------|
| **時間** | 一発で完了 | 時間的に展開 |
| **圏論** | 射の合成 | Terminal coalgebra / Coinductive process |
| **FEP** | 一回の推論 | VFE 最小化ループ / Active Inference ループ |

これは圏論的に、Cog の中に2種の射が共存することを意味する:
- **代数的射**: `f: A → B` (構造的変換)
- **余代数的射**: `f: A → F(A)` (F は状態遷移関手) — 「A から次の A を生成」のプロセス

## §3 N-series (Nomoi) の CCL/圏論的表現

### 3.1 N-series の位置づけ: 忘却された構造の回復

> **核心的修正 (Creator 指摘 2026-03-18)**: N-series は構造を「注入」するのではない。
> 構造を「回復」する。回復には前提として**忘却の自覚** (= U-series) が必要。
> 「知覚なくして運動なし」— U (忘却検出) → N (回復) が正しい順序。

N-series (12 Nomoi) は **忘れられた構造を知覚し、回復するための認知操作**。

圏論的に: **U-series が ker(U) (忘却の核) を検出 → N-series がその核を埋め戻す**

```
1. U_i が「忘れている構造」を知覚する (忘却関手の核の検出)
2. N_i がその構造を回復する:

  [忘却状態]  ──U_i検出──→  [自覚]  ──N_i回復──→  [構造保持状態]
       A                      ker(U)                    A'

  U_i: 何を忘れているかを知覚する
  N_i: 忘れていた構造を回復する
```

## §4 U-series の CCL/圏論的表現

### 4.1 U-series の位置づけ: 忘却関手の検出 = 反自然変換

U-series (B22-B34) は **忘れてしまう構造を検出するパターン**。

圏論的に: **U-series = 忘却関手 U: Rich → Poor の検出条件**

```
U は「必要な構造を忘れてしまう」関手:
  Rich (豊かな圏)    →U→    Poor (貧しい圏)
  射を持つ         ↦→      対象だけ
  精度ラベルつき   ↦→      ラベルなし
  因果構造つき     ↦→      相関だけ
```

### 4.3 N↔U の双対構造: 構築 vs 検出

N-series と U-series は**双対的**:

| | N-series | U-series |
|:---|:---------|:---------|
| **方向** | 構造を**構築**する (η: Id ⟹ Structure) | 構造を**忘れる**のを**検出**する (U の核 = ker(U)) |
| **態度** | 処方的 (「こうせよ」) | 診断的 (「これを忘れていないか」) |
| **圏論** | 自然変換 | 忘却関手 |
| **FEP** | Complexity 削減 (正しいモデルの構築) | Accuracy 低下の検出 (情報の逸失) |

> [主観] この双対性は **随伴** `N ⊣ U` として形式化できる [仮説 55%]。
> N が構造を注入し、U がそれを忘れるなら、N ⊣ U は「必要最小限の構造」を Fix として与える。
> Fix(U ∘ N) = 実際に必要な制約のみが残る不動点 = Kalon。

| Q13 Sc→Vl | 4 | /ops→/ele, /lys→/beb | 粒度→感情変化を伴う WF 実行 |
| Q11 Vl→Pr | 23 | /ele→/kat, /beb→/dok | 感情→確信変化を伴う WF 実行 |

Q11 (Vl→Pr) は n=23 で十分だが、**Q10 (Pr→Sc) は n=1**。
これは Pr→Sc 遷移 (確信度→粒度変更) が稀にしか発生しないことを示す。

```
収集戦略 (暫定):
1. /kat+ → /lys+ (確信固定後に詳細分析) を意識的に実行
2. /dok+ → /ops+ (打診後に俯瞰) のペアを増やす
3. tape 分析で Q10 遷移を capture するフィルタを追加
```

> [主観] ⑤ は最も検証困難な循環だが、「Dunning-Kruger の循環的定式化」として
> 検証できれば極めて興味深い。ただし n=1 を n≥10 に上げるには相当な WF 実行が必要。
> 急ぐ必要はなく、通常の作業の中で自然に蓄積するのを待つのが現実的。
