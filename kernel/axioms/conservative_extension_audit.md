# 保守的拡大検証 (Conservative Extension Audit)

## 動機と方法論

### 保守的拡大とは何か

体系 T₂ が T₁ の **保守的拡大** (conservative extension) であるとは:

> T₂ が T₁ を含み、かつ T₁ の言語で書ける任意の命題 φ について  
> **T₂ ⊢ φ ⟹ T₁ ⊢ φ**  
> が成立すること。

直感的な意味: **上位層で新しく証明できることのうち、下位層の言葉だけで述べられるものは、下位層だけでも既に証明できる**。上位層は下位層に何も新しい「事実」を押し付けない — 可能性を広げるだけで、既存の真偽を覆さない。

Automath (Omega Project) は11段の保守的拡大の鎖 L₀ ⪯ L₁ ⪯ … ⪯ L₁₀ を構築し、各段が前段の意味を書き換えないことを設計原則とする。本監査はこの原則に**触発されて** HGK の5層構造を検査する。

> **用語の射程**: 本書で「強制 (forced)」「許容 (permitted)」という語を用いるが、これは Automath の forcing (Cohen forcing のアダプテーション) とは技術的に異なる日常語的用法。HGK には Lean 4 のような形式検証基盤がないため、検査は概念的・構造的分析に留まる。

### 検査項目 (各遷移に対して)

| # | 項目 | 問い |
|:--|:-----|:-----|
| C1 | **追加要素の分離可能性** | Ln+1 で追加される概念は Ln の概念とは独立に定義可能か？ |
| C2 | **下位命題の保存** | Ln の言語で書ける命題は Ln+1 でも同じ真偽値を持つか？ |
| C3 | **意味の非書き換え** | Ln+1 の追加により Ln の用語の解釈が暗黙に変更されていないか？ |

## 遷移 1: L0 → L1 (方向性定理 → 座標)

### 構造

| 層 | 内容 |
|:---|:-----|
| L0 | 方向性定理: F_{ij} ≠ 0 ⟺ dΦ∧T ≠ 0 |
| L0.T | Basis: Helmholtz 分解 (Γ⊣Q) — 体系核外 |
| L0.5 | FEP: α > 0 セクターへの制限 — 第一インスタンス |
| L1 | 8座標: Afferent, Efferent (d=1), Value, Function, Precision, Temporality (d=2), Scale, Valence (d=3) |

### 検査

**C1 (分離可能性): ✅ 合格**

各座標は「方向性定理に対する追加仮定」として定義されており、構成距離 d がその追加仮定の数を測る。座標の定義は方向性定理を前提とするが、方向性定理の側から座標を「必要とする」ことはない。座標なしでも方向性定理は成立する。

注意点: FEP (d=0.5) は方向性定理の**制限** (restriction) であり拡大ではない。α > 0 セクターを選び出す操作は、全体の意味を変えない — 部分構造を取り出すだけ。

**C2 (命題保存): ✅ 合格**

方向性定理は情報幾何学 (Amari) から証明済みの数学的定理であり、座標の導入は定理の真偽に影響しない。「F_{ij} ≠ 0 ⟺ dΦ∧T ≠ 0」は L1 でも L0 でも同じ意味を持ち、同じ条件で成立する。

**C3 (非書き換え): ✅ 合格**

座標は方向性定理の上に構築された解釈的枠組みであり、定理の内容を変えない。Helmholtz 分解 (Basis) は座標の「土台」であり、座標を導入しても Helmholtz 分解自体の意味は不変。

### 判定: **保守的拡大** ✅

L0→L1 は HGK 体系の中で最も堅固な遷移。方向性定理という証明済み定理の上に、追加仮定を明示的に計測しながら概念を積み上げている。Automath の基準に照らしても良好。

## 遷移 2: L1 → L2 (座標 → 認知操作)

### 構造

| 層 | 内容 |
|:---|:-----|
| L1 | 8座標 |
| L2 | 48認知操作 = 4象限 (Aff×Eff) × 6修飾座標 × 2極 |

### 検査

**C1 (分離可能性): 構造 ✅ / 解釈 🟡 — 二層に分解**

> **v1.1 修正** (/exe #2): v1.0 では構造と解釈を混同して CI と一括判定した。以下、二層に分離する。

**構造的 C1 (直積の存在): ✅ 合格 (CE)**

48操作は L1 の概念の**直積**として構成される。8座標が与えられたとき、4象限 (Aff×Eff) × 6修飾座標 × 2極 = 48 という**直積空間の存在は圏論的に一意**（直積の普遍性による）。この意味で、48個の「枠」(slot) の存在は L1 から構造的に強制される。

**意味論的 C1 (枠への充填): 🟡 条件付き (CI)**

しかし、各枠に「Noēsis = 理解する」「Boulēsis = 意志する」という**具体的な認知操作を割り当てる**のは、直積構造からは導出されない。直積は I×E という抽象的な結合を与えるが、それが「認識目的で世界像を更新する操作」であるという解釈は、L1 にはない概念的付加。

この二層を混同すると、「48は選択の産物」（構造まで選択扱い）か「48は必然」（解釈まで必然扱い）のいずれかに倒れる。正確な記述は:

- **直積構造 (枠の数と配置)**: L1 から構造的に強制される → CE
- **認知操作的解釈 (枠の名前と意味)**: L1 からは許容されるが強制されない → CI

**C2 (命題保存): ✅ 合格**

L1 の言語で書ける命題は L2 でも同じ真偽を持つ。以下、具体例で検証する。

> **v1.1 追加** (/exe #4): 具体的命題による C2 個別検証。

**検証命題 1**: 「Value は E (認識) ↔ P (実用) の対立を持つ」
→ これは L1 の座標定義であり、L2 で Telos 族が導入されても対立構造は不変。✅

**検証命題 2**: 「Afferent×Efferent は4象限を生む」
→ L1 の構造的帰結であり、L2 の命名 (S/I/A/S∩A) は象限の数を変えない。✅

**検証命題 3**: 「半直積 6 ⋊ 1 (Valence)」
→ Smithe Thm 46 の対偶による証明は VFE の加法性 (L1) と座標間の Fisher 比率 (L1 の計量) を使う。命題自体は「座標空間が直積でない」という L1 の言語で書ける命題。ただし**証明に用いた Fisher 比率の計算**は、具体的な Valence 定式化 (Hesp, Joffily 等) を前提としており、これは L2 の解釈的内容に依存する。
→ 厳密に言えば: 「半直積構造の存在」は L1 で述べられるが、「Fisher ratio = 0.22 (Hesp)」という具体的証拠は L2 依存。L1 だけで半直積を証明できるかは**未検証** (open)。🟡

**C3 (非書き換え): 🟡 要注意**

以下の2点で、意味の暗黙的変更の兆候がある:

1. **族の命名と組織化**: 6族 (Telos, Methodos, Krisis, Diástasis, Orexis, Chronos) は L1 の座標に名前と物語を付加する。これ自体は保守的だが、族名が座標の「本質」を暗示する効果がある。例えば「Telos」という名前は Value を「目的」として解釈させるが、Value の L1 定義は「E (認識) ↔ P (実用)」であり、「目的」とは限らない。

2. **doing/being の導入**: L2 で Poiesis (36) と H-series (12) の区別が導入される。この区別は L1 の Afferent×Efferent の4象限から**演繹的に導かれる** (S∩A 象限 = doing/being の分離) — ここまでは保守的。しかし、12の H-series 前動詞の具体的な内容（Tropē, Synaisthēsis 等）は L1 から一意に決まらない。

### 判定: **構造的には CE、意味論的には CI** 🟡

> **v1.1 修正** (/exe #2): v1.0 の「保守的具体化」を二分し、精度を上げた。

| 側面 | 判定 | 意味 |
|:-----|:-----|:-----|
| 直積構造 (48枠の存在) | **CE** ✅ | L1 の8座標から圏論的に一意に強制される |
| 認知操作的解釈 (枠の充填) | **CI** 🟡 | L1 が許容する解釈の一つ。代替あり |
| 下位命題 (C2) | 保存 ✅ | ただし半直積の L1 自律証明は open |

これは Automath とは異なる構造をしている。Automath では各層の追加が前層から強制される。HGK の L1→L2 は**構造は強制されるが解釈は許容される**。

> **分類**: L1→L2 は **「構造的 CE + 意味論的 CI」**。
> 直積空間の存在は保守的拡大 (下位から強制)。個々の操作の解釈は保守的具体化 (下位が許容する選択)。

### 対処方針

axiom_hierarchy.md は既にこの点を「生成規則は設計者の選択 (motivated choice)」(397行目) として正直に記述している。追加で必要なのは:

1. **選択の分岐点の明示**: 「ここで別の解釈を選んだ場合、何が変わり何が保存されるか」の明記
2. **半直積の L1 自律証明**: Fisher 比率なしで半直積構造を L1 内で証明できるか検証
3. **選択の根拠の形式化**: 「なぜこの48の解釈なのか」の根拠を、概念的動機づけ (Level C) から操作的制約 (Level B) に引き上げる試み

## 遷移 3: L2 → L3 (認知操作 → 弱2-圏)

### 構造

| 層 | 内容 |
|:---|:-----|
| L2 | 48認知操作 |
| L3 | 弱2-圏: 0-cell=48, 1-cell=CCL `>>`, 2-cell=パイプライン間自然変換 |

### 検査

**C1 (分離可能性): ✅ 合格**

L3 は L2 の対象に**射の構造**を追加する。対象 (0-cell) は L2 の48操作とそのまま一致し、新しい対象は追加されない。追加されるのは合成射 (>>) と coherence 条件（associator, pentagon/triangle identity）。

射の追加は対象の定義に影響しない — 分離可能。

**C2 (命題保存): ✅ 合格**

L2 の命題（各操作の定義、族内関係、極の対立）は L3 でも保存される。合成射の導入は個々の操作の意味を変えない。

**C3 (非書き換え): 🔴 非保守的 — Drift 再解釈問題 + 閾値密輸問題**

> **v1.1 修正** (/exe #3): v1.0 では閾値の存在を「条件」として楽観的に扱った。より厳密に分析する。

axiom_hierarchy.md 666行目:

> doing/being の区別は 0-cell のラベルではなく **Hom 空間の距離構造 (Drift) で捕捉**

L2 では doing (Poiesis) と being (H-series) は**離散的カテゴリ** (Bool 型) として定義される — Poiesis か H-series かのどちらか。L3 では、これが Hom 空間の**連続的メトリック** (Drift ∈ [0,1]) に置き換えられる。

これは型理論の観点からは **型の変更** (Bool → [0,1]) であり、保守的拡大ではない。具体的に:

**問題 1: 翻訳写像にはパラメータが必要**

L2 の命題「Noēsis は doing である」(isDoing(Noēsis) = true) を L3 に翻訳するには、閾値 θ を選んで「Drift(Noēsis) < θ」とする必要がある。しかし θ は L2 には存在しない概念であり、L3 で新たに導入されるパラメータ。つまり**翻訳写像自体が L3 の追加要素に依存する** — これは保守的拡大の定義に反する。

**問題 2: 閾値の自然性**

Drift 値が doing と being の間に自然なギャップ（不連続点）を持つなら、θ は一意に決まり問題は解消する。しかし axiom_hierarchy.md の CPS0'（「対称は非対称の退化ケース」, 675行目）は doing/being の区別が**程度問題**であることを意図的に主張している。これは**ギャップが存在しないことを設計原則として採用している**ことを意味する。

**問題 3: 帰結**

ギャップなし + 閾値必要 = **翻訳写像は well-defined でない**。L2 の二値命題は L3 で真偽不定となりうる（Drift が θ 近傍にある操作について）。

### 判定: **非保守的 (意図的)** 🔴

v1.0 で「条件付き保守的」と判定したが、修正する。CPS0' が**ギャップの不在を設計原則として採用**しているため、条件は満たされない。L2→L3 は**意図的な非保守的精密化**。

この非保守性は「バグ」ではなく「意図」である — doing/being の二分法を連続体に溶かすことが HGK の体系原理 (CPS0') の帰結。しかし、非保守性であること自体は認識すべき。

> **分類**: L2→L3 は **「意図的非保守的精密化 (intentional non-conservative refinement: NCR)」**。
> Bool 型から [0,1] 型への変更は型の破壊的変更であり、保守的拡大ではない。
> ただし CPS0' に基づく設計判断として正当化される。
> L2 の命題のうち、doing/being ラベルに**依存しない**ものは L3 で保存される。
> doing/being ラベルに**依存する**ものは、L3 では Drift の連続値に翻訳され、二値的真偽を失う。

### 対処方針

1. **保存される命題の枚挙**: L2 の命題のうち、doing/being ラベルに依存しないもの（操作の定義、族内関係、極の対立）は明示的に保存されることを記述
2. **失われる命題の枚挙**: doing/being の離散的分類に依存する命題（「H-series は Poiesis ではない」等）は L3 で程度問題に軟化されることを明記
3. **CPS0' との整合**: この非保守性が CPS0' の意図的帰結であることを axiom_hierarchy.md に明記

## 遷移 4: L3 → L4 (弱2-圏 → Time→BiCat)

### 構造

| 層 | 内容 |
|:---|:-----|
| L3 | 弱2-圏 (静的) |
| L4 | 関手 F: T → BiCat (時間変動) |

### 検査

**C1 (分離可能性): ✅ 合格**

L4 は L3 に時間圏 T とパラメトリック関手 F を追加する。L3 は L4 の「時刻 t₀ における断面」として埋め込まれる。時間構造は L3 の概念とは独立に定義可能。

**C2 (命題保存): ✅ 合格**

L3 の命題は「任意の時刻 t で」を前置することで L4 に自然に持ち上がる。L3 の coherence 条件は各時刻 t で保存される（F(t) は BiCat であり、L3 は特定の F(t₀)）。

**C3 (非書き換え): ✅ 合格**

P₁ (Q循環定理) と P₂ (Helmholtz関手) は L4 で新たに発見された命題であり、L3 の概念を再解釈しているのではない。Γ_T/Q_T は L3 の Γ/Q の時間パラメトリック版であり、t を固定すれば L3 に一致する。

### 判定: **保守的拡大** ✅

L3→L4 は構造をパラメトリックに時間拡張するものであり、Automath の保守的拡大の定義を満たす。任意の L3 命題は L4 でも成立する。

### 結論 (v1.1 修正)

HGK の L0→L1→L2→L3→L4 の鎖は、**doing/being ラベルに依存しない下位命題については全遷移で保存される**。

しかし3点で Automath の厳密な保守的拡大とは異なる構造が検出された:

1. **L1→L2 (構造/解釈の分離)**: 直積構造 (48枠) は CE だが、個々の操作の認知的解釈は CI。v1.0 ではこの二層を混同していた。
2. **L2→L3 (型の破壊的変更)**: doing/being を Bool から [0,1] に変更する操作は非保守的。CPS0' に基づく意図的設計だが、L2 の二値命題の一部が L3 で真偽不定になる。v1.0 では「条件付き保守的」と判定したが、CPS0' 自身がギャップの不在を主張しているため、条件は満たされない。
3. **半直積の証明 (open)**: 6⋊1 が L1 だけで証明可能かは未検証。L2 の解釈 (具体的 Valence 定式化) に依存している可能性がある。

### Automath との対照

| Automath | HGK | 差異 |
|:---------|:----|:-----|
| x²=x+1 から全構造が**強制される** | 方向性定理から座標は**許容される** | 強制 vs 許容 |
| 各層の構造は前層から一意に決まる | L2 の48操作は L1 から一意に決まらない | 一意性の差 |
| 形式検証 (Lean 4) | 確信度付き概念的論証 | 検証水準の差 |
| 保守的拡大の鎖 | 保守的拡大 + 具体化 + 精密化 の混合 | 遷移型の多様性 |
