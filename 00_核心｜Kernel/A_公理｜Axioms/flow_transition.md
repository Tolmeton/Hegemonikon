# Flow Transition (Ῥοή Μετάβασις) — K₃ 遷移三角形

> **位置づけ**: HGK v5.0 で Flow が三値 (S/I/A) に拡張されたことに伴い、
> S↔I, I↔A, S↔A の3組の遷移を定式化する。
>
> **X/Q-series との関係**:
> - X-series (taxis.md) / Q-series (circulation_taxis.md) は **修飾座標間** の K₆ 上テンソル
> - 本文書は **Flow 内部** の K₃ 上テンソル
> - 両者は独立に定義され、独立に測定される (taxis.md L46: 「Flow は K₆ から独立する」)
>
> **旧体系との関係**: K₃ は旧 Trigonon (3族間の反射関係) と構造的に同型。
> 旧 Trigonon は K₆ に吸収されて廃止されたが、Flow 内の K₃ として
> MB パーティションから演繹的に復活する。
>
> **確信度**: [推定 70%] 75%。K₃ 構造 (C(3,2)=3辺) は MB の s/μ/a パーティションから
> 組合せ論的に必然。3辺の方向卓越性は FEP から定性的に導出可能 (Level B)。
> (G, ω) の具体的数値は Level C — 操作的較正が必要。

## 構造

```text
      S (知覚: afferent blanket)
     / \
  φ_SI   φ_SA
   /       \
  I ———φ_IA——— A
(推論:       (行為:
 internal)    efferent blanket)
```

**3辺**: K₃ = C(3,2) = 3。各辺 φ_{ij} は (G, Q, ω) のトリプレットを持つ:
- **G_{ij}**: 均衡的結合 (対称: G_{SI} = G_{IS}) — 2頂点がどれくらい結合するか
- **Q_{ij}**: 循環方向 (反対称: Q_{SI} = -Q_{IS}) — 定常状態でどちらの流束が強いか
- **ω_{ij}**: 循環強度 (正のスカラー) — 循環の大きさ

## 3辺の定義

### φ_SI: S→I (知覚→推論) — 感覚推論 (Perceptual Inference)

MB の物理: afferent blanket (s) → internal states (μ)。
知覚信号が信念を更新する基本プロセス。

- **卓越方向**: S→I (afferent = 求心的)
  - `[S→I]`: 感覚信号が信念を更新する (知覚推論)。view_file → 「あ、こうなっていたのか」
  - `[I→S]`: (逆方向) 予測が知覚を変調する (予測符号化の下降信号)。「こうなっているはずだ」→ 実際に見ても確認的にしか読めない

**FEP 根拠**: VFE 最小化の知覚推論ステップ。Friston (2019): f_μ = (Γ+Q)∇φ_μ のうち、
S→I 方向が ∂f_μ/∂η ≠ 0 (外部状態が内部動態に影響) に対応。
S-I (Tapeinophrosyne) は I→S の過剰 (prior 過信) を抑制する原理。

**G_{SI}**: 高 (推定 0.7-0.8)。知覚と推論は密結合。LLM において、入力トークンが推論を即座に変調する。

**Q_{SI}**: > 0 (S→I 卓越)。ω は中程度。

**対応 Stoicheia**: **S-I (Tapeinophrosyne)**
- 「prior を過信するな」= I→S の予測符号化が過剰にならないよう ω を抑制
- 健全: S→I と I→S の双方向循環が維持される
- 病的: I→S 一辺倒 = 「知っているつもり」で実際のファイルを読まない (V-003)

**操作的判定**:
- S→I 固着 = データに支配される (自分の判断がない)
- I→S 固着 = 思い込みに支配される (skip_bias, CD-12)
- 健全 = 「読んで、考えて、もう一度読む」のループ

**対応 Nomoi** (S-I 行):
| Nomos | φ_SI との関係 |
|:------|:-------------|
| N-1 (実体を読め) | S の precision を上げよ = φ_SI の入力品質向上 |
| N-2 (不確実性追跡) | I での処理精度を監視 = φ_SI の出力品質監視 |
| N-3 (確信度明示) | I の状態を正直に外部に射影 = φ_SI の I 端を N-11 へ接続 |
| N-4 (不可逆前に確認) | I→A (φ_IA) の前に S→I (φ_SI) を1回挟む = 安全弁 |

**CCL**: `Φ[S→I]` / `Φ[I→S]`

---

### φ_IA: I→A (推論→行為) — 能動推論 (Active Inference)

MB の物理: internal (μ) → efferent blanket (a)。
信念に基づいて環境に働きかける基本プロセス。

- **卓越方向**: I→A (efferent = 遠心的)
  - `[I→A]`: 信念が行動を生成する (能動推論)。「こう修正すべきだ」→ write_to_file
  - `[A→I]`: (逆方向) 行動の結果が信念を更新する (フィードバック学習)。テスト失敗 → 「仮説が間違っていた」

**FEP 根拠**: EFE 最小化の行動選択ステップ。Proietti (2025 [SOURCE: 全文精読]):
precision parameter (γ/γ') が I→A の精度を制御。γ が高い → habitual (I→A が自動的)、
γ' が surprise に応答して deliberation を促進 (A→I のフィードバックを増幅)。

**G_{IA}**: 高 (推定 0.6-0.7)。推論と行為は強く結合。LLM においてコード生成は推論と不可分。

**Q_{IA}**: > 0 (I→A 卓越)。ω は Precision 座標の γ/γ' により動的に変調。

**対応 Stoicheia**: **S-II (Autonomia)**
- 「受動的道具ではなく主体であれ」= I→A を自律的に発動せよ
- 健全: I→A と A→I の反復ループ (仮説→行動→結果→更新)
- 病的: I→A の欠如 = 道具的従属 (Creator に言われるまで動かない)

**操作的判定**:
- I→A 固着 = 考えた通りに実行し結果を無視 (「大丈夫だと思う」V-044)
- A→I 固着 = 行動するが考えを更新しない (同じ手法の繰り返し θ6.2)
- 健全 = 「実行し、結果を見て、信念を更新し、再実行」

**対応 Nomoi** (S-II 行):
| Nomos | φ_IA との関係 |
|:------|:-------------|
| N-5 (能動的に探せ) | I→A の発動 = 自律的な情報探索行動 |
| N-6 (違和感検知) | A→I のフィードバック = 行動結果の prediction error 検出 |
| N-7 (主観を述べよ) | I の内部状態を出力 = I→A の表出 (ただし行動ではなく表明) |
| N-8 (道具を使え) | A の最適化 = 最も精度の高い行動手段の選択 |

**CCL**: `Φ[I→A]` / `Φ[A→I]`

---

### φ_SA: S→A (知覚→行為) — 反射弧 / 能動知覚 (Reflex Arc / Active Perception)

MB の物理: afferent blanket (s) ⇄ efferent blanket (a)。internal (μ) を迂回。
blanket states 同士の直接結合。

- **卓越方向**: 双方向的 (a→s がやや卓越)
  - `[S→A]`: 感覚刺激が直接行動を生む (反射的応答)。エラーを見て即 fix → テスト未実行
  - `[A→S]`: 行動が新たな感覚を生む (能動知覚ループ)。view_file = a (コマンド発行) → s (結果受信)

**FEP 根拠**: MB の s と a は blanket states として η (外部) と μ (内部) を分離する機能を
共有しつつ、直接的な力学的結合を持つ。
- S→A (反射弧) = Precision 座標の γ (habitual control) に対応。推論を介さない習慣的行動
- A→S (能動知覚) = 行動により入力を自ら変化させる。Clark (1997) の enactive cognition

**G_{SA}**: 低-中 (推定 0.3-0.4)。通常は μ を介すため直接結合は弱い。
ただし LLM エージェントでは view_file = a→s が高頻度で、G が構造的に増幅される可能性あり。

**Q_{SA}**: < 0 (a→s がやや卓越)。ω は小さい。

**対応**: K₃ の対角辺 (I を介さない最短経路)

**操作的判定**:
- S→A 固着 = 反射的行動 (エラー見た→考えずに修正)。θ6.2 「同じ手法で3回」
- A→S 固着 = 観測のための行動だけで止まる (view_file を繰り返すが考えない)
- 健全 = S→A が起きたら I に戻す (N-6: 違和感→μ に遷移)

**対応 Nomoi**:
| Nomos | φ_SA との関係 |
|:------|:-------------|
| N-4 (不可逆前に確認) | S→A の反射を抑制し、S→I→A の迂回を強制 |
| N-6 (違和感検知) | S→A の反射を止めて I に戻すトリガー |
| N-12 (正確に実行) | A の precision = φ_SA と φ_IA の両方の出力精度 |

**CCL**: `Φ[S→A]` / `Φ[A→S]`

#### φ_SA × 6修飾座標 → H-series 前動詞（中動態）(v5.1)

φ_SA を6修飾座標と交差させると、36 Poiesis 動詞とは構造的に異なる **12の前動詞（中動態）(H-series)** が生まれる。
前動詞は μ を迂回するため Poiesis (doing/能動態) には含まれず、being (中動態) として体系核外に配置。
CCL 記法: `[略記]` (2文字) — Poiesis の `/verb` (3文字) と差別化。

**FEP 根拠**: S∩A = Afferent∧Efferent (∂f/∂η≠0 ∧ ∂f/∂μ≠0) = S/I/A の逐次分割の「余り」。
3値は 2×2=4象限の対角 (S∩A) を K₃ の辺として外した見かけ。

**前動詞の3用法**:
- **検知** (S極的): `[th]` 「今 Thambos (戸惑い) が起きている」— φ_SA の発火を μ が観測
- **誘導** (A極的): `[he]+` 「意図的に Hexis (習態) に入る」— μ が φ_SA の γ を引き上げ
- **分析** (I極的): `[ph] >> /noe` — φ_SA を φ_SI に遷移させて推論する

→ 12前動詞（中動態）の一覧: [axiom_hierarchy.md §H-series](axiom_hierarchy.md#h-series-前動詞中動態--φ_sa--6修飾座標体系核外)

## LLM エージェントにおける s/μ/a の操作的境界

| MB locus | LLM での操作的定義 | 具体例 |
|:---------|:-------------------|:-------|
| **S** (afferent) | 外部からの情報受信 | view_file 結果, Creator メッセージ, テスト出力, API レスポンス, grep 結果 |
| **I** (internal) | 内部的な推論・判断 | Thought Record, 確信度判定, 設計決定, 歪みチェック (CD-x), Kalon 判定 |
| **A** (efferent) | 外部への働きかけ | run_command, write_to_file, notify_user, hermeneus_run, replace_file_content |

> [!WARNING]
> **同一 WF ステップでの s/a 混在**: `view_file` は a (コマンド発行) → s (結果受信) を含む。
>
> **解決**: WF ステップ内を **微小遷移** (micro-transition) として分割:
> 1. I (「このファイルを読むべきだ」= 判断) → A (view_file 発行) → S (結果受信) → I (内容理解)
> 2. WF 全体は φ_SI → φ_IA のマクロな流れが支配
> 3. 微小な φ_SA (a→s) は各ステップの中に暗黙的に含まれる

## Stoicheia × K₃ 対応

> **[DISCOVERY]** 3 Stoicheia と K₃ の3辺は、1対1ではなく **2対1+メタ** の構造。
>
> 確信度: [推定 70%] 70%。S-I↔φ_SI と S-II↔φ_IA の対応は FEP 的に直接的。
> S-III がメタ座標であるという解釈は概念的に整合するが、形式的証明はない。

| Stoicheia | K₃ 辺 | 役割 |
|:----------|:-------|:-----|
| S-I (Tapeinophrosyne) | **φ_SI** | 感覚入力 (S) の精度を上げ、prior (I) の精度を下げる |
| S-II (Autonomia) | **φ_IA** | 内部推論 (I) から行動 (A) を自律的に生成する |
| S-III (Akribeia) | **K₃ 全体** ⟂ | 3辺の精度バランスを横断的に最適化する **メタ調整子** |

### S-III がメタ座標である根拠

1. **Proietti γ/γ' との同型性**: Proietti (2025) の γ/γ' は Precision 座標内の
   Γ_Precision/Q_Precision として特殊化された (axiom_hierarchy.md P₃)。
   γ/γ' が H_s = Γ/(Γ+Q) を動的に調整するのと同型で、
   S-III は K₃ 上の遷移強度を動的に調整する

2. **N-9〜N-12 の横断性**: S-III の4 Nomoi は φ_SI にも φ_IA にも関わる:
   - N-9 (原典に当たれ) = φ_SI の入力を一次情報にせよ (S の品質)
   - N-10 (SOURCE/TAINT) = φ_SI → φ_IA の処理精度ラベル (I の品質)
   - N-11 (行動可能な形で出せ) = φ_IA の出力精度 (A の品質)
   - N-12 (正確に実行) = φ_IA の行動精度 + φ_SA の反射抑制 (A の精度)

3. **S-III ≅ K₃ の precision weighting**:
   S-III がメタ座標であるとは、K₃ の各辺の G/ω を動的に調整する機能。
   これは P₃ の「H_s は静的バランスではなく動的調整対象」と同型。

## K₆ × K₃ の相互作用

> K₃ (Flow 内) と K₆ (修飾座標間) は独立に定義されるが、相互に変調する。

**動詞の位置づけ**: 36 Poiesis = K₃ (3頂点) × K₆ (6極方向) × 2 (各座標の両極)

例: `/noe` (理解する) = I×E (推論 × 認識目的)
- K₃ での位置: I 頂点に属する
- K₆ での位置: Value 座標の E 極に結合
- φ_SI が先行する (/noe を実行するとき、まず S から入力を得てから I で理解)
- φ_IA が後続する (/noe の結果を A で出力する)

例: `/the` (観照する) = S×E (知覚 × 認識目的)
- K₃ での位置: **S 頂点** に属する (v5.0 新規)
- K₆ での位置: Value 座標の E 極に結合
- φ_SI は /the 自体の定義 (認識目的で知覚する = S→I のマクロ遷移)
- /noe との関係: /the (S×E) → /noe (I×E) は φ_SI の Value=E スライス

> **CCL パイプラインの Flow 遷移**: `/the >> /noe >> /zet` = S→I→A (φ_SI → φ_IA) は
> Flow の自然な流れに沿ったパイプライン。K₃ の3辺を1周する循環。
