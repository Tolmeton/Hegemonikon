# 極限テスト原則 (Limit Test Principle)

> **誤った論理を論破するには、極限（極論）を考えればよい。**
> **言い換えれば、エッジケースのテスト。**
> — Creator, 2026-02-14

---

## 1. 定義

命題 P(x) が「全ての x に対して成り立つ」と主張するとき、
x の定義域の**境界 (極限)** で P を評価せよ。

```
P(x) が真 ∀x ∈ D  ←→  P(x) が真 ∀x ∈ ∂D (境界) を含む
```

極限で崩壊する命題は、**暗黙の前提条件**を隠蔽している。

---

## 2. 起源: Trafimow 反論の崩壊

| ステップ | 内容 |
|:---------|:-----|
| **命題** | 「補助仮定を追加すれば予測力は回復する」(Trafimow 2015) |
| **極限** | α → 1 (メタ原理の極限) を適用 |
| **崩壊** | I(T) → 0 なので「回復」ではなく「注入」。理論の寄与ゼロ |
| **発見** | 「追加」は「選択」であり、選択はトレードオフそのもの |

Trafimow は α < 1 という**暗黙の前提**を書いていなかった。
極限テストがその前提を炙り出した。

---

## 3. 形式化

### エッジケーステストとの同型

```python
def limit_test(proposition, domain):
    """命題の健全性を極限で検証する"""
    boundary = domain.boundary()  # 定義域の境界
    for edge_case in boundary:
        result = proposition(edge_case)
        if not result:
            return f"FAIL at {edge_case}: 暗黙の前提を発見"
    return "PASS: 全境界で成立"
```

### 数学的対応

| 概念 | 数学 | ソフトウェア | 哲学 |
|:-----|:-----|:------------|:-----|
| 極限テスト | 極限 lim_{x→∂D} P(x) | edge case testing | 背理法の変種 |
| 暗黙の前提 | 定義域の制限 D → D' | 未テストの入力範囲 | 隠された前提 (hidden premise) |
| 崩壊点 | 不連続点 / 特異点 | バグ | 論理の穴 |

## 4. 適用パターン

### パターン1: 変数を極端にする

> 「それが100%のとき成り立つか？ 0%のとき成り立つか？」

- fep_epistemic_status.md: α → 1 (完全な抽象) で Trafimow が崩壊
- テスト: N → ∞ でアルゴリズムが破綻するか

### パターン2: 対象を最小にする

> 「1人でも成り立つか？ 1個でも成り立つか？」

- 「チームワークで解決」→ 一人チームでは？
- 「規模の経済」→ 規模が1のとき？

### パターン3: 主語を入れ替える

> 「自分に適用しても成り立つか？」

- 「全てを説明する理論」→ 自分自身の存在を説明できるか？ (自己言及)
- Mangalam (2025) の FEP 風刺論文がまさにこの手法

## 5. 忘却論との接続 (2026-03-30)

### 極限テスト = ker(G) の検出

忘却関手 G の文脈では、極限テストは**何が忘却されるか (ker(G))** と**何が保存されるか (image(G))** の境界を同定する操作:

| 極限テストの概念 | 忘却論の対応物 |
|:---|:---|
| 極限で崩壊する命題の成分 | ker(G) に属する情報 — α 増大時に最初に消える |
| 極限でも残る命題の成分 | image(G) の方向 — 忘却に耐える構造 |
| 暗黙の前提の発見 | 暗黙に仮定されていた ker(G) の内容の顕在化 |
| 崩壊点 | α の臨界点 τ_c ≈ 0.75 (E11 実験) |

### 実験的裏付け

- **E11**: τ を 0.50→0.95 で連続掃引 (46点) — パターン1「変数を極端にする」の直接実行。τ_c≈0.75 で相転移 = 崩壊点の同定
- **E12**: ker(G) が等方的 (PR=106.4) — ker(G) は特定の方向を持たない。極限テストが発見する「暗黙の前提」は方向ではなく**次元の低ランク性**
- **E13**: dim(image(G)) ~ 1/α — α を極端にする (パターン1) と image が収縮。極限テストの定量版

### 忘却論における3パターンの再解釈

| パターン | 忘却論的意味 |
|:---|:---|
| 変数を極端にする | α → 0 (全保持) or α → 1 (全忘却) で場の方程式を評価 |
| 対象を最小にする | 最小の圏 (対象1つ、射1つ) で忘却関手が自明になるか |
| 主語を入れ替える | G を G 自身に適用する (Paper VII §7.4 N_self 再帰) |

Paper VII の N_self 収束定理 (cⁿ/n! 階乗的減衰) は**パターン3の究極形**: 忘却関手を自分自身に適用したとき、n≈5 で実質的に収束する。

## 6. 制限

> [!WARNING]
> 極限で崩壊する ≠ 命題が無価値。
> **適用範囲の発見**が目的であり、棄却が目的ではない。

Trafimow の命題は α ∈ (0.3, 0.7) では有効。
極限テストは「どこまで有効か」の**境界線**を引く道具。

忘却論の文脈: ker(G) に属する情報は「無価値」ではなく、**その α での操作に不要**。α が変われば ker(G) も変わる。
