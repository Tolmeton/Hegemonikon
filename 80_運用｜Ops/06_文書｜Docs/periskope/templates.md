# テンプレート集 — Periskopē

> **利用規約**: WF Phase 内で使用する。CCL マクロからは直接呼ばない。

## Periskopē MCP パラメータテンプレート

### `periskope_research` 呼び出し

```yaml
# L1 Quick
query: "{質問}"
depth: 1
sources: ["searxng"]
expand_query: false
multipass: false
max_results: 10

# L2 Standard — デフォルト
query: "{質問}"
depth: 2
expand_query: true
max_results: 10

# L3 Deep
query: "{質問}"
depth: 3
expand_query: true
multipass: true
max_results: 10

# ソース限定 (内部のみ)
query: "{質問}"
sources: ["gnosis", "sophia", "kairos"]

# 自動消化
query: "{質問}"
auto_digest: true
```

## 外部調査依頼書テンプレート

> Periskopē で不十分な場合に外部ツール (Perplexity 等) に送る形式。

### 外部経路

| 経路 | エンジン | 最適な場面 |
|:-----|:---------|:----------|
| **A** | Perplexity Pro | 最新情報、ファクトチェック、速報 |
| **B** | Gemini Deep Researcher | 学術調査、体系的レビュー |
| **C** | Claude Research (Max/Pro) | 個人コンテキスト型調査 |
| **D** | 複合 (A/B/C) | 重要テーマ、二重検証 |

### 依頼書フォーマット

```markdown
# 調査依頼: [テーマ]

> **決定事項**: [この調査結果で何を決定するか]
> **仮説**: [〇〇ではないか？/ オープンクエスチョン]

---

## 出力形式 (必読)

**必須**: 構造化テーブル（4列: 項目, 値, 根拠, URL）を返せ。
その後、2-3文の結論を formal トーンで記述。

---

## 時間制約 (情報鮮度)

- **採用範囲**: 過去6ヶ月以内の情報を優先
- **歴史的参考**: 1年以上前は「参考」としてのみ言及可
- **日付明記**: 各ソースに「取得時点: YYYY年MM月」を付記

---

## 調査すべき論点 (必須網羅)

### A. [主要論点1のカテゴリ]
- A1: [具体的な質問]
- A2: [具体的な質問]
- A3: [具体的な質問]

### B. [主要論点2のカテゴリ]
- B1: [具体的な質問]
- B2: [具体的な質問]

---

## 反証・検証

- 主要な発見と矛盾するソースを3件探すこと
- 発見した情報が古い場合、現在の状態を検証する方法を述べよ

---

## 成果物構成

1. **結論サマリー**（10行以内）
2. **比較テーブル**（必須列: 項目, 値, 根拠, URL）
3. **根拠リンク**（各セクション最低3件）
```

## 論点テンプレート集

### 技術比較を問う

```
### A. 技術比較
- A1: [技術X] と [技術Y] のベンチマーク比較（2024年以降）
- A2: 実運用環境での性能差（レイテンシ、スループット）
- A3: コスト比較（初期/運用）
- A4: 採用企業の事例と評価
- A5: 既知の制約・デメリット
```

### 実装選択を問う

```
### A. 実装選択
- A1: 公式推奨のアプローチは？
- A2: Stack Overflow/GitHub での頻出解決策は？
- A3: 既知のアンチパターンは？
- A4: パフォーマンス最適化のベストプラクティスは？
- A5: 将来の非推奨（deprecation）リスクは？
```

### 先人の知恵を調べるとき

```markdown
## 調査依頼: 既存解決策の探索

> **決定事項**: 自前実装するか既存ソリューションを使うか
> **仮説**: 既に解決済みの問題のはず

### 問題
[解決したい問題を1-2文で]

### 調査対象
- Stack Overflow: 類似の質問と回答
- GitHub Issues: 関連プロジェクトの既知の問題と解決策
- 技術ブログ: 実践的な解決事例
- 公式ドキュメント: 推奨されるアプローチ

### 出力形式
| ソース | 解決策 | 適用可能性 | URL |
|:-------|:-------|:-----------|:----|

### 判定
- 「そのまま使える」「参考になる」「該当なし」の3分類で
```
