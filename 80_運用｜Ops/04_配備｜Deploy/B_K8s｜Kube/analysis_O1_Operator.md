# OpenAI Operator — 行動仕様分析 (O1)

## 1. Executive Summary

OpenAI Operator (CUA: Computer-Using Agent) は、クラウド上の**分離された仮想ブラウザ (Secure Remote Browser)** で動作する、消費者向けの高度なWebタスク自動化エージェントである。GPT-4o のマルチモーダル能力を活用し、エラー発生時のユーザー委任や、パスワード管理などのセキュリティプロトコルを自然言語の対話 (Chat UI) とシームレスに統合している。

---

## 2. 操作プリミティブ表

| カテゴリ | 操作 | ターゲット | 特徴 |
| :--- | :--- | :--- | :--- |
| Navigation | `navigate` | URL | クラウドブラウザのアドレスバー操作 |
| Interaction | `click` | UI 要素 | Web 要素の中心点を特定してクリック |
| Input | `fill_input` | テキストボックス | ブラウザを通じてフォームに値を注入 |
| Wait | `wait_for_load` | ページ | ネットワークアイドルや特定の要素表示まで待機 |
| User Protocol | `ask_user` | チャットUI | ログイン認証コード入力や確認事項をユーザーに要求 |

---

## 3. OODA サイクル分析

* **Observe（観察）:**
  * **メカニズム:** 仮想ブラウザのレンダリング結果（スクリーンショット）と DOM ツリーから生成されたアクセシビリティツリーのハイブリッド。
* **Orient（情勢判断）:**
  * **メカニズム:** 「航空券を予約する」などの曖昧な指示を、現在の画面状態にマッピング。タスクの進捗状況をプロンプト内で継続追跡。
* **Decide（意思決定）:**
  * **メカニズム:** RL (強化学習) でファインチューンされた CUA モデルが、ブラウザ特化のアクションを選択。
* **Act（行動）:**
  * **メカニズム:** セキュアな仮想ブラウザ環境内で即座に Playwright 等のドライバ経由でアクションを実行。

## 4. エラーリカバリ表

| 失敗モード | 検知方法 | 回復戦略 |
| :--- | :--- | :--- |
| サイトアクセス拒否 (CAPTCHA/Block) | Observe (403画面やCAPTCHAパズルを視覚・DOMで検知) | `ask_user` で「このサイトはブロックされています。手動で操作するか、別サイトを探しますか？」と委任 |
| 複雑なUI/無限スクロール | Decide (一定回数のスクロールでも目的要素が見つからない) | 検索クエリを変更するか、ユーザーに「見つかりません。条件を変えますか？」と質問 |
| ログイン要求 (MFA) | Observe (パスコード入力画面の表示) | 処理を中断し、ユーザーのチャットUIに MFA コードの入力を促すプロンプトを表示 |

## 5. 設計判断マトリクス

| 設計軸 | トレードオフ / 課題 | OpenAI Operator のアプローチ |
| :--- | :--- | :--- |
| **セキュリティ** | ユーザーのローカル環境を操作するのは危険。 | **クラウド上の仮想ブラウザ (Sandboxed Environment)**。ユーザーのPCには一切触れず、クラウドで安全に隔離実行する。 |
| **ユーザーインタラクション** | どこまで自律させるか。 | **Consumer-Friendly (HITL重視)**。ログイン、決済、不確実な判断は積極的に Chat UI に返し、人間の確認を求める。 |
| **速度 vs 正確性** | LLM の推論は遅い。 | ChatGPT の UI の横に**ライブビュー (Live View)** を配置。エージェントが操作する画面をそのままストリーミングし、遅さを「作業を代行してくれている感」に転換。 |
