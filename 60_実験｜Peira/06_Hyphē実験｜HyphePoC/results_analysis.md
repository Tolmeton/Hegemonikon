## 6. Coherence τ 非依存性の理論的説明

> **確信度**: [推定] 75%。数値的証拠あり。解析的証明は未完。
> **由来**: 2026-03-13 /kop→/dok 実験結果分析中に発見

### 現象

τ を 0.60→0.80 に変化させると:
- chunks 数は 1.0→20.1 (20倍変化)
- chunk サイズは 67.0→3.4 (20倍圧縮)
- **coherence は 0.807→0.815** (1% 以下の変化)

### 素朴仮説の棄却

**素朴仮説**: coherence = E[ρ|ρ≥τ] (チャンク内は τ 以上の ρ のみ)

**棄却**: E[ρ|ρ≥τ] は τ に **依存する**:

| τ | E[ρ|ρ≥τ] | N(ρ≥τ) | 実測 coherence |
|:--|:----------|:--------|:---------------|
| 0.60 | 0.8078 | 858/858 (100%) | 0.807 |
| 0.70 | 0.8139 | 820/858 (95.6%) | 0.815 |
| 0.75 | 0.8297 | 691/858 (80.5%) | 0.814 |
| 0.80 | 0.8553 | 476/858 (55.5%) | 0.815 |

E[ρ|ρ≥τ] は 0.808→0.855 と変化するが、実測 coherence は変化しない。

### 原因: G∘F が coherence を保存する

**coherence 計算方法** (hyphe_chunker.py L10663):

```python
chunk.coherence = sum(chunk_sims) / len(chunk_sims)
```

ここで `chunk_sims` = チャンク内の全隣接ステップの ρ 値。

**τ が高いと小チャンクが生成され、各チャンクは高い ρ のペアのみを含む** — これは E[ρ|ρ≥τ] を上げるはず。しかし **G∘F が介入する**:

1. **F (merge)**: 小さすぎるチャンクを隣接にマージ → **低 ρ の境界ペアを取り込む**
2. **G (split)**: 低 coherence チャンクを再分割 → **高 ρ 領域に分離**
3. **Fix(G∘F)**: merge と split が均衡 → coherence が τ に依存しない不動点に収束

### 仮説: Coherence は G∘F の保存量

```
定義: C̄ = (1/|chunks|) · Σ_c coherence(c)

主張: Fix(G∘F) は C̄ を不変量として保存する。
      つまり C̄(Fix(G∘F; τ₁)) ≈ C̄(Fix(G∘F; τ₂))  ∀τ₁, τ₂

直感: G∘F は境界位置を最適化するが、全体の意味的密度は保存する。
      チャンクを細かくすれば各チャンクの coherence は上がるが、
      merge がそれを打ち消し、全体平均が一定に保たれる。
```

### linkage_hyphe.md §3.5 との接続

§3.5 の指数モデル:

```
λ(ρ) = a + b · exp(-β · ρ)
```

Fix(G∘F) では λ=1 (臨界点) ではなく λ<1 (収縮写像) の領域で動作する。
**仮説**: 収縮写像 G∘F が coherence C̄ を保存する理由は:

- F (発散) は Complexity を増大させるが coherence を上げる
- G (収束) は Complexity を減少させるが coherence を下げる
- F と G の coherence への効果が **正確に打ち消し合う**

これは VFE = -Accuracy + Complexity の構造そのもの:
- Accuracy ∝ coherence (§3.6 L(c) の Drift 項)
- G∘F は VFE を最小化 → Accuracy を最大化しつつ Complexity を最小化
- 不動点では Accuracy(≈coherence) の値が VFE ランドスケープで決まる
- この値は **τ ではなく similarity 分布の構造** で決定される

### 結論

| 項目 | 結論 |
|:-----|:-----|
| 現象 | coherence が τ に依存しない (0.807-0.815) |
| 素朴仮説 | E[ρ\|ρ≥τ] — **棄却** (τ 依存性あり) |
| 真の原因 | G∘F の merge/split が coherence を均衡させる |
| 理論的意味 | **coherence は G∘F の保存量** — 位相的不変量の候補 |
| FEP 接続 | VFE 最小化の不動点で Accuracy (≈coherence) が決定される |
| 検証 | → §6.1 で **実証済み** ✅ |

1. **τ=0.60**: 両条件で同一 (1.0 chunks, coherence=0.807) — τ が低いと全データが1チャンクになり G∘F の介入余地がない

**G∘F の操作的意味**: 高τ で生じる小チャンク (size=2.3) は人工的に高い coherence を持つ。G∘F の merge が境界ペアを取り込み、coherence を「自然な」値 ≈ 0.815 に正規化する。

#### 理論的帰結

```
定理候補 (Coherence Invariance):
  C̄(Fix(G∘F; τ)) ≈ const  for all τ ∈ (τ_min, τ_max)

  where:
    C̄ = mean coherence
    τ_min = 下限 (全データが1チャンク)
    τ_max = 上限 (各ステップが1チャンク)
    const ≈ mean(similarity_distribution) ≈ 0.808

  推定確信度: [確信] 90%
  証拠: 104実験で再現。G∘F OFF で棄却 (range=0.121)
  制限: 1 embedding モデル (gemini-embedding-001, 768d), 13セッションのみ
```

### 7.5. 分析

#### Drift 改善のメカニズム

knn 類似度 (k=5) は各ステップの k 近傍との平均類似度を計算する。pairwise (隣接ペアのみ) と比較して:

1. **局所的な意味構造を広く捉える**: 隣接1ペアではなく k 近傍の情報を使うため、孤立した高/低類似度ペアに影響されにくい
2. **drift 検出が早い**: 意味的に離れたステップが混入した瞬間に knn 類似度が下がり、境界を正確に捉える
3. **分割数が多い**: knn は pairwise より多くの境界を検出する (平均 1.3〜1.5倍)

#### Coherence のトレードオフ

knn は coherence が ~3% 低い (0.78 vs 0.81)。原因:

- knn が生成する多数のチャンクは、境界付近のステップを別チャンクに分離する
- 分離されたチャンクの内部 coherence は高い vs 低い のバラつきが大きくなる
- 平均としては pairwise より低くなる

**評価**: coherence の 3% 低下は drift の 10-16% 改善で十分にペイする。

#### G∘F 収束

全24実験で G∘F 収束。iteration 数は knn が pairwise より若干多い (2.0 vs 1.3)。
knn が生成する細かいチャンクに対して merge (F) が介入する回数が増えるが、2回以内に収束。

> **確信度**: [推定] 75%

### 9.1. 実験設計

```
独立変数: precision (チャンク内隣接ステップの cosine similarity の平均)
```

## 10. Sloppy Spectrum 解析 (E3)

> **実行**: 2026-03-14 — 4データセット (session, handoff, papers, all)
> **ファイル**: `e1_sloppy_spectrum_real.json`, `e1_full_spectrum_analysis.json`

### 10.1. 固有値スペクトル

| データセット | n_points | dim | k_signal | sloppy_ratio | Fisher rank |

### 10.2. 拡張解析 (all×2000, 40次元)

| 指標 | 値 |
|:-----|:---|

| k_eff_99 | 40 |

| 第1固有値 | 0.00663 |

### 10.3. 解釈

1. **k_signal = 2-3**: embedding 空間の「意味的主成分」は極めて低次元 (3072次元中 2-3)
2. **k_eff_90 ≈ 33**: 分散の90%を説明するには 33 次元必要。残り 3039 次元はノイズ

- **τ* の物理的意味**: 相転移点 τ*≈0.720 は、k_signal = 2-3 の低次元意味空間上での「分離閾値」
- **precision の低変動の根本原因**: k_eff_90 ≈ 33 → cosine similarity の変動が圧縮される (§12 で詳述)
- **座標完全性 (axiom_hierarchy §定理⁴) との接続**: K₆ の6座標 >> k_signal の 2-3。embedding は座標の一部しか捉えていない

## 11. EFE 分解実験 (E4)

> **確信度**: [推定] 65%
> **実行**: 2026-03-14 — 13セッション (v0.7 拡張版)
> **ファイル**: `precision_v07_results.json`
> **目的**: L(c) を epistemic + pragmatic に分解し、チャンク品質の多面的評価を検証

### 11.1. EFE 成分統計 (全13セッション)

| 成分 | 平均 | 範囲 | 備考 |
|:-----|:----|:-----|:-----|
| epistemic | 0.013 | 0.0-0.036 | 予測安定性 (低い = 良い) |
| pragmatic | 0.158 | 0.0-0.375 | 行動可能性 (境界チャンクで高い) |
| efe (合計) | 0.089 | 0.0-0.190 | epistemic + pragmatic |
| loss | -0.005 | -0.086 〜 +0.059 | drift - efe |
| boundary_novelty | 0.145 | 0.0-0.375 | 境界での意味的新規性 |

### 11.2. 構造的パターン

1. **最終チャンクの efe → 0**: セッション末尾のチャンクでは pragmatic ≈ 0 (次の行動がない)。これは理論通り
2. **最終チャンクの loss > 0**: efe が 0 に近いため loss = drift のみ。締めくくりの drift だけ残る
3. **中間チャンクの loss < 0**: efe が drift を上回り、情報獲得が品質を担保。期待通りの EFE 構造
4. **boundary_novelty ≈ pragmatic**: 両者がほぼ同値。セマンティックな「新規性」は「行動可能性」と同義

### 11.3. Precision の問題 (v0.7)

**H1 仮説**: precision = チャンク内の embedding gradient (隣接ステップ間の cosine similarity の min-max 正規化)

**結果**: precision が 0.0 または 1.0 にしかならない (位置依存的)。

```
原因: min-max 正規化の分母 (global_max - global_min) が全チャンクで同一
      → 各チャンクの precision はチャンク内位置 (first/last) でのみ変動
      → 情報量ゼロ
```

**H1 は棄却**。§12 で代替アプローチを議論。

## 12. Precision Gradient 問題と診断 (E5)

> **確信度**: [仮説] — 進行中
> **関連セッション**: Hyphē Precision Debugging, Diagnosing Embedding Anisotropy, Precision-Aware Lambda Experiment

### 12.1. 問題の定義

Hyphē の理論フレームワーク (linkage_hyphe.md §3.5) では、precision π はチャンク品質の重要な制御パラメータ:

```
λ(ρ) = a + b · exp(-β · ρ)     — §3.5 指数モデル
```

ρ (similarity) に基づく precision gradient が期待されるが、実験では:

| 計算方法 | precision range | 判定 |
|:---------|:---------------|:-----|
| cosine min-max (v0.5) | ≈ 0 | ❌ 変動なし |
| cosine min-max (v0.7) | 0 or 1 | ❌ 位置依存のみ |
| PA-Lambda (cosine avg) | 0.66 ± 0.01 | ❌ 変動極小 |

### 12.2. 根本原因: Embedding Anisotropy

並列セッション「Diagnosing Embedding Anisotropy」の診断結果:

```
モデル: gemini-embedding-001 (768d) / bge-m3 (1024d)
有効次元 k_eff_90: 32-36 (3072次元中)
cosine similarity baseline: 0.80±0.065
→ cosine 類似度がアンカリングされており、gradient を検出できない
```

**表現退化 (representation degeneration)**: 高次元 embedding 空間の大部分が使用されておらず、全ベクトルが狭い円錐内に集中。結果として cosine similarity の分散が圧縮される。

### 12.3. 進行中の対策

| 対策 | セッション | 状態 |
|:-----|:---------|:-----|
| **Whitening** (等方化) | run_whitening_experiment.py | 設計済み |

| **Gemini Embedding 2** | Gemini Embedding Migration session | API 調査済み |
| **代替指標** (KL-div, Fisher) | 未実施 | 候補 |

### 12.4. 暫定結論

| 項目 | 結論 |
|:-----|:-----|
| precision gradient は存在するか | **判定保留** — embedding の問題であり理論の問題ではない |

3. **gf_off の一貫性**: 両モデルとも τ↑ → coherence↑ の単調増加。G∘F 非活性時の τ 依存性はモデル非依存の構造的パターン
4. **G∘F の普遍性**: 異なる embedding アーキテクチャ・次元数でも G∘F は coherence を τ 非依存の不動点に収束させる
