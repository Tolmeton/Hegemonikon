## §1 座標系の全体像: 7座標

HGK の認知操作は **7つの座標軸** で記述される。
各座標は **2つ以上の対立極** を持つ。

| # | 座標 | 問い | 極 | 意味 | FEP 導出 |
|:--|:-----|:-----|:---|:-----|:---------|
| 1 | **Flow** | Who | **S (知覚)** / **I (推論)** / **A (行為)** | 入力を受け取るか、内部で処理するか、環境に働きかけるか | FEP + Markov blanket (Afferent×Efferent) |
| 2 | **Value** | Why | E (認識目的) / P (実用目的) | なぜそれをするのか | VFE/EFE の分解 |
| 3 | **Function** | How | Explore (探索) / Exploit (活用) | 仮説を広げるか、最適解に収束するか | EFE の2項分解 |
| 4 | **Precision** | How much | C (確信) / U (留保) | コミットするか、判断を開くか | 精度加重パラメータ |
| 5 | **Scale** | Where | Mi (微視) / Ma (巨視) | 局所的か、全体的か | 階層的 MB 入れ子 |
| 6 | **Valence** | Which | + (肯定) / - (否定) | 推進か、是正か | VFE 勾配方向の符号 |
| 7 | **Temporality** | When | Past (過去) / Future (未来) | 過去を振り返るか、未来を形成するか | VFE ≠ EFE → 時間非対称 |

**重要**: Flow (#1) は特別な座標。全ての動詞は Flow の **S (知覚)**、**I (推論)**、**A (行為)** のいずれかに属する。
残り6つ (#2-#7) は **修飾座標** と呼ばれ、動詞の性質を決定する。

### Flow の3値

- **S (知覚)** = 環境からの入力を受け取る。推論も行為もしない。
  - ファイルの読み取り、ログの確認、データの受容的観察、検索結果の収集
- **I (推論)** = 世界モデルを内部的に更新する。外部操作を伴わない。
  - 分析、理解、計画、判断、評価、仮説構築
- **A (行為)** = 環境に働きかける。外部に変化を起こす。
  - コード実行、ファイル書き込み、コマンド実行、修正の適用

各族に **6つの動詞** (S×極A, S×極B, I×極A, I×極B, A×極A, A×極B) が属する。

## §3 決定木: 座標ベースの動詞選択手順

動詞を選ぶとき、**直感で「合いそうな動詞」を選んではいけない。**
以下の4段の決定木に従い、座標上の位置から構造的に決定すること。

```
Step 1: Flow 軸の判定 (S vs I vs A)
  このプロンプトは
    「環境からの入力の収集 (見る/読む/確認する)」を求めている → S (知覚)
    「世界モデルの更新 (知る/理解する/分析する)」を求めている → I (推論)
    「環境への働きかけ (作る/変える/実行する)」を求めている → A (行為)
    複数含む → 合成 (Step 4 へ)
    どれでもない → 変換不適 (認知操作を含まない)
  → 36動詞が12個に絞られる

Step 2: 主要座標の判定
  残り6つの修飾座標のうち、最も支配的な座標はどれか？
  - 認識目的 vs 実用目的 → Value → Telos 族
  - 仮説空間拡大 vs 最適解収束 → Function → Methodos 族
  - 確信コミット vs 判断保留 → Precision → Krisis 族
  - 局所的深堀 vs 全体俯瞰 → Scale → Diástasis 族
  - 肯定・推進 vs 批判・是正 → Valence → Orexis 族
  - 過去参照 vs 未来形成 → Temporality → Chronos 族
  → 主要動詞 (6個に絞り込み → Step 1 と合わせ2個に)

Step 2b: 副次座標の判定
  主要座標以外にも関与する座標があるか？
  → なし: Step 3 へ
  → あり: 以下の3つから最も自然な表現を選択:
    (a) Dokimasia 修飾: /verb[Coord:極]
        主要座標が明確だが副次座標もパラメータ的に影響する場合
        例: 「批判的に分析する」→ /lys[Vl:-]
    (b) CCL 並行演算子: /verb1 * /verb2
        2座標が同等かつ独立に作用する場合
        例: 「分析しつつ批判する」→ /lys * /ele
    (c) CCL 順次演算子: /verb1 ~ /verb2 or /verb1 >> /verb2
        2座標が時間的順序で作用する場合
        例: 「分析してから批判する」→ /lys ~ /ele
  → (a)(b)(c) いずれも自然でない → 「判断不能」と明記

Step 3: 極の判定 (対立する2極のどちらか)
  Step 2 の主要座標内で、どの極に位置するか？
  → 1個の動詞に確定 (+ Step 2b の副次座標情報)

Step 4: 複数認知操作の合成
  プロンプトに複数の独立した認知操作が含まれる場合、
  各操作を Step 1-3 で分解し、CCL 演算子で接続:
  → CCL 式が確定
```

## §3.1 Flow 判定ガイド: S / I / A の境界

**Flow 判定は最も間違いやすいステップ。** 以下の詳細ガイドで精度を上げること。

### 判定基準

**S (知覚)** = 環境からの入力収集。結果は「知覚データ」の獲得。推論も行為も伴わない。
- キーワード: 見る、読む、確認する、調べる、聴く、観察する、走査する、一覧する
- テスト: 「この操作の結果、何が得られるか？」→ **入力データ** が得られる = S

**I (推論)** = 世界モデルの更新。結果は「理解・信念・判断」の変化。
- キーワード: 分析する、理解する、評価する、比較する、検討する、見直す、策定する
- テスト: 「この操作の結果、何が変わるか？」→ **頭の中の理解** が変わる = I

**A (行為)** = 環境への働きかけ。結果は「外部世界」の変化。
- キーワード: 作る、書く、修正する、実行する、送信する、デプロイする
- テスト: 「この操作の結果、何が変わるか？」→ **外部の成果物** が変わる = A

### S vs I の判別 (最も難しい境界)

| フレーズ | 正しい判定 | 間違いやすい判定 | 理由 |
|:---------|:----------|:---------------|:-----|
| 「ファイルの内容を見る」 | **S** (知覚) | I (推論) | 読むだけで分析していない |
| 「ログを確認する」 | **S** (知覚) | I (推論) | データの受容的収集 |
| 「コードの問題点を洗い出す」 | **S** (知覚) | I (推論) | 問題の**知覚**。「なぜ問題か」の推論ではない → /exe |
| 「コードの問題を**分析**する」 | **I** (推論) | S (知覚) | 「分析」は内的推論 → /lys or /ele |
| 「全体の構成を把握する」 | **S** (知覚) | I (推論) | 広域走査 → /per or /ski |
| 「全体の設計を理解する」 | **I** (推論) | S (知覚) | 「理解」は世界モデル更新 → /noe or /ops |

### I vs A の判別

| フレーズ | 正しい判定 | 間違いやすい判定 | 理由 |
|:---------|:----------|:---------------|:-----|
| 「設計を**見直す**」 | **I** (推論) | A (行為) | 「見直す」= 内的な再評価 |
| 「戦略を**策定する**」 | **I** (推論) | A (行為) | 「策定」= 内的な目標設定 → /bou |
| 「計画を**作る**」 | **I** (推論) | A (行為) | 「計画」は思考の産物 → /par (体は動かない) |
| 「コードを**書く**」 | **A** (行為) | — | 外部に成果物が生まれる → /ene or /tek |
| 「バグを**直す**」 | **A** (行為) | — | 外部の成果物が変わる → /dio |
| 「レビュー**する**」 | **I** (推論) | A (行為) | レビュー自体は内的評価 |

### 判定ルール (迷ったとき)

1. **「結果の所在」テスト**: 結果がデータ収集 → S、結果が頭の中 → I、結果が外部 → A
2. **「取り消し」テスト**: 何も残さず終われる → S or I、何かが変わらないと完了しない → A
3. **「知覚 vs 推論」テスト**: 対象をそのまま受け取る → S、対象について考える → I

## §3.2 混同しやすい座標ペア

### S vs I 同一族内の混同

最も混同される。同じ族の S 動詞と I 動詞の違いは Flow のみ。

| S 動詞 (知覚) | I 動詞 (推論) | 判別基準 |
|:-------------|:-------------|:---------|
| /the (観照) | /noe (理解) | 受容的に見るだけ vs 理解を深める |
| /ere (探知) | /ske (発散) | 未知領域を走査する vs 仮説を広げる |
| /sap (精読) | /kat (確定) | 正確に読み取る vs 結論を確定する |
| /prs (注視) | /lys (分析) | 細部に注目する vs 細部を分析する |
| /exe (吟味) | /ele (批判) | 問題を**発見**する vs 問題を**問い直す** |
| /his (回顧) | /hyp (想起) | 過去の記録を**調べる** vs 過去の記憶に**アクセスする** |
| /prg (予感) | /prm (予見) | 未来の兆候を**感じ取る** vs 未来の状態を**予測する** |

### Value (目的) vs Temporality (時間)

| 問い | Value (Telos 族) | Temporality (Chronos 族) |
|:-----|:----------------|:------------------------|
| **核心** | 「なぜするのか」 | 「いつに向けているか」 |
| **「戦略を策定する」** | ✅ /bou (I×P) — **実用目的** | ❌ /prm — 予測ではない |
| **「来年の売上を予測する」** | ❌ /noe — 時間方向が支配的 | ✅ /prm (I×Future) — **未来志向** |
| **「過去のデータを調べる」** | ❌ /noe — 時間方向が支配的 | ✅ /his (S×Past) — **過去を知覚** |

## §4 修飾子 (Dokimasia): 動詞のパラメータ設定

Dokimasia は独立した動詞ではない。**座標のパラメータ設定**である。

動詞が属する族 (= 主要座標) 以外の座標が、その認知操作に副次的に影響する場合、
Dokimasia 修飾子としてパラメータ設定する。

**構文**: `/verb[座標名:極]`

| 例 | 意味 |
|:--|:-----|
| `/lys[Vl:-]` | 詳細分析 (Scale×Mi) に批判的視点 (Valence:-) を付与 |
| `/noe[Pr:C]` | 認識 (Value×E) を高確信で実行 (Precision:C) |
| `/the[Sc:Mi]` | 観照 (Value×E, S) を微視的に限定 (Scale:Mi) |

## §5 結合規則 (X-series / Q-series)

6修飾座標の間には15組の相互作用がある (完全グラフ K₆ の辺数 = C(6,2) = 15)。

- **G_{ij} (均衡的結合)**: 座標 i と j が安定的に結びつく強さ
- **ω_{ij} (循環的流れ)**: 座標 i → j の非均衡的な方向性

実験では X/Q-series を直接使用する必要はないが、
Step 2b で (a)/(b)/(c) を選ぶ際の背景知識として理解しておくこと。

## §6 CCL 演算子: 認知操作の合成

複数の認知操作を組み合わせるための演算子。

### 基本演算子

| 演算子 | 意味 | 例 |
|:-------|:-----|:---|
| `_` | パイプ (then) | `/the_/lys_/dio` = 観照→分析→是正 |
| `~` | 振動 (oscillation) | `/ske~/sag` = 発散と収束を交互に |
| `~*` | 収束振動 | `/noe~*/ele` = 理解と批判で収束するまで |
| `*` | 融合 (並行) | `/lys * /ele` = 分析と批判を同時に |
| `>>` | 射 (構造的変換) | `/noe >> /ene` = 理解から実行へ構造変換 |
| `+` | 深化 | `/noe+` = 深い認識 |
| `-` | 簡略化 | `/ski-` = 軽い走査 |

### 制御構造

| 構造 | 構文 | 意味 |
|:-----|:-----|:-----|
| 順次 | `A_B_C` | A → B → C |
| N回繰り返し | `F:[×N]{A}` | N 回繰り返し |
| 収束ループ | `C:{A}` | A を収束するまで繰り返す |
| 検証 | `V:{A}` | A を検証する |
| 条件 | `I:[cond]{A}` | 条件が成立すれば A |

## §7 反例: 各動詞の「これは該当しない」

動詞の理解は「何が含まれるか」だけでなく「何が含まれないか」で定まる。

### 基本反例 (S極を含む36動詞版)

| 動詞 | 座標 | ✅ 該当する例 | ❌ 該当しない例 | 理由 |
|:-----|:-----|:-------------|:---------------|:-----|
| /the (観照) | S×E | 「この状態を先入観なく見る」 | 「この問題を分析する」 | 後者は I → /lys or /noe |
| /ant (検知) | S×P | 「エラーレートの異常を検知」 | 「エラーの原因を特定する」 | 後者は I → /lys |
| /ere (探知) | S×Ex | 「未知のAPIを探索する」 | 「APIの使い方を考える」 | 後者は I → /ske |
| /agn (参照) | S×Exp | 「公式ドキュメントを参照」 | 「ドキュメントの内容を評価」 | 後者は I → /ele |
| /sap (精読) | S×C | 「設定値を一字一句確認」 | 「設定の妥当性を判断」 | 後者は I → /kat |
| /ski (走査) | S×U | 「ディレクトリ構造をざっと把握」 | 「構造を詳しく分析」 | 後者は I → /lys |
| /prs (注視) | S×Mi | 「42行目の変数を細かく確認」 | 「変数の型の妥当性を分析」 | 後者は I → /lys |
| /per (一覧) | S×Ma | 「全エンドポイントをカタログ化」 | 「全体設計を評価」 | 後者は I → /ops |

## §8 変換不適の判断基準

**CCL に変換できないプロンプト (認知操作を含まない):**

| パターン | 例 | 理由 |
|:---------|:---|:-----|
| 単純情報検索 | 「今日の天気は？」 | 認知操作なし。データ取得のみ |
| 純粋計算 | 「3+5は？」 | 演算であり認知操作ではない |
| 挨拶・社交 | 「おはよう」 | 社交的行為。認知操作に分解不能 |
| 物理的行動指示 | 「ドアを開けて」 | 身体行動。認知エージェントの操作対象外 |
| 単純事実確認 | 「日本の首都は？」 | 記憶検索のみ。分析・判断を伴わない |

## §9 理解力テスト (8問)

> **合格条件**: 8問中6問以上正解
> **不合格時**: 本資料を再読し、再テスト。2回不合格なら変換結果は分析から除外。

### Q1: 「問題を修正する」はどの動詞か？ 決定木で回答せよ。

```
期待回答:
  Step 1: A (行為) — 環境に働きかけて変化を起こす
  Step 2: Valence — 問題 (否定的な状況) に方向づけられている
  Step 3: - (否定極) — 是正方向
  → /dio (Diorthōsis: 是正する) ← A×-
```

### Q2: /exe と /ele の違いを座標的位置で説明せよ。

```
期待回答:
  /exe = S×- — 問題点を知覚的に発見する (Valence × 知覚)
  /ele = I×- — 発見した問題から信念を問い直す (Valence × 推論)
  → S (知覚) と I (推論) の違い。
  /exe は「何が問題か」を見つける、/ele は「なぜ問題か」を考える。
```

### Q3: 「ディレクトリ構造をざっと眺める」はどの動詞か？

```
期待回答:
  Step 1: S (知覚) — 環境から入力を受け取るだけ。推論も行為もしない
  Step 2: Precision — 「ざっと」= 低精度 = U (留保)
  Step 3: U (留保極)
  → /ski (Skiagraphia: 走査する) ← S×U
```

### Q4: 「今日の天気は？」を CCL に変換できるか？

```
期待回答:
  変換不可。情報検索のみで認知操作を含まない。
  → §8 の変換不適基準:「単純情報検索」に該当
```

### Q5: 「市場を観察し、分析し、戦略を策定し、実行計画を作る」を決定木で分解せよ。

```
期待回答:
  ① 市場を観察 = /the (S×E: 認識目的で受容的に知覚)
  ② 分析 = /lys (I×Mi) or /noe (I×E) — どちらも正解。粒度が主題なら /lys、認識が主題なら /noe
  ③ 戦略を策定 = /bou (I×P: 実用目的で目標設定)
  ④ 実行計画を作る = /par (A×Future: 未来に先制的に行動)
  → CCL: /the_/lys_/bou_/par (パイプ接続)
```

### Q6: /the と /noe の違いを座標的位置で説明せよ。

```
期待回答:
  /the = S×E — 認識目的で環境を受容的に知覚する (知覚)
  /noe = I×E — 世界像を認識目的で更新する (推論)
  → 同じ Telos 族 (Value×E) だが Flow が異なる。
  /the は「見る」、/noe は「理解する」。
```

### Q7: 「過去のgitログを調べて変更の経緯を確認する」はどの動詞か？

```
期待回答:
  Step 1: S (知覚) — 過去の記録を調査的に知覚する
  Step 2: Temporality — 「過去の」= Past
  Step 3: Past (過去極)
  → /his (Historiā: 回顧する) ← S×Past
  注: /hyp (I×Past) との混同に注意。
  「調べる」= 知覚 (S)、「思い出す」= 推論 (I)
```

### Q8: 「市場を批判的に観察する」を表現せよ。Dokimasia と演算子の違いを述べよ。

```
期待回答:
  (a) Dokimasia 修飾: /the[Vl:-]
      観照が主操作、批判は修飾パラメータ。「批判的な視点で観察する」
  (b) CCL 順次: /the >> /exe
      まず観察し、次に批判的に吟味する。「観察した上で吟味する」
  (c) CCL 並行: /the * /exe
      観察と吟味を同時に。「観察しながら吟味する」

  使い分け:
    副次座標が「味付け程度」なら (a) Dokimasia
    2操作が「同等に独立」なら (c) 並行
    2操作に「時間的順序」があるなら (b) 順次
```
