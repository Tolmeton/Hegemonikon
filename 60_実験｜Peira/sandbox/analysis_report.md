## 実験結果

| 指標 | 値 |
| :--- | :--- |
| Mean(S_implicit) | 1.0000 |

## 圏論的観点からの構造的差異の考察

本検証では、記号的な AST走査に基づく忘却率分布 $\Xi_{\mathrm{discrete}}$ の Gini 係数が0 (分散完全欠落) となる一方、特徴ベクトルに基づく $\Xi_{\mathrm{search}}$ が約0.57 の構造的不均一性を持つことが確認されました。この「構造の抽出（S）と特徴ベクトルの分散（Ξ）」の差異について、以下の圏論的観点から説明します。

### 1. 記号AST抽出における過剰な忘却 (Gini ≈ 0)
AST 走査による S(e) の計算は、Python の制御構文（`F:`, `I:`, 演算子など）を直接抽出します。しかし、これら制御フローそのものは、それ単体では Hegemonikón の6次元座標（Explore/Exploit, Epistemic/Pragmatic など）のセマンティクスを持ちません。
圏論的には、これを関手 $U_{\mathrm{AST}}: \mathrm{Code} \to \mathrm{AST}$ と捉えると、この関手はコードが「どのような意図や目的でその制御フローを用いたか」という高次の関係性（射や自然変換）を完全に忘却する**強力な忘却関手**として働いています。その結果、成分ごとの暗黙座標ベクトルへの写像が空となり、すべての座標で $S(e) = 1$（完全忘却）が発生し、分布の構造 $\Xi_{\mathrm{discrete}}$ は平坦（Gini = 0）になってしまいます。

### 2. 特徴ベクトルの分散表現が保持する構造 (Gini ≈ 0.57)
一方、`ccl_features` 等のベクトル空間への埋め込みは、コード全体の構造パターン（内包表記、例外処理の有無や頻出度合いなど）を統合して多次元の連続値として捉えます。
これを関手 $F_{\mathrm{feature}}: \mathrm{Code} \to \mathbb{R}^{49}$ と捉えた場合、個々の記号が持つ直接の座標（対象）ではなく、コード全体の構成要素同士の「組み合わせや偏り（射の合成や構造の歪み）」を連続空間の計量として保存・射影しています（より faithful な変換）。これにより、特定の機能や領域への偏り（= 意図の強さや文脈）がベクトル変動のばらつき（CV）として現れ、結果として Gini 係数が有意な値を持つ非自明な構造を形成します。

### 3. 結論
この差異は、「局所的な記号としての抽出」が目的や機能の次元を削ぎ落としてしまうのに対し、「高次元特徴ベクトルへの射影」が、変数や制御構文間の関係性（morphism）といった背後の構造を効果的に保持していることを証明しています。
$S(e)$ の構造欠落は、AST 構造だけでは FEP の「どのように推論が行われているか」という情報を捉えきれないことを示しており、**コードの検索や意味分析において、単純な AST マッチングではなく多次元の分散表現 ($\Xi$) を利用することの圏論的および理論的妥当性**を裏付ける結果です。
