# 【品質管理 Vol.1】コードをPushしたら20人の「AI専門家」が待ち構えている世界線 — Synergeiaによる超並列レビュー

「プルリクエスト（PR）を出してから、レビューが返ってくるまでの待ち時間」に、もどかしさを感じたことはありませんか？

あるいは、先輩エンジニアにレビューを依頼したものの、「てにをは」や「インデントのズレ」ばかり指摘されて、肝心のロジックやセキュリティの議論が深まらなかった経験はないでしょうか。人間がやるべきは本質的な設計の議論であり、機械的なチェックではありません。

もし、あなたがコードをPushした瞬間に、**20人の各分野のトップエキスパート**が一斉にあなたのコードを読み込み、数分で完璧なフィードバックを返してくれるとしたらどうでしょう？

今回は、そんな未来の品質管理システム、マルチエージェント・レビューシステム「**Synergeia（シナジェイア）**」のアーキテクチャと、それを実現する思考法を紹介します。

## 単なる「AIレビュー」ではない。20人の「専門家集団」

最近では GitHub Copilot や ChatGPT にコードレビューを頼むことも一般的になりました。しかし、単一のプロンプトで「このコードをレビューして」と投げても、返ってくるのは広く浅い指摘ばかりです。

**Synergeia** のアプローチは異なります。ここには、**明確な役割（ペルソナ）を持った20体のAIエージェント**が存在します。

彼らは「何でも屋」ではありません。セキュリティの専門家はセキュリティのことしか見ませんし、可読性の専門家は変数名の命名規則に命をかけています。

### 専門家（エージェント）のペルソナ例

20人のうち、主要なメンバーを紹介しましょう。彼らはそれぞれ異なる「システムプロンプト（人格と評価基準）」を持っています。

| エージェント名 | 専門分野 | レビュー基準・口癖 | 出力フォーマット |
| :--- | :--- | :--- | :--- |
| **Sentinel (番人)** | セキュリティ | OWASP Top 10に基づき、SQLインジェクションやXSSの可能性を徹底的に探す。「入力値はすべて疑え」 | 脆弱性レベル (Critical/High) + 修正案 |
| **Speedster (走者)** | パフォーマンス | N+1問題、無駄なループ、メモリリークの予兆を検知。「O(n^2)は許さない」 | 計算量解析 + ボトルネック指摘 |
| **Librarian (司書)** | 可読性・保守性 | 変数名の明確さ、関数の粒度、認知負荷の低さを評価。「半年後の自分が読めるか？」 | リファクタリング提案 |
| **Lawyer (弁護士)** | ライセンス・依存 | `package.json` を監視し、互換性のないライセンスや古い依存関係を指摘。 | コンプライアンス警告 |
| **Storyteller (語り部)** | ドキュメント | JSDocやコメントが実装と乖離していないか、READMEが更新されているかを確認。 | ドキュメント修正差分 |

このように役割を分割することで、汎用的なLLMでは見逃されがちな「深い」指摘が可能になります。

## CI/CD パイプラインへの統合

この20人は、GitHub Actions などのCI/CDパイプラインの中で待ち構えています。開発者がPRを作成したトリガーで、彼らは並列に起動します。

以下は、このワークフローをイメージするための GitHub Actions の設定例です。

```yaml
name: Synergeia AI Review

on:
  pull_request:
    types: [opened, synchronize]

jobs:
  ai_council:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      
      # 変更されたコードの抽出
      - name: Get Diff
        id: diff
        run: |
          git diff origin/main > changes.diff

      # 20人のエージェントを並列実行（マトリックス戦略）
      - name: Run Specialist Agents
        strategy:
          matrix:
            agent: [security, performance, readability, testing, documentation, ...]
        run: |
          python run_agent.py --role ${{ matrix.agent }} --input changes.diff --output review_${{ matrix.agent }}.json

      # 結果の統合とフィルタリング
      - name: Aggregate & Post Comment
        run: |
          python aggregate_reviews.py --input "review_*.json"
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

ここで重要なのは、**「レビュー結果のマージ（統合）」**プロセスです。

### 20人の声をどう統合するか？

20人がそれぞれ好き勝手にコメントをPRに書き込むと、通知が爆発し、開発者はやる気を失います。そこで、Synergeiaには**「議長（Chairperson）」**という役割のエージェントが存在します。

議長は以下のロジックで意見を統合します。

1.  **確信度スコアリング**: 各エージェントは指摘事項に「確信度（0.0〜1.0）」と「重要度（Critical/Info）」を付与します。
2.  **ノイズ除去**: 確信度が低く、重要度が「Info」レベルの細かい指摘（例：好みの問題に近いスタイル指摘）は、設定された閾値以下なら切り捨てます。
3.  **要約と優先順位付け**: 重複する指摘をまとめ、修正すべき優先順位順に並べ替えます。

結果として、PRにはたった1つの、しかし濃密なコメントが投稿されます。

> **🤖 Synergeia Review Summary**
> 
> **🚨 Critical (修正必須)**
> *   [Security] `user_input` がサニタイズされずにSQLクエリに埋め込まれています。(確信度: 0.98)
> 
> **⚠️ Warning (検討推奨)**
> *   [Performance] `users.map` 内で `await` が使用されています。`Promise.all` の使用を検討してください。(確信度: 0.85)
> 
> **📝 Note**
> *   その他、可読性に関する軽微な提案が3件あります（折りたたみ表示）

## 導入後の実測効果

このシステムを導入した開発チームでは、劇的な変化が起きました。

1.  **人間によるレビュー時間の短縮（-40%）**
    *   人間は「ビジネスロジックが正しいか」「仕様を満たしているか」の確認に集中できるようになりました。構文やセキュリティの基本チェックはAIが済ませているからです。
2.  **バグ発見率の向上**
    *   特に「依存ライブラリの脆弱性」や「エッジケースでのパフォーマンス劣化」など、人間が見落としがちなポイントを専門エージェントが拾い上げます。
3.  **心理的安全性の向上**
    *   人間から「ここ、書き方汚いよ」と言われるとカチンときますが、AIから「可読性向上の提案」として提示されると、素直に受け入れられるという意外な効果もありました。

## 読者が今すぐ試せること

いきなり20人のエージェントシステムを構築するのは大変ですが、**「ペルソナを与えてレビューさせる」**という手法は今すぐChatGPTやClaudeで試せます。

普段のプロンプトを、以下のように変えてみてください。

**【いつものプロンプト】**
> このコードをレビューしてください。
> `[コード貼り付け]`

**【マルチペルソナ・プロンプト（推奨）】**
> あなたは**世界最高峰のセキュリティエンジニア**です。以下のコードに潜む脆弱性だけを、OWASP Top 10の観点から厳しく指摘してください。パフォーマンスや可読性は無視して構いません。
> `[コード貼り付け]`

これを「パフォーマンスエンジニア版」「初心者エンジニア版（可読性チェック用）」など、視点を変えて3回投げるだけで、レビューの質は劇的に向上します。
