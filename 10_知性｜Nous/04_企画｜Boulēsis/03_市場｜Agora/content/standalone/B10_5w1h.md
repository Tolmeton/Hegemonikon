
# 「正論だけど動けない」をなくす：AIのアウトプットを行動可能にする5W1H強制法

## 綺麗なレポートより、泥臭い手順書が欲しい時がある

AIに業務マニュアルの作成や、トラブル対応の指示を求めたとき、こんな回答が返ってきたことはありませんか？

「状況を把握し、適切な部署と連携して、迅速に対応してください。」

文章としては完璧です。正論です。しかし、現場でトラブルに直面している担当者からすれば、**「で、具体的に何をすればいいの？」**と言いたくなります。「適切な部署」とはどこなのか？ 「迅速に」とは何分以内なのか？

AIの出力は往々にして「正確だが、行動できない（Not Actionable）」ものになりがちです。本記事では、AIのアウトプットを抽象的な概念論から、明日から使える具体的な行動計画へと変換する**「5W1H強制法」**について解説します。

## 「正しさ」と「行動可能性」は別物

AIは学習データに含まれる大量のビジネス書や一般論の影響を受けるため、放っておくと「コンサルタントのような抽象的なアドバイス」を生成しがちです。

しかし、実務で必要なのは**Actionability（行動可能性）**です。これを担保するために、以下の2つのテスト基準を導入します。

1.  **読み手テスト**: そのテキストを初めて読んだ人が、誰にも質問せずに一人でタスクを完了できるか？
2.  **BC-15（Blue Collar 15）**: 15歳のアルバイト初日のスタッフでも理解し、行動できるか？（他者理解可能性）

これらをクリアするために有効なのが、古典的ですが最強のフレームワーク「5W1H」の強制適用です。

## 5W1Hを強制するテクニック

AIに対して、単に「具体的に書いて」と言うだけでは不十分です。各要素を埋めるように強制します。特に重要なのは**How（手順）**と**Where（場所）**です。

*   **Who（誰が）**: 「担当者」ではなく「総務部の山田さん」や「root権限を持つ管理者」。
*   **When（いつ）**: 「迅速に」ではなく「障害検知から15分以内に」。
*   **Where（どこで/どこに）**: 「設定ファイル」ではなく「`/etc/nginx/nginx.conf`の35行目」。
*   **What（何を）**: 「確認する」ではなく「値が`on`になっているか目視する」。
*   **Why（なぜ）**: 行動の理由（省略可能な場合もあるが、モチベーション維持に重要）。
*   **How（どのように）**: 具体的なコマンド、操作手順、クリックするボタンの名称。

これらが一つでも欠けていると、読み手はそこで立ち止まってしまいます。

## 具体事例：サーバー障害対応マニュアル

**【改善前：抽象的な指示】**
「サーバーの応答がない場合、ログを確認して原因を特定してください。必要に応じて再起動を行ってください。」

これでは、新人は動けません。「ログってどこ？」「再起動していい基準は？」と迷います。

**【改善後：5W1H強制適用】**
*   **When**: 監視アラート（HTTP 500）を受信してから5分以内に
*   **Who**: オンコール担当エンジニアが
*   **Where**: 管理画面の「System Logs」タブ（URL: xxx/logs）にて
*   **What**: 最新のエラーログを確認し、「Out of Memory」の文字列があるか検索する
*   **How**:
    1. ターミナルを開く
    2. コマンド `ssh user@192.168.1.10` を入力
    3. コマンド `sudo systemctl restart web-app` を実行
*   **Why**: メモリリークによるプロセス停止を一時的に復旧させるため

ここまで書かれていれば、誰でも迷わず行動できます。

## 抽象的回答 vs 行動的回答

| 要素 | 抽象的なAI回答（NG） | 行動可能なAI回答（OK） |
| :--- | :--- | :--- |
| **主語** | 「私たち」「チーム」 | 「プロジェクトマネージャー」「Aさん」 |
| **期限** | 「なるべく早く」「適宜」 | 「2024年10月15日 17:00までに」 |
| **対象** | 「関連資料」「データ」 | 「フォルダ `Z:\ProjectA\Docs` 内の `spec_v2.pdf`」 |
| **アクション** | 「検討する」「調整する」 | 「メールを送る」「会議を設定する」「値を書き換える」 |
| **判断基準** | 「状況に応じて」 | 「数値が80%を超えた場合」 |

## 読者が試せること：行動強制プロンプト

AIにマニュアル作成やアドバイスを求める際、以下のプロンプトを末尾に追加してください。これだけで、出力が「評論家の意見」から「現場監督の指示書」に変わります。

```markdown
# Output Constraint: Actionability Check
出力する内容は、以下の「5W1H」の要素をすべて具体的に含み、読者が迷わず行動できるものにしてください。

1. **Who**: 具体的な役割名や担当者を指定する。
2. **When**: 具体的な期限、タイミング、トリガー条件を指定する。
3. **Where**: ファイルパス、URL、物理的な場所、メニューの階層を指定する。
4. **What**: 対象となる具体的なオブジェクトや数値を指定する。
5. **How**: ステップ・バイ・ステップの手順、具体的なコマンド、スクリプト、定型文を提供する。
6. **Why**: その行動が必要な理由を簡潔に添える。
