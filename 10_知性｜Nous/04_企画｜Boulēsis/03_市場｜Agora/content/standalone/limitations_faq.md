## HGK Skills 全般

### Q: HGK Skills は何が違うのですか？
**A:** 一般的な AI プロンプトと HGK Skill の違い:

| | 一般的プロンプト | HGK Skill |
|:--|:----------------|:----------|
| 形式 | 自然言語指示 | 構造化 .md ファイル |
| 持続性 | 毎回入力 | ファイルとして永続 |
| 品質保証 | なし | 4-Tier 品質ゲート |
| 組み合わせ | コピペ | CCL 合成演算 |

### Q: どの AI で使えますか？
**A:** SKILL.md 形式は Claude (Anthropic) に最適化されていますが、基本的には AI アシスタントに .md を読ませる仕組みです。

| AI | 対応度 | 備考 |
|:---|:-------|:-----|
| Claude (Anthropic) | ◎ | 主要対象。YAML frontmatter 対応 |
| Gemini (Google) | ○ | GEMINI.md として使用可能 |
| ChatGPT (OpenAI) | △ | Instructions として部分使用可 |
| Cursor / Windsurf | ○ | .cursorrules / Rules として |

## 苦手なこと (正直な開示)

### Q: HGK Skills が苦手なことは？
**A:** 以下は**苦手**です。代替を提示します。

| 苦手なこと | 理由 | 代替提案 |
|:----------|:-----|:---------|
| **画像生成** | テキストベース体系 | Midjourney, DALL-E |
| **リアルタイム検索** | ファイルベース設計 | Perplexity, Grok |
| **GUI 操作の自動化** | CLI/API 前提設計 | Selenium, Playwright |
| **大規模データ分析** | トークン制限あり | Python + pandas 直接 |
| **法律/医療の専門判断** | 責任範囲外 | 専門家に相談 |

### Q: セキュリティは大丈夫ですか？
**A:** Skill は **ローカルのテキストファイル**です。

- ✅ データはあなたのマシンに残る
- ✅ 外部サーバーへの送信なし (Skill 自体は)
- ⚠️ AI に渡した内容は各 AI サービスの利用規約に従う
- ⚠️ API キーや機密情報は Skill に直書きしない

### Q: 動かないときはどうすれば？
**A:** トラブルシューティング順序:

1. **YAML frontmatter を確認** — `---` で囲まれているか？
2. **triggers を確認** — 発動キーワードは正しいか？
3. **AI に直接聞く** — 「この SKILL.md を読んで、何ができるか教えて」
4. **最小構成で試す** — 手順を 3 ステップまで減らして再試行
5. **別の AI で試す** — Claude で動かなければ Gemini で試す

## 商品としての限界

### Q: 無料のプロンプトと何が違うのですか？
**A:** 正直に言えば、T0 (SKILL.md 単体) は無料プロンプトと大差ありません。

| Tier | 無料プロンプトとの差 | 価格帯 (想定) |
|:-----|:-------------------|:-------------|
| T0 | 小さい (構造化のみ) | 無料 / フリーミアム |
| T1 | 中 (実行例付き) | ¥500-1,000 |
| T2 | 大 (自動化込み) | ¥2,000-5,000 |
| T3 | 明確 (品質保証済み) | ¥5,000-10,000+ |

> **T0 は無料で配る。T2-T3 で稼ぐ。** 差別化は自動化と品質保証。

### Q: 返金はできますか？
**A:** デジタルコンテンツのため原則返金不可。ただし:
- T0 は無料公開しているので購入前に試せる
- 動作不良の場合はサポート対応
- 記載内容と実際の動作が異なる場合は返金対応

## HGK 体系との関係

### Q: HGK を知らなくても使えますか？
**A:** **はい。** Skill は独立して動作します。

- `/xxx` コマンドを覚えるだけで使える
- HGK の理論 (FEP, 圏論等) は知らなくてよい
- より深く使いたい場合は体系の理解が役立つ

### Q: CCL って何ですか？
**A:** Cognitive Control Language — Skill を組み合わせる記法です。

```
/analyze~/review    ← 分析と確認を交互に実行
/plan >> /build     ← 計画が収束したら実装に進む
```

初日は知らなくて OK。使いこなしたくなったら覚える。
