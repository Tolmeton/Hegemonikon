
# VectorDBアダプタの抽象化設計 — 技術選定の迷いから解放されるアーキテクチャ

RAG（Retrieval-Augmented Generation）システムを開発する際、避けて通れないのが **VectorDB（ベクトルデータベース）** の選定です。

Pinecone、Weaviate、Qdrant、Chroma、Faiss... 選択肢は無数にあり、毎月のように新しい技術が登場します。「今はこれがベスト」と思って選んでも、半年後には「あっちの方が早くて安い」となるのがこの界隈の常です。

特定のライブラリにべったり依存したコード（密結合）を書いてしまうと、後で乗り換える際に地獄を見ます。そこで私は、**Adapter Pattern** と **Abstract Base Class (ABC)** を組み合わせ、VectorDBをいつでも差し替え可能な設計にしました。

## Factory + Adapter パターンによる抽象化

私が採用したのは、典型的な「依存関係逆転の原則」に基づく設計です。アプリケーションのコアロジックは、具体的なライブラリ（例えばFaiss）のことを一切知りません。知っているのは「検索ができる」「保存ができる」という抽象的なインターフェースだけです。

### 設計の全体像

1.  **VectorDBInterface (ABC)**:
    `add_documents()`, `search()`, `save()` といったメソッドを定義した抽象基底クラス。
2.  **Concrete Adapters**:
    上記のインターフェースを実装した具体的なクラス群。
    *   `HNSWlibAdapter`
    *   `FaissAdapter`
    *   `SQLiteVSSAdapter`
3.  **VectorDBFactory**:
    設定ファイル（config.yaml）を読み込み、指定されたアダプタのインスタンスを生成して返す工場。

この構成により、コードを1行も書き換えることなく、設定ファイルの `type: faiss` を `type: hnswlib` に書き換えるだけで、裏側のデータベースエンジンを切り替えることが可能になりました。

## 実装した3つのアダプタと使い分け

実際に実装し、切り替えて運用している3つのライブラリの比較は以下の通りです。

| ライブラリ | 特徴 | メリット | デメリット | 推奨ユースケース |
| :--- | :--- | :--- | :--- | :--- |
| **HNSWlib** | 高速な近似近傍探索 | 非常に軽量で高速。メモリ効率が良い。 | 機能がミニマル。永続化の管理が手動。 | 小〜中規模のオンメモリ検索。開発時のプロトタイプ。 |
| **Faiss** | Meta製のデファクト | CPU/GPU対応。アルゴリズムが豊富。 | 依存関係が重い（特にGPU版）。インストールが面倒な場合も。 | 大規模データ。精度と速度のチューニングが必要な本番環境。 |
| **SQLite-VSS** | SQLiteの拡張 | DBと同じファイルで管理可能。ポータビリティ最強。 | まだ発展途上で機能制限あり。大規模には不向き。 | エッジデバイスや、単一ファイルで完結させたい配布用アプリ。 |

当初はFaissを使っていましたが、ローカルでの軽量なテスト環境が欲しくなり、HNSWlibアダプタを追加しました。この際、メインロジックには一切触れずに済んだため、この設計の有効性を痛感しました。

## 読者が試せること：PythonでInterfaceを定義してみる

あなたのプロジェクトでも、将来変わりそうな依存先（VectorDBに限らず、LLMのAPIクライアントなど）があれば、まずは抽象クラスを定義してみてください。

```python
from abc import ABC, abstractmethod

class VectorStore(ABC):
    @abstractmethod
    def search(self, query_vector: list[float], top_k: int = 5):
        pass

    @abstractmethod
    def add(self, vector: list[float], metadata: dict):
        pass
```

たったこれだけのことですが、これを継承して実装クラスを作る癖をつけるだけで、あなたのコードの寿命と柔軟性は劇的に向上します。「技術選定の迷い」をコードに持ち込まない。それが、変化の激しいAI時代を生き抜くコツです。
