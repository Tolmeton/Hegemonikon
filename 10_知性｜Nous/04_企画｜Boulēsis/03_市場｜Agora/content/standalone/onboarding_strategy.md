## 1. プロンプトを磨くな、テキストファイルを磨け

> Ruben: "Don't polish your prompts. Polish your text files."

### HGK 翻訳

プロンプトエンジニアリングは**毎回の対話コスト**。テキストファイル (KI) は**一度の投資で永続効果**。

| 一般向け表現 | HGK 等価物 | ユーザーへの説明 |
|:-----------|:----------|:-------------|
| ブランドルール .md | Identity Stack (GEMINI.md) | 「AI の設定ファイル。一度書けば毎回読み込まれる」 |
| 過去の作品集 .md | KI (Knowledge Item) | 「あなたの専門知識を AI に教えるファイル」 |
| 指示書 .md | Skill (SKILL.md) | 「特定の作業手順を AI に教えるレシピ」 |

### Agora 適用

Skill を販売する際、「使い方のプロンプト」ではなく「設定ファイル (.md) を置くだけ」というオンボーディングを設計する。

## 2. 30分セットアップ — 人間向け /boot

> Ruben の 30分構造: 10分(Cowork設定) + 10分(テキスト準備) + 10分(初回対話)

### HGK 翻訳

`/boot` は AI 向けのセッション初期化。人間にも「最初の30分」の構造が必要。

| 分 | Ruben のステップ | Agora 版 |
|:--|:---------------|:---------|
| 0-10 | Cowork 設定 | Skill インストール + 環境確認 |
| 10-20 | テキストファイル準備 | KI テンプレートの記入 (自分の業務を記述) |
| 20-30 | 初回対話 | `/dok` (小さな実験) で最初の成功体験 |

### 設計原則

- **最初の成功体験を30分以内に**: 抽象的な説明より体験
- **テンプレートを用意する**: 白紙から書かせない。穴埋め式
- **段階的複雑化**: 初日は 1 Skill。1週間後に追加

## 3. Plugin = Skill 軽量版

> Ruben: Plugin は slash command で専門能力を追加する

### Agora 適用

| Anthropic Plugin パターン | Agora Skill 設計への示唆 |
|:------------------------|:----------------------|
| 1クリックインストール | SKILL.md をコピーするだけで動く設計 |
| slash command で発動 | WF 名 (/xxx) で発動する一貫 UX |
| 専門化 (Markdown Wizard 等) | 業界特化 Skill (FM, 翻訳, 分析等) |
| 組み合わせ可能 | CCL の合成演算 (~, >>, *) |

### 競争優位

Anthropic の Plugin は **T3 (tier-3)**。HGK の Skill は **T1 (tier-1)** まで到達可能:
- SKILL.md + scripts/ + examples/ + REFERENCE.md
- 4-tier 権限モデルによる品質保証
- CCL による Skill 間の合成

## 4. 弱点の正直な開示

> Ruben: 「Claude は画像生成不可。検索は Grok が上」

### Agora コミュニケーション戦略

**弱点を隠さず、代替を提示する** = N-3 (確信度明示) の商業的応用。

| やってはいけない | やるべき |
|:-------------|:---------|
| 「何でもできます」 | 「X は得意。Y は苦手。Y には Z を推奨」 |
| 弱点を無視 | 弱点を明示 + 代替ツール提案 |
| 過剰な約束 | 実績ベースの説明 |

### 信頼構築の公式

信頼 = 正直な弱点開示 × 具体的代替提案 × 実績
