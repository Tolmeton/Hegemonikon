# 【CCL全貌 #3】思考をコード化する「CCL」1088行の設計と実装

## 1. CCL (Cognitive Control Language) とは

CCLは、AIの思考プロセス（認知ワークフロー）を定義するためのドメイン固有言語（DSL）です。

SQLがデータベースへの命令を記述するように、CCLは**LLM（大規模言語モデル）の脳内で行われる推論の手順**を記述します。

通常、私たちは「段階的に考えて」と指示しますが、CCLでは「拡散せよ」「蒸留せよ」「振動（試行錯誤）せよ」といった抽象度の高い認知操作を記号で表現します。これにより、複雑な推論チェーンを数行のコードで定義し、再現可能な形に落とし込むことが可能になります。

## 2. 思考の演算子：文法定義

CCLの核心は、認知操作を物理演算のように捉えた「演算子」にあります。仕様書で定義されている主要な記号を見てみましょう。

| 演算子 | 名称 | 意味 | 認知プロセスへの作用 |
| :--- | :--- | :--- | :--- |
| `/` | **Call** | ワークフロー呼出 | 定義済みの認知モジュール（関数）を呼び出す |
| `+` | **Expand** | 展開・拡散 | アイデアを広げる、詳細化する、可能性を列挙する |
| `-` | **Distill** | 蒸留・要約 | 情報を削ぎ落とす、本質を抽出する、抽象化する |
| `~` | **Oscillate** | 振動・検討 | 対立概念を行き来する、メリット・デメリットを揉む |
| `~*` | **Converge** | 収束 | 振動した議論を一つの結論に落とし込む |
| `>>` | **Serial** | 直列接続 | 前の思考結果を次の入力として渡す（パイプライン） |
| `{}` | **Group** | 並列処理 | 複数の思考を同時に走らせる |
| `()` | **Scope** | 部分式 | 演算の優先順位を制御する |

これらを組み合わせることで、**「広げて（+）、揉んで（~）、絞る（-）」**という思考の呼吸を記述できます。

## 3. 深度派生：System 1 と System 2 の制御

人間の思考に「直感（速い思考）」と「熟考（遅い思考）」があるように、CCLでは同じモジュールでも処理深度（Depth）を指定できます。これを**深度派生（Depth Derivation）**と呼びます。

例えば、`/noe`（Noesis：思考）というモジュールがあった場合：

```ccl
/noe-   // L1: Lightweight（直感）
        // トークン数を節約し、表層的な回答や概要を素早く生成する。

/noe    // L2: Standard（標準）
        // 通常の推論。論理的整合性を保ちながら回答する。

/noe+   // L3: Deep（深層）
        // ステップ・バイ・ステップで深く掘り下げる。
        // 自己批判や多角的な検証プロセスが自動的に付与される。
```

末尾に `+` や `-` を付けるだけで、LLMの計算リソース（トークン量と推論時間）の配分をコントロールできる設計です。

## 4. マクロ：思考パターンの再利用

プログラミング同様、よく使う思考パターンはマクロとして定義可能です。仕様書には標準ライブラリとしていくつかのマクロが含まれています。

*   **@repeat(n)**: 指定した思考プロセスをn回繰り返す（ブレインストーミングなどで使用）。
*   **@proof**: 論理的証明モード。前提条件の確認→推論→結論という厳格なパスを強制する。
*   **@dig**: 「なぜ？」を繰り返す深掘りモード。根本原因分析（RCA）に使用。

```ccl
// 例：アイデアを出し、それを深掘りして検証する
/idea+ >> @dig >> @proof
```

## 5. パーサー：Hermēneus（解釈者）

CCLは単なる文字列置換ではありません。**Hermēneus（ヘルメネウス）**と呼ばれるパーサーが実装されています。

処理の流れは以下の通りです：

1.  **Lexer**: CCL文字列をトークンに分解。
2.  **Parser**: 抽象構文木（AST）を構築。ここで構文エラー（閉じ括弧忘れなど）を検知。
3.  **Optimizer**: 実行計画の最適化。冗長なパス（例：`++` 展開してすぐ展開など）を整理。
4.  **Executor**: LLMへの具体的なプロンプト（自然言語）にトランスパイルして実行。

この構造により、将来的にLLMのモデルが変わっても、CCLのコード自体は変更せずに、トランスパイラ部分の調整だけで対応が可能になります。

## 6. 実装例：戦略策定の「数式」

では、実際に複雑な思考をCCLで書いてみましょう。
以下は、あるビジネス課題に対して「目的設定から戦略を立て、シミュレーションを行い、実行可能性を診断する」という一連の流れです。

**CCLコード:**
```ccl
/bou+ _ /s+ ~ (/p* /a) _ /dia _ /pis
```
※ `_` は可読性のためのセパレータ（無視されます）

この1行が、内部では以下のような壮大な思考フローに展開されます。

1.  **`/bou+` (Boulesis Expand)**
    *   **目的の展開**: ユーザーの曖昧な要望から、真の目的（Will）を多角的に展開・定義する。
2.  **`/s+` (Strategy Expand)**
    *   **戦略の立案**: 定義された目的に対し、複数の戦略オプションを広く生成する。
3.  **`~ (/p* /a)` (Oscillate Parameters & Actions)**
    *   **シミュレーション**: ここが重要です。
    *   `/p` (Parameters/変数値) と `/a` (Action/行動) の間で `~` (振動) させます。
    *   つまり、「この行動をとるとパラメータ（売上やコスト）はどう動くか？」「逆にこのパラメータを達成するにはどう行動すべきか？」を行き来させ、動的なシミュレーションを行います。
4.  **`/dia` (Diagnosis)**
    *   **診断**: シミュレーション結果に対し、実現可能性やリスクを診断する。
5.  **`/pis` (Pistis)**
    *   **確信度**: 最終的なプランに対する「確信（Belief）」を提示し、Go/No-Goの判断材料とする。

たった1行の文字列が、これほどリッチな認知プロセスをトリガーします。これがCCLの威力です。

## 読者が試せること

CCLのパーサー（Hermēneus）そのものは複雑なシステムですが、**「LLMにCCLの仕様を理解させて、擬似的に実行させる」**ことは今すぐ可能です。

以下のプロンプトをChatGPTやClaudeに貼り付けて、CCLのパワーを体験してみてください。

### 🧪 CCLインタプリタ・プロンプト

```markdown
あなたは高度な認知制御言語「CCL」のインタプリタです。以下の仕様に従い、ユーザーが入力するCCLコードを解釈し、その思考プロセスをステップ・バイ・ステップで実行してください。

# CCL仕様概要
- `/`: 思考モジュール呼出 (例: /idea = アイデア出し)
- `+`: 展開・拡散 (詳細化、列挙)
- `-`: 蒸留・収束 (要約、本質抽出)
- `~`: 振動・検討 (メリット/デメリット、対立概念の往復)
- `>>`: 直列処理 (前の出力を次の入力へ)

# 入力
テーマ: 「リモートワークにおけるチームの結束力向上」
コード: `/problem+ >> /solution+ ~ /risk >> /plan-`

# 指示
上記のコードを解析し、各ステップの見出し（例: 【/problem+ 問題の展開】）をつけた上で、実際の思考内容を出力してください。
```

このプロンプトを実行すると、AIはまず問題を多角的に分析し、解決策を出した後、リスクと解決策の間で議論（振動）を行い、最後に具体的な計画へと蒸留して出力してくれます。
