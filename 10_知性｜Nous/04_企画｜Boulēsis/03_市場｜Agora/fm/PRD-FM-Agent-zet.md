## Jobs To Be Done (JTBD)

### S1: 社長の会社

| When... | I want to... | So I can... |
|:--------|:-------------|:------------|
| Forge が XML を生成した時 | 自動で FM に貼り付けたい | 手作業のミスを排除し、配備を数分で完了する |
| レイアウトを変更した時 | DDR と照合して変更影響を自動検知したい | 変更漏れ (フィールド欠落、ポータル不整合) を即座に発見する |
| 納品前のテスト時 | ポータルフィルタ・検索・ソートが正しく動くか自動確認したい | リグレッションを防ぎ、「動かない」クレームを事前に排除する |
| 顧客に進捗を説明する時 | 自動テストの結果レポートを見せたい | 品質保証の根拠を可視化し、信頼を獲得する |

### S2: FM 開発者一般

| When... | I want to... | So I can... |
|:--------|:-------------|:------------|
| FM スクリプトを大量に配備する時 | CLI から一括貼付したい | 反復的な GUI 操作から解放される |
| CI/CD パイプラインを構築したい時 | FM の変更を自動テストしたい | 他のプラットフォーム同様の開発フローを実現する |

## Value Propositions

| セグメント | 差別化要因 | 代替手段 | FM-Agent の優位性 |
|:-----------|:-----------|:---------|:-------------------|
| S1 (自社) | HGK 統合 (Forge → Agent → テスト) | 手作業 | E2E 自動化で工数 50%+ 削減 |
| S2 (FM 開発者) | FM 向け初の GUI 自動化ツール | 手作業 / MBS Plugin (有料) | VLM ベースで FM バージョン非依存 |
| S3 (情シス) | ノーコード感覚でテスト定義 | 外部委託 | 内製でテスト維持可能 |

**核心的価値**: FM Pro のような「API のないプラットフォーム」に対して、VLM で「仮想 API」を提供する。
