## 1. 戦略原理 — Bridge-to-Lemma Conversion

外部監査 (GPT) は論文 I を「過積載」と判定した。批評の想定する対処は「荷を減らせ」(分割 or 降格)。本 WB はそれを**拒否**する。

**原理**: 野心と厳密さは反比例しない、**比例する**。過積載とは荷が重いことではなく、**梁が細いこと**である。大聖堂は小屋より多くの荷を負う。

**変換**: 批評が「比喩」「proxy」「類推」と呼んだ各接続を、**定義と補題**に変換する。攻撃面は減らない — むしろ増える — が、各攻撃点が**定理として殴り返せる**ようになる。降格ではなく補題化。これを Bridge-to-Lemma Conversion と呼ぶ。

**降格戦略との違い**:
- 降格「力は比喩です」→ 批評「じゃあタイトル変えろ」
- 補題化「力とは以下の曲率 2-形式と定義する。この定義のもとで次の定理が成り立つ」→ 批評「…反論困難」

正直さは降格ではなく、**定義の明示**によって達成される。

## 3. 7 本の梁 (Beams)

各梁は以下のフィールドを持つ。Status は `not_started` / `attempting` / `proven` / `retreating` / `outlook` のいずれか。

### Beam-1 (最優先リトマス試験紙): A_i の一意性定理化

- **Status**: not_started
- **対応する監査攻撃**: #3 (§3.2/§3.4 の一意性)
- **目標命題**: A_i = ∂_i Φ + (α/2) Φ T_i が、指定した公理系のもとで**相対一意**に決まる
- **候補公理 4 本**:
  - A1: Φ のスカラー再スケーリング Φ → cΦ に対する共変性
  - A2: α に対する線形性 (Chebyshev 族の要請)
  - A3: Φ と T の最低次オーダー (Φ に 1 次、T に 1 次、それ以下は除外)
  - A4: α=0 で Levi-Civita に還元
- **進捗メモ**: 未着手。まず候補公理セットで同階候補項 (|dΦ|², Φ·T·dΦ, Φ²|T|²_g 等) が全て排除できるか検査する必要がある
- **停止条件**: 最小公理セットが書けず相対一意性が証明できなかった場合 → 挑戦路線を撤退し、前セッションの A案 (定理論文化) または B案 (Program 宣言化) へ降格

### Beam-2: dT=0 適用範囲の maximality 定理化

- **Status**: not_started
- **対応する監査攻撃**: 一般性の過剰 (§5.4)
- **目標命題**: 「正則・有限次元・自然パラメータ座標を持つ指数型分布族の class 𝓔 は dT=0 を満たす最大 class の一つ」
- **進捗メモ**: 少なくとも必要条件側は書ける (自然パラメータを持たない分布で dT≠0 の counter-example)
- **停止条件**: maximality 証明が無理なら「𝓔 で成立」の記述に留め、maximality 主張は outlook へ

### Beam-3: FEP 埋め込みを関手として正確に書く

- **Status**: not_started
- **対応する監査攻撃**: #5 (§6.5 FEP embedding)
- **目標命題**: 関手 𝓕: C → D を定義する
  - Domain C = (正則指数型分布族 + Chebyshev 接続)
  - Codomain D = (平衡点まわりで線形化された FEP 生成モデル圏)
  - 保存するもの: 曲率 (up to scale), 接続の反対称部分
  - 保存しないもの: 時間発展, 空間次元 (これは Paper II の別関手 𝓖 の領域)
- **進捗メモ**: 「時間拡張未実装」を**スコープ宣言**として前向きに述べる。「別稿依存」→「明示的スコープ」への変換
- **停止条件**: 関手の domain/codomain が厳密に書けない場合 → §6.5 は outlook 節へ分離

### Beam-4: Proxy を estimator として定義する

- **Status**: not_started
- **対応する監査攻撃**: #6 (Ξ, CKA, τ, coherence の対応欠如)
- **目標命題**: 各経験量を理論量の estimator として定義し、バイアス評価と仮定を明記
  - Ξ := ⟨d(ΦT) の empirical estimator 式⟩, バイアス O(n^-1/2), 仮定 H1 (i.i.d.)
  - CKA 経由の曲率測度 = 特定条件下で |F|_g の推定量、誤差範囲付き
  - Chunk coherence = ⟨T の成分の時系列自己相関⟩ の推定量
- **進捗メモ**: これが書けたら「proxy に逃げられる」批判が死ぬ。proxy ≡ estimator として定義されていれば、反証時は理論ではなく仮定 H1 を攻撃する方向が自動的に決まる
- **停止条件**: estimator の定義がどうしても書けない場合 → §5.5-§5.8 を別稿へ完全分離

### Beam-5: g^(α) を定義する

- **Status**: not_started
- **対応する監査攻撃**: #4 (α-SAM / OA-SAM 足場欠如)
- **目標命題**: g^(α) を「α-接続に対する最小自然計量」として明示的に定義
- **進捗メモ**: Amari / Čencov の Chebyshev 文献に対応物があるはず。候補具体形: g^(α)_{ij} = g^{(0)}_{ij} + (α²/4) T_i T_j
- **重要な意味論修正**: α-SAM を**性能改善定理ではなく表現制御定理**として書き直す。「α を動かすと representation が d(ΦT) 方向にシフトする」— これは §6.8.2 が実際に示していることで、p=0.067 が無効化される (性能ではなく方向の主張に変わる)
- **停止条件**: g^(α) の定義が Chebyshev 文献で見つからない場合 → §6.7-§6.8 を Paper I から削除

### Beam-6: 数値再現パッケージの添付

- **Status**: not_started
- **対応する監査攻撃**: 数値再現性 (§4.5 と §10 の 1.4e-10 vs 0.11% 矛盾)
- **目標**: コードと data を添付し、1.4e-10 と 0.11% が何の誤差なのかを 1 行で書き分ける (恐らく片方は解析式残差、もう片方はサンプリング誤差)

## 5. 停止判定 (リトマス試験紙)

**Beam-1 が公理化できなかった場合の降格経路**:

- **A案 (前セッション提案)**: 定理論文化。§5.4 で閉じる。15-20 頁。題名を退屈に (例「指数型分布族における Chebyshev 接続の曲率」)。査読耐性 ∞ だが忘却論シリーズの旗艦になれない
- **B案 (前セッション提案)**: Program 宣言化。明示的 manifesto として書き直す。定理ブロックは rigorous core として厳密に立て、FEP/LLM/最適化/哲学は working program として隔離。各項目に `[CONJECTURE]` `[PROXY-DEPENDENT]` `[FUTURE]` ラベル。野心を保ったまま正直さで防御する

**最大の罠**: 挑戦路線の失敗条件は 1 つ — 梁を増やす作業の途中で「この補題、実は証明できない」と気づいた時に、**それでも走らせてしまう**こと。橋の補題化は各補題が実際に証明されて初めて補強になる。証明できない補題を書いたら「過積載 + 嘘」という最悪の組み合わせになる。

**ルール**: 各梁について、証明できなかったら即座にそこを outlook に落とす。梁は書こうとして折れたら撤去、書けたら残す、**途中で止めない**。
