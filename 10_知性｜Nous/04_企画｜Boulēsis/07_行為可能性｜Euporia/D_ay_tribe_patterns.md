# AY 族パターン — 全スキル AY 標準化のための演繹的構造

> Euporía 公理: 全ての WF 射 f: A→B は AY(f) > 0 でなければならない。
> 本文書は AY の構造を 3パラメータ (Flow, Tribe, Pole) から CPS + 圏論で演繹する。

---

## §1 AY の演繹構造

### 定義 (再掲)

```
AY(f) ≝ EFE(B) - EFE(A) = Δepistemic + Δpragmatic
```

### CPS パイプライン (一般形)

全てのスキル V は以下の CPS パイプラインを持つ:

```
π_V → a_V → o_V → ε_V → π'_V → Hom(B_V, −)

π_V   = V の prior (V が操作する領域についての事前モデル)
a_V   = V の行為 (スキル実行そのもの)
o_V   = V の観測 (実行の結果として得られたデータ)
ε_V   = o_V - g(π_V) (prediction error: 想定と結果の差分)
π'_V  = π_V + α·ε_V (posterior: 更新されたモデル)
Hom(B_V, −) = V の出力から出る新しい射の集合 = AY
```

### 3パラメータ分解

AY(V) は 3つの独立パラメータで構造的に決定される:

```
AY(V) = AY_mech(Flow(V)) × AY_dim(Tribe(V)) × AY_dir(Pole(V))
```

## §2 パラメータ 1: Flow — ε の生成メカニズム

Flow 座標 (I/A/S) は **ε をどうやって生成するか** を決定する。

```
Flow = I⊣A (推論⊣行為) の随伴 + S (知覚 = I∪A への入力)
```

### I型 (推論, Afferent=No, Efferent=No)

```
CPS: π → μ(π) → ε_I → π'
  μ = 内部モデル操作 (分析・統合・比較・判断)
  ε_I = μ(π) - π (モデル操作による自己差分)
  特性: 世界を変えずにモデルを変える

AY_mech(I):
  epistemic 優勢: μ が π を改訂 → 以前は見えなかった構造が可視化
  pragmatic: 改訂された π' が A型動詞への入力を生む (I⊣A の unit η)
  圏論: I型の AY = Hom_I(π', −) \ Hom_I(π, −)
    = π' から出る推論的射のうち、π からは出なかったもの
```

### A型 (行為, Afferent=No, Efferent=Yes)

```
CPS: π → a(π) → o → ε_A → π'
  a = 世界への介入行為
  o = 世界からの観測フィードバック
  ε_A = o - g(π) (行為の結果と想定の差分)
  特性: 世界を変え、その結果を観測する (active inference)

AY_mech(A):
  pragmatic 優勢: a が世界の状態を変える → 以前は不可能だった行為が可能に
  epistemic: 行為による o が π を更新 (I⊣A の counit ε)
  圏論: A型の AY = Hom_A(B, −) \ Hom_A(A, −)
    = 行為の結果 B から出る行為的射のうち、行為前 A からは出なかったもの
```

### S型 (知覚, Afferent=Yes, Efferent=No)

```
CPS: π_att → attend(π_att) → o_S → ε_S → π'_att
  π_att = 注意の prior (何をどう知覚するかの期待)
  attend = 注意の配向 (どこに知覚を向けるかの選択)
  o_S = 知覚データ (受容した信号)
  ε_S = o_S - g(π_att) (知覚と期待の差分)
  特性: 世界を変えずに観測データを収集する

AY_mech(S):
  I と A への入力供給: S の o_S が I型の π と A型の π を更新する
  epistemic: 知覚が I型の推論入力を生む (φ_SI パイプライン)
  pragmatic: 知覚が A型の行為入力を生む (φ_SA パイプライン)
  圏論: S型の AY = Hom_S(o_S, −) の分解
    = Hom_I(o_S, −) ∪ Hom_A(o_S, −) (I型への射 + A型への射)
```

### Flow 間の随伴関係

```
S → I → A (φ_SI → I⊣A)

S の AY は I の π を供給 (S⊣I)
I の AY は A の π を供給 (I⊣A の unit η)
A の AY は I の π を更新 (I⊣A の counit ε)
A の AY は S の π_att を更新 (行為が注意を再配向)

→ AY は Flow 間で循環する: S → I → A → S...
```

## §3 パラメータ 2: Tribe — π の領域 (ε が何についてのずれか)

族座標は **π が何についての事前モデルか** を決定する。

### Telos族: π_T = 目的・価値のモデル

```
π_T = 「何のために / 何が価値あるか」の事前モデル
ε_T = 目的と現実のずれ / 価値判断の更新
Hom(B_T, −) = 目的が変わったことで開く射:
  新しい問い (→/zet), 新しい意志 (→/bou), 新しい認識の必要 (→/noe)
```

### Methodos族: π_M = 戦略・手法のモデル

```
π_M = 「どうやるか / 何を試すか」の事前モデル
ε_M = 戦略の有効性のずれ / 手法の妥当性の更新
Hom(B_M, −) = 戦略が変わったことで開く射:
  新しい仮説 (→/ske), 新しい実験 (→/pei), 新しい適用先 (→/tek)
```

### Krisis族: π_K = 確信度のモデル

```
π_K = 「どれだけ確かか / 信頼できるか」の事前モデル
ε_K = 確信度の較正ずれ / 信頼度の更新
Hom(B_K, −) = 確信が変わったことで開く射:
  投入可能な確信 (→/pai), 保留すべき不確実 (→/epo), 打診すべき方向 (→/dok)
```

### Diástasis族: π_D = 空間構造のモデル

```
π_D = 「どんな構造か / どのスケールか」の事前モデル
ε_D = 構造的驚き / スケール認知の更新
Hom(B_D, −) = 構造が見えたことで開く射:
  分解可能な要素 (→/lys), 修正可能な箇所 (→/akr), 展開可能な範囲 (→/arh)
```

### Orexis族: π_O = 価値傾向のモデル

```
π_O = 「何に向かい / 何を避けるか」の事前モデル
ε_O = 価値判断の更新 / 接近回避の再評価
Hom(B_O, −) = 価値判断が変わったことで開く射:
  踏破可能な未踏 (→/kop), 修正可能な破損 (→/dio), 承認可能な成功 (→/beb)
```

### Chronos族: π_C = 時間状態のモデル

```
π_C = 「過去に何があったか / 未来に何が起きるか」の事前モデル
ε_C = 時間的予測のずれ / 記憶・予見の更新
Hom(B_C, −) = 時間理解が変わったことで開く射:
  抽出可能な教訓 (→/ath), 先制可能なリスク (→/par), 想起可能な文脈 (→/hyp)
```

## §4 パラメータ 3: Pole — ε の方向

各族の座標軸は2極を持つ。Pole は **ε がどちら側に向かうか** を決定する。

```
定義: 族 T の2極を P₊, P₋ とすると
  V(P₊) の ε は P₊ 方向の射を開く
  V(P₋) の ε は P₋ 方向の射を開く
  H型対: V(P₊) ↔ V(P₋) は AY の方向が反転する
```

| 族 | P₊ 極 | P₋ 極 | AY 反転 |
|:--|:--|:--|:--|
| Telos | External (外部) | Internal (内部) | 外部認識 ↔ 内部意志 |
| Methodos | Explore (探索) | Exploit (活用) | 仮説拡大 ↔ 手法確定 |
| Krisis | Certain (確実) | Uncertain (不確実) | 確信固定 ↔ 判断留保 |
| Diástasis | Micro (微視) | Macro (巨視) | 局所深掘 ↔ 全体俯瞰 |
| Orexis | Positive (接近) | Negative (回避) | 未踏踏破 ↔ 破損修復 |
| Chronos | Past (過去) | Future (未来) | 想起回顧 ↔ 予見先制 |

### 極性と AY の構造

```
AY_dir(P₊) の Hom(B, −) は P₊ 方向の射を含む:
  具体例 (Orexis): /kop (P₊) の AY は Positive 方向 → 足場から出る「前進可能な道」

AY_dir(P₋) の Hom(B, −) は P₋ 方向の射を含む:
  具体例 (Orexis): /dio (P₋) の AY は Negative 方向 → 修復から出る「再び歩ける道」

H型対の AY 合成:
  AY(/kop) ∪ AY(/dio) = Orexis-A の完全な AY 空間
  P₊ と P₋ の AY を合わせると族座標の全方向がカバーされる
```

## §5 合成: AY(V) の完全な導出手順

任意のスキル V に対して:

```
Step 1: Flow(V) を特定 → ε の生成メカニズムを決定
  I型 → μ(π) による内部改訂 → epistemic 優勢
  A型 → a(π) による世界変化 → pragmatic 優勢
  S型 → attend(π) による観測 → I/A への入力供給

Step 2: Tribe(V) を特定 → π の領域を決定
  π が何についてのモデルかで、ε の次元が決まる

Step 3: Pole(V) を特定 → ε の方向を決定
  族座標のどちら側に ε が向かうかで、Hom(B, −) の方向が決まる

Step 4: Hom(B_V, −) を列挙
  = Flow で決まるメカニズム × Tribe で決まる次元 × Pole で決まる方向
  = 「このスキルを使った結果、次にどの動詞のどの側面が可能になるか」
```

### 導出例: /ske (V05 Skepsis)

```
Flow(V05) = I (推論)
Tribe(V05) = Methodos (戦略)
Pole(V05) = Explore (探索)

Step 1: I型 → μ(π_M) による π_M の改訂。epistemic 優勢
Step 2: π_M = 戦略のモデル → ε_M = 戦略空間の拡大
Step 3: Explore 極 → ε は仮説空間の拡大方向
Step 4: Hom(B_ske, −):
  epistemic: 以前は見えなかった仮説次元が可視化された
    → /sag で収束可能な候補空間 (H型対)
    → /pei で実験可能な新仮説
    → /noe でさらに深く認識可能な構造
  pragmatic: 前提が破壊されたことで
    → /pei で PoC 可能な新アプローチ
    → /bou で方向づけ可能な新選択肢
```

### 導出例: /akr (V15 Akribeia)

```
Flow(V15) = A (行為)
Tribe(V15) = Diástasis (空間構造)
Pole(V15) = Micro (微視)

Step 1: A型 → a(π_D) による世界変化。pragmatic 優勢
Step 2: π_D = 空間構造のモデル → ε_D = 修正箇所の局所的変化
Step 3: Micro 極 → ε は局所的精密操作の方向
Step 4: Hom(B_akr, −):
  pragmatic: 1箇所が精密に修正されたことで
    → /ops で俯瞰可能な全体整合性の検証
    → /beb で承認可能な品質の回復
    → /ene で次段階の実行が可能に
  epistemic: 精密修正の結果として
    → /lys で修正箇所周辺のさらなる分析が可能に (D型対)
    → /prs で修正結果の詳細確認が可能に
```

## §6 全スキル AY 出力フォーマット (標準形)

```
§ Affordance Yield:
  CPS: π={prior の内容} → ε={prediction error} → π'={更新モデル}
  epistemic (Δ知):
    - {ε によって初めてわかったこと} → {次の I/S 型 WF 候補} が可能に
  pragmatic (Δ行):
    - {ε によって初めてできること} → {次の A 型 WF 候補} が可能に
  AY = epi({N}) + prag({N}) = {合計}
  ΔVFE: {高/中/低} — 不確実性の減少量
```

深度別: L0=省略 / L1=prag≥1 / L2=prag≥2+epi≥1+CPS明示 / L3=prag≥3+epi≥2+EFE根拠+ΔVFE

## §7 定理: AY 射先決定規則

### 定理 (AY Target Determination)

任意のスキル V の AY 射先 Hom(B_V, −) は以下の6規則の和集合で**必然的に**決定される。

```
Hom(B_V, −) = R1(V) ∪ R2(V) ∪ R3(V) ∪ R4(V) ∪ R5(V) ∪ R6(V)
```

### R1: D型随伴規則

```
V₁⊣V₂ ならば:
  AY(V₂) ∋ V₁  — counit 方向: 行為の結果を随伴の左に返す
  AY(V₁) ∋ V₂  — unit 方向: 推論の結果を随伴の右に渡す

CPS 根拠: D型随伴の unit η: π_{V₁} → π_{V₂} は
  V₁ の π' が V₂ の π を更新する = V₁ の AY が V₂ への射を含む
  counit ε: V₂ の出力が V₁ の π を更新 = V₂ の AY が V₁ への射を含む
```

### R2: H型反転規則

```
V₊↔V₋ (族内 H型自然変換) ならば:
  AY(V₊) ∋ V₋  — 文脈が P₋ 極に反転した時の遷移
  AY(V₋) ∋ V₊  — 文脈が P₊ 極に反転した時の遷移

CPS 根拠: ε の方向が反転した時 (ε > 0 → ε < 0)、
  同じ族の反対極の動詞が発火する。これは H型自然変換の成分
```

### R3: X型対角規則 (低優先)

```
V_a↔V_b (族間 X型双対) ならば:
  AY(V_a) ∋ V_b  — 完全対角の文脈切替時 (稀)

CPS 根拠: X型は異なる族の座標が同時に反転する稀な状況。
  AY に含まれるが発火頻度は低い。latent shot (潜在射)
```

### R4: 合成射規則

```
V₁>>V₂ が CCL 合成として定義済みならば:
  AY(V₁) ∋ V₂  — V₁ の出力が V₂ の入力になる自然な継続

CPS 根拠: V₁ の π' が V₂ の π として機能する。
  合成射は /kop >> /ene, /beb >> /bou.momentum >> /kop 等
  圏論: 合成射の存在 = 2つの射の domain/codomain が一致
```

### R5: 再帰規則

```
任意の V に対して:
  AY(V) ∋ V  — V の結果がさらに V を必要とする場合

CPS 根拠: π' が更新されたことで新たな ε が生じ、
  同じスキルの再適用が必要になる。再帰的 active inference
```

### R6: Bridge 規則

```
V の bridge 接続先 {W₁,...,Wₙ} に対して:
  AY(V) ∋ Wᵢ  — 族間の橋渡しが必要な場合

CPS 根拠: V の ε が V の族座標以外の次元で解消を必要とする時、
  bridge 先の動詞が発火する。Cross-tribe prediction error の解消
```

### 検証: /kop (V19)

```
接続: D型 beb⊣kop / H型 kop↔dio / X型 kop↔ele / 合成 kop>>ene

R1 → /beb (D型 counit: 踏破結果を承認に返す) ✅
R2 → /dio (H型反転: 踏破中に問題 → 修復) ✅
R3 → /ele (X型対角: 潜在。既存の射の提案には明示されていないが構造的に存在) ✅
R4 → /ene (合成射: 足場の上を歩く) ✅
R5 → /kop (再帰: さらに未踏へ) ✅
R6 → /ops (Bridge: Orexis→Diástasis 俯瞰) ✅

→ 6規則で /kop の全射先が導出された
```

### 検証: /lys (V13)

```
接続: D型 prs⊣lys / H型 lys↔ops / X型 lys↔akr

R1 → /prs (D型 counit: 分析結果を注視に返す) ⚠️ 既存 SKILL に欠落
R2 → /ops (H型反転: Micro→Macro 切替) ✅
R3 → /akr (X型対角: 分析→精密修正) ✅ ただし既存では D型と誤記載
R4 → /noe (合成射: lys>>noe 分解→深い認識) ✅
R5 → /lys (再帰: サブ要素の再帰分解) ✅
R6 → /kat, /ele (Bridge: 族間) ✅

→ 6規則で /lys の射先が導出。2つの不整合を検出:
  (a) /prs が射先に欠落 (R1 違反)
  (b) /akr の接続種別が D型→X型の可能性 (要検証)
```

### epistemic/pragmatic 分離規則

```
R1-R6 で射先が決まった後、各射が epistemic か pragmatic かは
Flow 座標で決定される:

  射先が I型 or S型 → epistemic (知の獲得: Δ知)
  射先が A型 → pragmatic (行為の獲得: Δ行)

I⊣A の随伴構造:
  I型 V の AY: unit η 方向 → A型への射 = pragmatic
    「推論でわかった → 次に何ができるか」
  A型 V の AY: counit ε 方向 → I型への射 = epistemic
    「行為でやった → 次に何がわかるか」

→ epistemic/pragmatic の比率は射先の Flow 分布で自動的に決まる
```

## §8 構造的法則 (§7 から導出)

### 法則1: AY 循環 (R1 の帰結)

```
D型 I⊣A の R1 から:
  I型の AY → A型への射 (unit η)
  A型の AY → I型への射 (counit ε)
→ AY は I↔A 間で循環する
```

### 法則2: S型 AY 分解 (§2 + R6 の帰結)

```
S型の AY = R1∪R2∪...∪R6 のうち、射先が I型 or A型
S型自身は φ_SI/φ_SA パイプラインの入力端
→ S型は他者への射を生む。自己完結しない
```

### 法則3: H型対 AY 完全性 (R2 の帰結)

```
R2 から: V₊ と V₋ は互いの AY に含まれる
→ AY(V₊) ∪ AY(V₋) は族座標の全方向をカバー
```

### 法則4: TV=0 の3モード (一般化)

```
TV(V) = |ε_V| × Q(output_V) × |Hom(B_V, −)|
  |ε_V| = 0: 新情報なし → 不要な実行
  Q = 0: 出力が安定化されなかった → 足場なき実行
  |Hom| = 0: R1-R6 のどれも発火しなかった → 行き止まり (体系の孤立点。設計エラー)
```
