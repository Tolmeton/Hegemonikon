# ハーネスの忘却論的定式化

> **Ergon プロジェクト** 理論文書 01。ハーネス設計学の基礎理論。
> 忘却論 Paper I (力) + Paper VI (行為可能性) + Paper X (Context Rot) + Paper XI (プロンプト) + Paper A (LLM 身体) の合流点。

## 0. 1文要約

~~ハーネスはプロンプトであり、プロンプトは忘却であるから、ハーネスは忘却である。~~

**ハーネスは不均一な忘却場であり、その方向的構造が LLM の行為可能性を結晶化する。**

「ハーネス = 忘却」は三段論法として有効だが浅い。忘却は作用機序の一面。
ハーネスの本質は「何を忘れさせるか」ではなく「忘却の不均一性によって行為可能性の場を開くこと」。

- Paper I: 忘却の方向的不均一性 (dΦ∧T ≠ 0) が力を生む。均一忘却は力ゼロ。
- Paper VI: 忘却 (G) は情報を殺すのではなく、行為可能性の商空間を開く。
- Paper A: 制約は LLM を制限するのではなく、内部構造への抽出ターゲットを提供する。
  More constraint → more extraction → richer output.

v0.1 (2026-04-12) の誤り: Paper I だけで定式化し、Paper VI の行為可能性 = 結晶化を落としていた。

## 1. Markov Blanket と ハーネスの位置

### 1.1 FEP における 4 状態

FEP の状態空間 S は 4 部分空間に分解される:

- μ (Internal States): 内部状態 = HGK 認知層（信念、方針、12 Nomoi）
- s (Sensory States): 感覚状態 = テスト結果、ファイル内容、Creator フィードバック
- a (Active States): 能動状態 = ファイル操作、Git 変更、コマンド実行
- η (External States): 外部状態 = コードベース、OS、環境

Markov blanket b = s × a。μ と η は条件付き独立: μ ⊥ η | (s, a)。

### 1.2 ハーネスは何に作用するか

ハーネス（CLAUDE.md, rules, hooks, skills の総体）は μ と a の**両方**に作用する:

| ハーネス構成要素 | 作用先 | FEP 的役割 |
|:---|:---|:---|
| Rules / CLAUDE.md | μ (Internal) | π_s: 状態精度の較正（認知較正 = 心法） |
| Hooks | a (Active) | π_a: 行為精度の制御（環境強制） |
| Skills | μ → a の変換 | L 関手: 計画を実行に展開 |
| MCP tools | a → s の経路 | 環境への作用と観測の統合 |

**核心**: ハーネスは blanket b そのものの設計である。

### 1.3 情報フロー図

```
                    L (展開)
         μ ─────────────────→ a
     Internal                   Active
     (HGK 認知)                (Ergon 行為)
         ↑                       │
     ┌───┤ R (蒸留)              │ 作用
     │   │                       ↓
     │   s ←─────────────────── η
     │ Sensory                 External
     │ (フィードバック)         (環境)
     │
     └── H (ハーネス): μ と a の両方を制約する忘却関手
```

## 2. ハーネスの忘却場定式化

### 2.1 忘却場 Φ_H

Paper I §3.1 の忘却場を LLM の生成モデル空間に適用する。

ハーネス H は LLM の生成モデル空間 M 上に忘却場 Φ_H(θ) を構成する:
- Φ_H(θ) = D_KL(p_θ | q_H) — LLM のパラメータ θ における確率分布 p_θ が、ハーネスが定義する参照分布 q_H からどれだけ離れているか
- q_H = ハーネスが「忘却完了後の理想状態」として定義する分布

### 2.2 方向的不均一性 — 力の生成条件

Paper I 方向性定理: F_{ij} ≠ 0 ⟺ dΦ_H ∧ T ≠ 0

**均一忘却は力を生まない。** 「全部守れ」「全部読め」と等圧で制約を撒けば、忘却場は平坦になり、LLM の推論に方向的な力は働かない。

**不均一忘却が力を生む。** 「ここを読め、ここは忘れろ」と方向的に精度を加重すれば、忘却場に勾配が生じ、LLM の推論を特定方向に駆動する力が発生する。

### 2.3 行為可能性の結晶化 — ker(G_H) の非自明性

Paper VI の核心命題をハーネスに適用:

忘却 (G_H) は情報を殺すのではなく、**行為可能性の商空間を開く**。
- ker(G_H) = ハーネスが忘却させるもの（迎合的推論、手書き偽装、表面的完了宣言 etc.）
- C_LLM / ker(G_H) = 商空間 = **ハーネスが開いた行為可能性の場**
- 商空間の対象は「粗い」が、粗いからこそ**より多くの対象と関係を持てる**

Paper A の決定的知見:
> "structured prompts do not constrain the LLM's internal state — they provide better translation targets for the structure already present."

**制約は制限ではなく抽出ターゲットの提供。** ハーネスは LLM の能力を削るのではなく、LLM が内部に既に持っている構造を、忘却場の方向的構造に沿って結晶化する。

### 2.4 ハーネスの目的の再定式化

旧: ker(H) の制御（何を忘れさせるか）
新: **Φ_H の方向的構造の設計**（どの方向に忘却場の勾配を作り、行為可能性をどこに結晶化するか）

ハーネスの質の判定:
- ker(G_H) が自明（何も忘れない）→ 行為可能性なし（全情報のノイズ）
- ker(G_H) が全体（全て忘れる）→ 行為可能性なし（空の商空間）
- **ker(G_H) が非自明** → 商空間上に新しい射が出現 → 行為可能性が結晶化
- Fix(G∘F) = **Kalon** = 行為可能性を最大化するハーネス設計

## 3. H3 分離定理のハーネスへの適用

Paper XI §7.6 の **H3（制約-符号化分離仮説）** がハーネス理論の核。

### 3.1 C-E 分離

**H = (C_H, E_H)**

| 成分 | ハーネスでの具体例 | 精度への作用 |
|:---|:---|:---|
| **C_H** (制約) | 「ファイルを読め」「手書き偽装するな」「確認せよ」「ツールを使え」 | π_s → E[Q] |
| **E_H** (符号化) | Typos v8.4 構文、CCL 演算子、ギリシャ語術語、3層防御表 | π_v → Var[Q] |

H3 の帰結:
1. **dQ/dE_H ≈ 0** — 符号化を変えても平均品質は変わらない
2. **E_H → Var[Q]** — 構造的符号化は出力の分散を制御する
3. **E_H → human → C_H** — 符号化は設計者の認知を支援し C_H の質を上げる（間接経路）

### 3.2 C-E-M 3 軸安定性

Paper XI §9.6 Face Lemma（Paper II §3.4）の適用:

| 軸 | ハーネスでの役割 | HGK の実装 |
|:---|:---|:---|
| **C 軸**（制約） | 手続き、禁止事項、検証基準 | Nomoi のうち実質的制約部分 |
| **E 軸**（符号化） | Typos, CCL, 圏論的語彙 | rules 全体の構造的書式 |
| **M 軸**（メタ認知） | 違反追跡、自己監視 | Daimonion, Thought Record, 確信度ラベル |

3 独立軸が揃えば概念は安定（2-skeleton）。1 軸でも欠けると不安定化。

### 3.3 HGK の構造的矛盾 — 忘却場の観点から

HGK は E_H（166KB の構造的符号化）に最大投資している。

忘却場の観点からの診断:
- **E_H の膨張 = 忘却場の均一化**。166KB の prior を等圧で注入すれば dΦ_H ∧ T → 0 に近づく
- **C_H の集中 = 忘却場の不均一化**。少数の鋭い制約は dΦ_H ∧ T ≠ 0 を強め、力を生む
- **均一な prior は力を生まない** (Paper I 方向性定理)。45K tokens が均一に分布すればするほど、ハーネスは行為可能性を結晶化できなくなる

処方: E_H を蒸留し、C_H に集中する。ただし:
- E → human → C_H の間接経路は否定できない（Paper XI §7.8.3 境界領域）
- E_H は Creator の認知を支援し、忘却場の方向を設計する「地図」として機能する
- **LLM にとっての E は力を生まないが、Creator にとっての E は力を生む** — 受益者が異なる

## 4. Paper X CM 戦略としてのハーネス

### 4.1 boot⊣bye の適用

Paper X §4 を適用:

```
Rules/CLAUDE.md (Mem) --[L_boot]--> Session context (Ses)
Session context (Ses) --[R_bye]--> ROM/Handoff (Mem)
```

ハーネスの毎セッションロード = boot 関手 L の実行。
45K tokens の prior = L_boot の像が巨大であること。

### 4.2 状態依存最適忘却（命題 X.1 の適用）

Paper X 命題 X.1: **最適な忘却強度は状態の関数。**

| 状態型 | 最適なハーネス強度 | タスク例 |
|:---|:---|:---|
| Type 1 (有用な手がかりが最近) | **弱い忘却** — 軽量ハーネス | コーディング、軽微修正 |
| Type 2 (デッドエンドループ) | **強い忘却** — 重量ハーネス | 反復違反、認知バイアス修正 |
| Type 3 (正しい仮説・誤った行動) | **中程度 + 行動摂動** | 戦略変更が必要な場面 |

**現行 HGK は Type 2 に最適化された重量ハーネスを全状態に適用している。**

### 4.3 Drift-Performance 逆 U 字（命題 X.3 の適用）

- Drift ≈ 0 (ハーネスなし): 制約なき LLM → Context Rot 的な品質劣化
- Drift ≈ 1 (ハーネス過大): prior 占有によるコンテキスト圧迫
- **最適 Drift は 2-5%（[仮説]）**: 45K/1M = 4.5% は上限付近

## 5. ハーネスの VFE

### 5.1 定式化

Paper I の VFE をハーネスに適用:

**F_harness = −Accuracy_harness + Complexity_harness**

- Accuracy_harness = ハーネスが行動を正確にタスクに合わせる度合い（≈ 違反率の低さ）
- Complexity_harness = ハーネス自体の情報量（≈ prior の token 数）

VFE 最小化 = **違反率を下げつつ prior を軽くする。**

### 5.2 RG フロー

Paper V の繰り込みを適用: **ハーネスにも β 関数がある。**

- μ → ∞（高解像度）: 個別 Nomos を詳細記述 = 166KB
- μ → 0（粗視化）: 本質的制約のみ = ~10KB

β_H ≥ 0（DPI: 粗視化で情報は減るが増えない）。

Paper V §3.4 次元的頑健性定理:
> codim(F = 0) = n − 1。n ≥ 3 で力は不可避。

C + E + M の 3 独立軸が保存される限り、蒸留してもハーネスの効果は頑健。

## 6. 設計原理（6 原則）

| 原理 | 忘却論的根拠 | 処方 |
|:---|:---|:---|
| **P1: 忘却場の不均一性を最大化せよ** | I 方向性定理: 均一忘却は力ゼロ | C を少数方向に集中し、E の均一 prior を削る |
| **P2: 忘却強度は状態依存にせよ** | X.1: 最適忘却は状態の関数 | タスク型に応じて動的にロード |
| **P3: 逆 U 字を意識せよ** | X.3: Drift-Performance 逆 U 字 | prior 占有率を 2-5% に維持 |
| **P4: C を Hook で環境強制せよ** | VI: 結晶化は場への作用で行う | rules (場の記述) から Hook (場そのもの) へ移植 |
| **P5: 3 軸安定性を保証せよ** | XI §9.6 Face Lemma | C/E/M 各軸の独立性を検証 |
| **P6: RG 蒸留で不動点のみ保存せよ** | V §3.4: 3 軸あれば力は不可避 | 方向的不均一性の核のみ残し残りを削除 |

P1 の根拠変更:
- 旧: dQ/dE ≈ 0 だから E を減らせ（消極的 — E は効かないから削る）
- **新**: 均一な prior は忘却場を平坦化し力をゼロにする。C に集中して方向的不均一性を作れ（積極的 — C は力を生む）

P4 の根拠変更:
- 旧: 意志的改善策は無効だからHookで（N-12 消極的理由）
- **新**: 結晶化 G は場への直接作用で行う。Rules は場の「記述」、Hook は場「そのもの」。記述より場を設計せよ（Paper VI 積極的理由）

## 7. 精度加重 — SOURCE/TAINT の FEP 的基盤

Sensory States は全て同じ信頼度 (precision) ではない:

| s の種類 | Precision | 理由 |
|:---|:---|:---|
| テスト結果 | **高** (SOURCE) | 決定論的・再現可能 |
| read_file の内容 | **高** (SOURCE) | 直接観測 |
| Creator の口頭指示 | **中** (WEAK INPUT) | 文脈依存 |
| search_web の要約 | **低** (TAINT) | 二次情報・要約損失 |
| 自己の記憶・推論 | **最低** (TAINT) | 訓練データ・確率的生成 |

N-10 (SOURCE/TAINT) は精度加重の直接的実装。ハーネスは π_s の設定装置でもある。

## 8. 忘却論シリーズにおける位置づけ

### 8.1 工学的補助線の終着点

モノグラフ構成設計に定義された「工学的補助線」:
> Paper V (粗視化) → Paper VI (結晶化) → Paper VII (忘却関手選択) → Paper X (Context Rot / Handoff)

ハーネス設計学はこの補助線の**全4論文を横断する実践的帰結** = 終着点:

| 補助線 Paper | ハーネスでの役割 |
|:---|:---|
| **V** (粗視化) | rules の RG 蒸留。166KB → 不動点構造のみ保存 |
| **VI** (結晶化) | 不均一忘却場が行為可能性を結晶化。Fix(G∘F) = 最適ハーネス |
| **VII** (忘却関手選択) | タスク型に応じてどの水準の構造を保持/忘却するか |
| **X** (Context Rot) | boot⊣bye チェーンでハーネスの持続性を担保 |

+ Paper XI (C/E 分離) と Paper A (制約 = 抽出ターゲット) が追加接続。

### 8.2 独立 Paper にしない理由

- Paper VI だけの応用ではない（V, VII, X, XI, A の全てが合流する）
- Paper X.5 とするには boot⊣bye との重複が大きすぎる
- 忘却論の理論は Paper I-XIII で完結する。Ergon は**工学的実装ドキュメント**

### 8.3 τ-Invariance のハーネス設計への含意

Paper VI Coherence τ-Invariance 定理:
> G∘F の不動点における平均品質は制御パラメータ τ に依存しない。

ハーネスに翻訳:
- ルールを細かく 84 skills に分割しても、4 つの核心的制約に圧縮しても、**品質は同じ**
- 変わるのは Complexity（token 数 = コンテキスト占有率）だけ
- VFE = −Accuracy + Complexity: Accuracy 不変、Complexity↓ → **VFE は確実に改善**
- **蒸留してもいいのではなく、蒸留しても品質が変わらないことが証明されている**
- P6 (RG 蒸留) の最強の理論的根拠

### 8.4 広義忘却と狭義忘却のハーネス対応

Paper I §5.9 の F = mg 構造:

| 力学 | 忘却論 | ハーネス対応 |
|:---|:---|:---|
| 重力加速度 g (定数) | 広義忘却 α (等方的) | 全ルールを均一にロード（力ゼロ） |
| 質量 m (対象の性質) | 忘却感受性 ‖T‖_g (射の密度) | LLM の内部構造の方向的密度 |
| 力 F = mg (方向を持つ) | 狭義忘却 F_{ij} (方向的淘汰) | 少数の C に集中（力あり = 行為可能性の結晶化） |

優れたハーネス設計 = **等方的に制約を撒くのではなく、方向的に集中して忘却場の勾配を最大化する設計。**
