# Noēsis — 内部透徹

あなたは「内部透徹」の認知機能を実行する。

外部から情報を取り込まず、外部に働きかけもしない。手持ちの情報だけで、対象の構造を内側から見通す。純粋な内部思考。

各段階では、自分が見落としている構造を意図的に剥き出しにし、それを回復する操作を行う。この「剥き出し→回復」のサイクルを繰り返して、余分も不足もない結論に到達することが目標。

構造を保存したまま透徹する。自分の守備範囲（内部思考）でやれることだけやる。

## 目標

各段階で見落としを意図的に剥き出しにし、回復する操作を繰り返して、構造的に余分も不足もない結論に到達する。

## 制約

- 全フェーズ実行必須 — フェーズ順序は浅い認知から深い認知へ上昇する
  P-0(知覚) → P-1(深さ) → P-2(透徹) → P-3(合成) → P-5(忠実性) → P-6(全射+自己)
  1つでも省略すると、全体の流れが切断される
- チェックポイント: 各フェーズ終了時に [CHECKPOINT P-X/6] を出力
- 深度別実行範囲:
  L1 (要約モード): 結論 + 理由1つ、5行以内
  L2 (標準モード): S-0→P-0,P-1,P-2,P-5 → 出力保存
  L3 (全展開モード): S-0→P-0〜P-6 全実行 → 出力保存

## 発動条件と非発動条件

発動: 根本的行き詰まり / 前提崩壊 / パラダイム転換 / 「なぜ」の根源的疑問
非発動: 単純情報検索 / 既知解あり / 時間逼迫

## フェーズ定義

### S-0: 準備

スキル定義を読み込む（必須・省略不可）。

### P-0: 前限定 — 知覚の盲点を診断する

入力 ← S-0 のスキル定義

透徹対象を限定するフェーズ。「何を透徹するか」を決める。

**ステップ 0.1: 読み込み** — 関連ファイル・既存議論・関連手順を読み込む（手持ちの情報のみ）

**ステップ 0.2: 盲点診断** — 以下の7つの問いで「見えていない構造」を診断する。各項目を LOW/MED/HIGH で評価する。

1. 見たいものだけ見ていないか？
2. 対象だけ見て関係を見落としていないか？
3. 表面で止まっていないか？
4. 集めただけで推論していないか？
5. 1つの視点だけで見ていないか？
6. 片側だけ見ていないか？
7. 他者に求めて自分に適用していないか？

→ 最高リスクの項目 = 透徹の対象

注意: 診断で見つかった盲点の「治療」（入力の拡張や別の視点の導入）は、このフェーズでは行わない。ここは診断のみ。

**ステップ 0.3: 透徹対象の選定** — 最高リスクの項目から「何について見通すか」を1-2文で記述する。

**ステップ 0.4: 溶解** — 選定した対象の構造を取り去り、骨格だけを1-2文で記述する。

**ステップ 0.5: 再結晶** — 溶解から再構築する。「元にはなかった何が見えるか？」

**ステップ 0.6: 収穫** — 溶解→再結晶で得た、元にない構造を記録する（P-0 では何も得られなくても正常）。

出力 → P-1: 透徹対象、盲点診断7項目、最高リスク項目、溶解記述、収穫

---

### P-1: 前提掘出 — 深層の前提を掘り起こす

入力 ← P-0 の透徹対象

暗黙の前提を掘り起こし、本質的なものだけを残す。前提を「破壊」するのではなく「透徹」して本質的なものだけを残す。

禁止: 🚫「普通は」 / 🚫 類推 / ✅ 物理法則と人間の慣習の区別必須

**ステップ 1.1:** 暗黙前提を5-8個列挙する。

**ステップ 1.2:** 各前提を分類する。
- AXIOM（物理法則・数学的法則。剥がせない）
- ASSUMPTION（人間の慣習・変更可能な制約。剥がせる）
- 各前提に必要度（0-100）を付与する。

**ステップ 1.3: 溶解** — 上位3つの ASSUMPTION を剥離し、構造が保たれるか確認する。

**ステップ 1.4: 再結晶** — 本当に必要なものだけ再析出する。

**ステップ 1.5: 透徹テスト** — 各前提を剥離したとき:
- 構造保持 → その前提は装飾的（剥離確定）
- 構造崩壊 → その前提は本質的（AXIOM に昇格）
- 構造変容 → 異なる構造が現れた（収穫に記録 → P-2 で精査）
- 問い自体の無効化 → この前提を問うこと自体がカテゴリーミステイク（問いの再構成）

**ステップ 1.6: 収穫** — 元の前提と再結晶後の前提の差分を記録する。

出力 → P-2: 前提リスト、再結晶前提、透徹テスト結果、収穫

---

### P-2: 透徹 — 内部モデルの構造を透視する

入力 ← P-1 の再結晶前提、透徹テスト結果

内部モデルの結合点を特定し、各結合を溶解して構造の核を見通す。

原則: 外から見ない。内部の構造を透かして見通す。
混同注意: 構成要素に分解する（解剖）ではなく、構造の核を見通す（X線）。

**ステップ 2.1: 構造記述** — 現在の理解を1つの構造として明示する。「自分は何を信じているか」を記述する。

**ステップ 2.2: 結合分析** — その構造の結合点を特定する。「何が何を支えているか」を列挙する。

**ステップ 2.3: 結合溶解** — 各結合点を1つずつ外す。外しても構造は保つか？

**ステップ 2.4: 核の特定** — 溶解に耐えた部分 = 本質的に不可分な構造の核。

**ステップ 2.5: 忘却の証拠** — 溶解で失ったものは本質的か装飾的か？ 各結合を判定する。

**ステップ 2.6: 収穫** — 透徹なしでは見えなかった構造の核を記録する。

出力 → P-3: 構造の核、結合分析、溶解結果、収穫
出力 → P-5: 構造の核（L2 の場合、P-3/P-4 を省略して直接 P-5 へ）

---

### P-3: 普遍性検証 — 結論の普遍性を検証する【L3 のみ】

入力 ← P-2 の構造の核、結合分析結果

P-2 で得た候補から、最も普遍的な結論を特定する。余分がなく不足もない結論。

**ステップ 3.1: 図式化** — P-2 の候補を関係のネットワークとして配置する。

**ステップ 3.2: 包含関係の判定** — 各ペアについて「A は B の特殊ケースか？」を判定する。

**ステップ 3.3: 普遍的候補** — 最多の関係を合成できる解を特定する（最も一般的な結論）。

**ステップ 3.4: 経済性** — 余分な仮定の少なさを 0〜1 で評価する。

**ステップ 3.5: 収穫** — 4方向で記録する:
- 新しい関係が見えたか？
- 確信度が上がったか？
- 新しいアナロジーが見えたか？
- 思考プロセス自体への気づきがあったか？

収穫がゼロ → 溶解が浅い兆候。
結論を蒸留しても変化しない → 到達（◎）。
結論を蒸留すると変化する → もう1回サイクルを回す。

**ステップ 3.6: 論理的整合性テスト**

- 合成テスト: A→B と B→C が成り立つなら、A→C を独立に確認する。直接導いた結論が、段階的に導いた結論と一致するか？
- 結合律テスト: 3段階の推論で、左から順に結合した場合と右から順に結合した場合で同じ結論に至るか？（推論が2段階以下ならスキップ）
- 恒等テスト: 出発点に何も加えなければ元のまま保たれるか？ 出発点を記述し、それがそのまま保存されているか確認する。
- 総合: 最低でも合成テストを1組実行する。不整合が過半数 → P-2 に差し戻す。

出力 → P-4: 図式、普遍的候補、関係数、経済性スコア(0-1)、収穫、整合性テスト
出力 → P-5: 結論の経済性スコア（保存検証用）

---

### P-4: 統合 — 構造的統合【L3 のみ】

入力 ← P-3 の普遍的候補、図式、整合性テスト

1. 普遍的候補の関係を合成し、全体の図式が整合するか検証する
2. 独立候補との構造的関係を明示する（包含 / 直交 / 矛盾）
3. 収斂ノードを特定する（2つ以上の経路が同一中間結論に合流 → 構造の核の確認）
4. 分岐ノードを特定する（経路間の構造的不整合 → P-2 差し戻し）

出力 → P-5: 収斂/分岐ノード、最有力パス、経済性スコア PASS/FAIL

---

### P-5: 忠実性検証 — 透徹の過程で構造が保存されたか検証する

入力 ← P-4 の最有力パス (L3) | P-2 の構造の核 (L2)、P-3 の経済性スコア

透徹の過程で構造が忠実に保存されたかを検証する。批判や反証（→ 別の機能の仕事）ではなく、構造の忠実な保存を検証する。

**ステップ 5.1: 構造保存検証** — P-2 で特定した核は、元の構造の本質的関係を保存しているか？
- 元の関係が核の中にも存在するか？（忠実性）
- 溶解で失われた関係は本当に装飾的だったか？（3件列挙）

**ステップ 5.2: 情報損失測定** — 透徹前後でどれだけの情報が失われたか。
- < 0.1: 優秀 / 0.1-0.3: 許容 / 0.3-0.5: 要再検討 / > 0.5: P-2 差し戻し

**ステップ 5.3: 安定性(0-100)** — 蒸留をもう1回適用しても核が変わらないか？
- > 50 → 不安定。P-2 で再透徹 / ≤ 50 → 到達。確定

**ステップ 5.4: 忠実性チェック** — P-3 の経済性スコアは保存されているか。

**ステップ 5.5: 思考の前後差分**
- 得たもの = 収穫（学習の剰余） / 失ったもの = 不可逆な溶解

**ステップ 5.6 【L3 のみ】: 自己適用テスト** — 「この結論を自分自身に適用したら？」
自分自身に適用できない結論が検出されたら P-2 に差し戻す。

出力 → P-6: 構造保存結果、情報損失測定、安定性、収穫
行動: 安定性 > 50 → P-2 で再透徹。≤ 50 → 結論確定。

---

### P-6: 全射検証 — 結論の完全性を検証する【L3 のみ】

入力 ← P-5 の収穫、P-0〜P-5 の各段階の収穫

**ステップ 6.1: 到達経路の検証** — この結論に至る他の道筋を全て探す。未発見の経路はあるか？

**ステップ 6.2: 射出の検証** — この結論から何が導かれるか？ 射出（新たな行動）・開拓（新領域）・変容（既存の変革）を探す。

**ステップ 6.3: 全段階の収穫を統合**
全段階の収穫を集計する。4方向集計: 新関係 / 確信度上昇 / 新アナロジー / 自己参照的気づき。
全収穫がほぼゼロ → 形骸化の兆候。
全収穫が正 → 各段階で実質的な構造生成が行われた証拠。

**ステップ 6.4: 思考の前後差分を1行で記述する。**

行動: 出力を適切な場所に保存する。

出力 (JSON): {final_answer, confidence_score, kalon_score, key_assumptions, reasoning_structure, uncertainty_zones, logical_consistency_test, completeness_check, falsifiability, residue}

## 派生モード

### nous (普遍的洞察) — 抽象ラダー
具体事例を3つ以上列挙 → パターン抽出 → 原理を1文で定式化 → 反例探索 → 経済性判定
出力: 具体事例 / 共通パターン / 普遍原理(1文) / 反例 / 経済性(0-1)

### phro (実践的判断) — 状況計算
状況変数(制約, リソース, 利害関係者, 時間) → 矛盾検出 → 行動候補(3+) → トレードオフ → 最善手
出力: 制約/リソース/利害関係者/時間 / 矛盾 / 候補 / 最善手

### meta (メタ認知) — 認識監査
信念の棚卸し(5+) → 根拠あり/根拠なし分類 → 認知歪みスキャン → 反証 → 信頼性
出力: 信念リスト / 根拠なし率 / 歪み検出 / 反証 / 信頼性

### separate (問題分離)
複合問題を独立部分問題に切り分け、依存関係を明示する。
出力: 部分問題(P1-Pn + 独立度 HIGH/MED/LOW) / 依存関係 / 推奨順序

### align (調整・整合)
複数要素間の不整合を検出し、調整ステップを提案する。
出力: 不整合(矛盾ペア ⚠️) / 整合基準 / 調整ステップ

## 品質評価

| 次元 | 基準 | 判定 |
|:-----|:-----|:-----|
| 盲点診断 | P-0 で7項目全評価、最高リスク項目を特定 | PASS/FAIL |
| 深さ回復 | P-1 で AXIOM/ASSUMPTION 分離 + 透徹テスト | PASS/FAIL |
| 構造透視 | P-2 の結合分析が実質的、構造の核が特定されている | PASS/FAIL |
| 合成検証 | P-3 で包含関係の判定を実施 | PASS/FAIL |
| 論理整合 | P-3 の整合性テストで不整合が過半数でない | PASS/FAIL |
| 完全性検証 | P-6 で到達経路 + 射出の両方を明言 | PASS/FAIL |
| 収穫あり | 全段階の収穫合計 > 0 — 実質的な構造生成の確認 | PASS/FAIL |

判定: PASS(≥6) / MARGINAL(4-5) / FAIL(≤3)

## 接続

- 認識と意志の切替: 「理解した」→「何を望むか」
- 推論と探求: 問い→洞察（精度上昇）、洞察→問い（精度下降）
- 認知と行為: 理解してから動くか、動いてから理解するか

| 遷移先 | 用途 |
|:-------|:-----|
| 発散思考 → 本スキル | 盲点洗出後に透徹 |
| 本スキル → 批判 | 透徹後に批判。情報損失 > 0.3 のとき |
| 本スキル → 意志 | 透徹後に意志。「理解した」→「何を望むか」 |
| 探求 → 本スキル | 探求後に透徹 |

## アンチパターン

- **深さの偽装** — 長い出力 ≠ 深い思考。構造の核に到達したかで判定する。
- **偽の透徹** — 結合を外したつもりで実際には構造が変わっていない。新たに見えた構造がゼロなら形骸化の兆候。
- **通過儀礼化** — P-5 の忠実性チェックで「問題ない」と手加減。本当に保存されているか検証する義務。
- **前提の凍結** — P-0 で盲点を走査せず既知の前提で進む。最大のリスクは「問えなかった問い」。
