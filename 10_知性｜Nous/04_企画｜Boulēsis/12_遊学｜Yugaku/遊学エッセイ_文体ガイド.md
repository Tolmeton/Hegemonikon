## §0　生成原理

このガイドの規則は **VFE 最小化** から導出される。

> **読者の VFE を最小化せよ。**
>
> $$F_{\text{style}} = -\text{Accuracy}_{\text{読者}} + \text{Complexity}_{\text{読者}}$$

| VFE 項 | 最適化方向 | 導出されるスタイル規則 |
|:---|:---|:---|
| **Accuracy ↑** | 読者の予測精度を上げる | 数式で裏付ける (§4) / 反論を先に取り込む (§6) / 結論を先に出す (§1) / 参考文献を示す (§8) |
| **Complexity ↓** | 読者の認知コストを下げる | 短文 (§2) / だ・である調 (§2) / 感情語排除 (§2) / 比較テーブル (§4) / 「直感的に言えば」(§3) |
| **Prior precision ↓** | 読者の固着を緩める | 挑発的文体 (§2) — 「バカ」と書くことで読者の prior を揺さぶり、posterior の更新を可能にする前処理 |
| **構造認識** | 最小の閉構造で伝達 | メタファー三連 (§3) — 三角形 (2-simplex) は圏における最小の閉じた構造。3点で初めてパターンが浮かぶ |

以降の §1-§9 は、この関手の**像**（具体的な射の列挙）である。表記慣習（脚注記号、§ 形式、人称）のみが VFE 外の約束事である。

## §1　全体構造

### タイトル
- **主題**: 挑発的かつ命令的な命題（例: 「バカをやめたいなら構造を見ろ」）
- **副題**: 主題の数学的・構造的裏付けを予告する（例: 「— なぜ構造を見る者は要素を見る者に勝つのか」）
- 副題は em ダッシュ（—）で始める

### セクション構成
- `## §N　タイトル` の形式。全角スペースで § と番号を区切る
- サブセクションは `### N.M`
- セクション間に `---`（水平線）を挿入
- 全体は以下の弁証法的構造に従う:

| 位置 | 役割 | 例 |
|:---|:---|:---|
| §1 | **挑発 + 結論先行** | 結論3点を箇条書きで先出し |
| §2-§3 | **基礎概念の導入** | VFE → GIGO → 二段階 |
| §4-§5 | **数学的証明** | 米田 → 関手 → Smithe |
| §6 | **相補性・文脈化** | 他の推論形式との関係 |
| §7 | **反論の取り込み** | 偽アナロジーの診断 |
| §8 | **結語** | 知見の要約 + 主観的提言 |

### 結論先行の原則
- §1 で結論を**箇条書き3点**で先出しする
- 「結論を先に述べれば ——」のフレーズで導入
- 「これらの主張を、以降の節で順に証明する。」で締める

## §2　文体・トーン

### 基本文体
- **だ・である調**（敬体は使わない）
- 簡潔。一文は原則40字以内を目指す。長くなる場合は em ダッシュ（—）で分割
- 感情語禁止。「素晴らしい」「興味深い」は使わない
- 断定する。ただし主観は明示する（「筆者の完全な主観ではあるが」）

### 挑発と厳密さの共存
- 挑発的な命題（「勤勉なバカは怠惰な天才に負ける」）を**数学的定義で裏付ける**
- 挑発の直後に「これは挑発ではない。本稿で数学的に示すのは〜」と宣言する
- メタファーは**必ず数学的対応を持つ**。飾りとしてのメタファーは使わない

### 主語の選択
- 著者 → 「筆者」（「私」は使わない。ただし脚注内では許容）
- 読者 → 「読者」（直接呼びかけない。「あなた」は禁止）
- 概念の主体 → 「天才は〜」「バカは〜」（人格化した概念で語る）

## §3　修辞技法

### 定義ブロック
- 重要な定義は**引用ブロック（>）で囲む**
- 太字で核心語を強調

```markdown
> **バカ** = VFE が高い状態。
> すなわち、複雑なモデルを抱えているのに的を外し続けている認知状態。
```

### メタファーの三連構造
- メタファーは原則**3つ組**で提示する
- 異なる領域から取り、構造の普遍性を示す

```markdown
- 間違った地図を**自力で**精密に描き直しても、その地図は正確にはならない
- 砂上**に**楼閣を増築しても、基盤の脆弱性は変わらない
- **路傍の石**をいくら丹念に磨いても、宝石にはならない
```

### 対比構造
- 「X は〜。Y は〜。」の並置で概念の差を際立たせる

```markdown
勤勉なバカは「対象」を集める — 事実を暗記し、データを蓄積する。
天才は「射」を見る — 事実間の関係を捉え、構造を発見する。
```

### 「直感的に言えば」パターン
- 数式や厳密な議論の直後に、一文の平易な言い換えを添える
- 「直感的に言えば:」で導入

```markdown
直感的に言えば: 暗記は事実を1個ずつ買うが、アナロジーは構造を**まとめ買い**する。
```

### 具体例の導入
- 「〜を考えてみよう。」または「〜を考えよう。」で始める
- 具体例は**誰でも知っているもの**を選ぶ（E=mc², りんご, 太陽系）

## §4　数式と技術用語

### 数式の扱い
- LaTeX `$$...$$` で独立行。インライン数式は `$...$`
- 数式の直後に**各項の日本語説明**を箇条書きで置く

```markdown
$$F = -\underbrace{...}_{\text{Accuracy}} + \underbrace{...}_{\text{Complexity}}$$

- **Accuracy**: モデルの予測がどれだけ現実に合っているか（高いほど良い）
- **Complexity**: モデルがどれだけ複雑か（低いほど良い）
```

### 技術用語の導入
- 初出時に**太字 + 英語原語 + 略語**を括弧で併記
- 以降は日本語名または略語のみ

```markdown
**変分自由エネルギー** (Variational Free Energy; VFE)
```

### 比較テーブル
- 思考法や概念の比較には**テーブル**を使う
- 列は「対象」「コスト」「効果」「効率」のような対称構造

## §5　脚注体系

脚注は em ダッシュや括弧とは異なる機能を持つ。**本文の流れを止めずに、別の知的層を開く**。

| 記号 | 用途 | 例 |
|:---|:---|:---|
| `※` | **自己言及・メタ観察** | 「この説明自体がアナロジーである」 |
| `†` | **前方参照** | 「この概念は §4 で厳密に定義する」 |
| `‡` | **認識論的留保** | 「ゴミには二種類ある。本稿は前者を扱う」 |
| `§` | **技術的補足** | 忠実/充満関手の追加説明 |
| `‖` | **技術的但し書き** | 「ベイズ的帰納では〜」 |
| `※※` | **数学的根拠で結語を補強** | prior precision が∞のとき |

### 脚注の配置
- 本文中に `<sup>記号</sup>` でマーカーを挿入
- セクション末に脚注テキストをまとめて配置
- `> <sup>記号</sup>` の形式で引用ブロック内、または `<sup>記号</sup>` で直接記述

### 自己言及的脚注
- エッセイの主張がエッセイ自身で実証されている箇所では、※ で指摘する
- これは読者への知的サービスであり、著者の自己満足ではない

## §6　反論の構造的取り込み

### 先回り型
- 読者が持つであろう反論を**著者が先に述べる**
- 「ここで当然の反論が予想される:」で導入

### 反論を武器にする
- 反論を否定するのではなく、**反論が本稿の主張を強化することを示す**
- 「これは〜を否定するのではなく、〜の重要性を再確認するものだ」

### 限界の明示
- 結論の射程を明示する。「本稿は X を示した。Y は対象外である」
- 限界の明示は弱さではなく、知的誠実さ

## §7　結語のパターン

### 三段構成
1. **非情な事実**: 「現実は非情である」から始める
2. **しかし希望**: 「だが、〜方法がないわけではない」
3. **再帰的閉鎖**: エッセイの冒頭メタファーを回収して締める

### 著者の主観の明示
- 結語でのみ「筆者の完全な主観ではあるが」と前置きして意見を述べる
- 本文中では数学的に示し、結語でのみ主観を許す

### 最終文
- 冒頭のメタファーを**変形して**回収する構造
- 最後の一文は短く、余韻を残す

## §9　禁止事項

| 禁止 | 理由 |
|:---|:---|
| 「〜と思います」「〜でしょうか」 | 敬体。だ・である調に統一 |
| 「興味深い」「素晴らしい」 | 感情語。事実と数学で語れ |
| 「読者の皆さん」「あなた」 | 直接呼びかけ。距離感を保て |
| 飾りのメタファー | 数学的対応のないメタファーは禁止 |
| 「〜について述べる」で終わる段落 | 何を述べたか言え。予告だけで終わるな |
| 箇条書きだけのセクション | 散文での展開が先。箇条書きは要約用 |

## §10　概要 (Abstract) のパターン

### 概要の4類型 (MECE)

有名論文の概要から抽出した4つの構造パターン。全ての概要はこのいずれか（または合成）に分類される。

| 型 | パターン | 構造 | 代表論文 |
|:---|:---|:---|:---|
| **Type α** | 「X は実は Y」 | テーゼ → メカニズム → 帰結 → 哲学的含意 | Verlinde (2011), Jacobson (1995) |
| **Type β** | 「古典では X、しかし Y」 | 常識 → 「However...」→ 定量的帰結 | Hawking (1975) |
| **Type γ** | 「問題の再定義」 | 背景 → 根本問題の再公理化 → スコープ限定 | Shannon (1948) |
| **Type δ** | 「統一」 | 既存理論群 → 共通原理 → 統一フレームワーク | Friston (2010) |

### 各型の詳細

#### Type α: 「X は実は Y」（同一性の発見）

最も破壊力が高い。「X」と「Y」が概念的に遠いほどインパクトが大きい。

> **Verlinde (2011)**: *Starting from first principles and general assumptions we present a heuristic argument that shows that Newton's law of gravitation naturally arises in a theory in which space emerges through a holographic scenario. **Gravity is identified with an entropic force caused by changes in the information associated with the positions of material bodies.***

> **Jacobson (1995)**: *The Einstein equation is derived from the proportionality of entropy and horizon area together with the fundamental relation δQ=TdS. ... **Viewed in this way, the Einstein equation is an equation of state.** This perspective suggests that it may be no more appropriate to canonically quantize the Einstein equation than it would be to quantize the wave equation for sound in air.*

**構造**: 1文目でテーゼ → 2文目でメカニズムの骨格 → 最終文でパラダイムシフト

#### Type β: 「古典では X、しかし Y」（常識の反転）

反直感的な結論が核にあるとき最も効く。

> **Hawking (1975)**: *In the classical theory black holes can only absorb and not emit particles. **However it is shown that quantum mechanical effects cause black holes to create and emit particles as if they were hot bodies** with temperature T ∼ κ/2π.*

**構造**: 1文目で常識を確定 → 2文目で「However」→ 3文目で帰結

#### Type γ: 「問題の再定義」（公理の設定）

新分野を創設するとき、あるいは問題のフレームそのものを変えるとき。

> **Shannon (1948)**: *The fundamental problem of communication is that of reproducing at one point either exactly or approximately a message selected at another point. **The semantic aspects of communication are irrelevant to the engineering problem.***

**構造**: 問題の再定義 → 「関係ないもの」の排除 → スコープ限定

#### Type δ: 「統一」（包摂関係の発見）

既存の理論群を一つのフレームワークから見る。

> **Friston (2010)**: *Crucially, one key theme runs through each of these theories — optimization. Furthermore, if we look closely at what is optimized, the same quantity keeps emerging, namely value or its complement, surprise.*

**構造**: 既存理論の列挙 → 共通原理の抽出 → 「統一できる」と宣言

### 合成概要 = 全4型の合流

稀に、**全4型に同時に合致する**論文が存在する。この場合、単一型に制限するのは損失。

**忘却力の場合:**

| 型 | 忘却論での実現 |
|:---|:---|
| **α** | 「力は忘却の方向性から生まれる」— 力 = 忘却の同一性 |
| **β** | 「古典的には力は基本だが、実は忘却の曲率から創発される」— 常識の反転 |
| **γ** | 「力の問題を統計多様体上の忘却の問題に再定義する」— フレームの再設定 |
| **δ** | 「情報幾何 + FEP + ゲージ理論を忘却で統一する」— 統一 |

**合成概要の設計原則:**
1. 文1 = **α** (テーゼ)。「忘却が力を生む」で殴る
2. 文2 = **β** (反転)。「古典では力は先験的だが」で読者の prior を揺さぶる
3. 文3 = **γ** (再定義)。「統計多様体上の不均一性に再定式化する」で新しい枠組みを提示
4. 文4 = **δ** (統一)。「Yang-Mills と構造的に同型。FEP と自然対応する」で統一を示唆
5. 文5 = 予測。「測定可能な帰結」で着地

### 共通ルール（全型に適用）

| ルール | 説明 |
|:---|:---|
| **1文目で世界観が変わる** | 読者の prior を最初の1-2文で破壊せよ |
| **数式は最小限** | Hawking: 1式。Verlinde: 0式。Jacobson: δQ=TdS のみ |
| **「何をしたか」より「何が起きるか」** | 手法の詳細は本文に任せる |
| **最終文で視野を広げる** | Jacobson: 「量子化すべきでないかもしれない」 |
| **≤150語** | 6本中5本が150語以内。密度のみが正義 |
