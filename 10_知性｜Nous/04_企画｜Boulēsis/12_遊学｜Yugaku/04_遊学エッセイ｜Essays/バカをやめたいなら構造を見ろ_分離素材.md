## §9　自己言及 — 系のバカ度を測る構造的基準

### 9.1　自分を記述できない系は普遍的でない

§8 で「疑え」と述べた。しかし疑い方にも質がある。

構造を見ない者が「間違っているかもしれない」と漠然と疑っても、代替の系は見つからない。§5 で示した通り、系の外に出るためにはアナロジー（関手）が必要だからだ。では、系の外に出るべきかどうかを、系の**内側**から判定する方法はあるだろうか？

ある。**自己言及** (self-reference) である。

Lawvere の不動点定理 [13] は、ある条件を満たす系は必ず不動点を持つことを示す。その対偶を取ると ——

> **系が自分自身を記述できない（自己言及できない）ならば、その系は不動点を持てない。**

§4 で導入した圏論の言葉で言い換えれば: 系 C の射の空間 Hom(C, C) が十分に豊かでなければ、C の中に非自明な不動点は存在しない。不動点がなければ、§5 で述べた G∘F サイクル（発散と収束の反復）は収束先を持たない。つまり ——

> **自己言及できない系の中では、いくら推論を重ねても構造的な安定点に到達できない。**

これは §2 の GIGO を深化させた主張である。GIGO は「ゴミの入力からはゴミの出力」と述べたが、自己言及不可能性は「ゴミかどうかを**系の内部から**判定する方法すら存在しない」ことを意味する。

### 9.2　入れ子構造 — 系は系の中に系を持つ

系は入れ子構造を持つ。原子が分子を構成し、分子が細胞を構成し、細胞が生体を構成するように、小さな系は大きな系に包含される:

$$C_0 \hookrightarrow C_1 \hookrightarrow C_2 \hookrightarrow \cdots$$

数学における具体例を挙げよう:

- $C_0$ = **プレスバーガー算術** — 自然数の加法のみを扱う系。この系は**完備**であり**決定可能**である。すなわち、この系の内部で述べられる全ての命題は、この系の内部で真偽が決定できる。一見、完璧な系に見える
- $C_1$ = **ペアノ算術** — 自然数の加法と乗法を扱う系。$C_0$ を包含するが、乗法の追加によって系の表現力は飛躍的に増大する。そしてこの系は Gödel コーディングにより**自分自身を記述できる**
- $C_2$ = **ZFC 集合論** — 現代数学の基盤。$C_1$ を包含し、さらに豊かな構造を持つ

ここで決定的な事実がある:

$C_0$ は完備かつ決定可能という意味で「完璧」だが、**自己言及ができない**。$C_0$ の内部には $C_0$ 自身を記述する手段がない。従って、$C_0$ の限界は $C_0$ の内部からは原理的に見えない。

$C_1$ は不完全である — Gödel の不完全性定理が示す通り、$C_1$ には $C_1$ 内で証明も反証もできない命題が存在する。しかし $C_1$ は**自己言及ができる**。$C_1$ は自分の限界を知ることができる。

> **完備だが自分を見れない系と、不完全だが自分を見れる系。後者が「より賢い」。**

この対比は §2 の「バカ vs 天才」に直結する。$C_0$ のバカは、自分がバカであることを知る術がない — なぜなら $C_0$ は自分を記述できないからだ。$C_1$ の住人は、自分の系に限界があることを知ることができる — 不完全性定理がまさにそれを教える。

FEP の言葉で言えば: §8 の脚注で「prior の precision が無限大のとき信念は更新されない」と述べた。自己言及不能は、prior の precision を測定する手段すら持たないことに等しい。自分が疑うべきかどうかを判断する情報が、系の中に存在しないのだ。

### 9.3　上位置換 — バカをやめる構造的方法

§9.1–9.2 の議論は、バカをやめるためのもう一つの道を示す。

§8 は「既知を疑え」と述べた。これは正しいが、疑った先に何があるかは述べていなかった。自己言及の議論は、その先を教える:

> **自分の系がバカであることに気づいたら、自己言及可能な上位の系に移れ。**

これを**上位置換** (upper replacement) と呼ぶ<sup>◆</sup>。§5.3 で示した忠実関手の Complexity 削減は、系の**水平的**な移動（同じレベルの別の系へ）だった。上位置換は**垂直的**な移動（下位の系から上位の系へ）である。

水平移動 (アナロジー) が「別の宝石を持ってくる」なら、上位置換は「宝石と石を区別できる目を手に入れる」。前者は効率的 (§5)、後者は根本的。

上位置換が可能なのは、自分の系が自己言及不能であること——すなわち自分がバカであること——に**気づいた**ときだけだ。そして、気づくためには**疑い**が必要である。§8 の「疑え」はここに接続する: 疑いは発見の必要条件であると述べたが、より厳密には、疑いは上位置換の前提条件なのだ。

合わせると:

1. **疑う** — 自分の系を疑う (§8)
2. **自己言及を試みる** — 自分の系で自分の系を記述できるか問う (§9.1)
3. **記述できないなら** — 上位の系を探す (アナロジーで見つけるか、アブダクションで創るか) (§9.3 + §6)
4. **上位系に移る** — ここで米田の補題（§4）が再び効く: 上位系の射を見ることで、以前の系で見えなかった構造が見える

このプロセスは §6.2 の知覚-行動ループの垂直版——**認識の階段**——である。

> <sup>◆</sup> 圏論的には、$C_n$ 上のローカルな構造を $C_{n+1}$ に「最良近似」として持ち上げる操作は Kan 拡張に対応する。ただしこの数学的詳細は本稿の範囲を超える。

### 9.4　バカとは構造を忘れた者である

ここまでの議論を整理し、次へ向かおう。

§2 でバカを「VFE が高い状態」と定義した。§4 でバカが構造（射）を見ない者であることを示した。§5 でアナロジー（関手）が構造を効率的に転写することを示した。§7 で自然変換がアナロジーの品質を測ることを示した。

§9 はもう一つの軸を加える:

> **バカとは「構造を忘れた者」である。**

圏論には**忘却関手** (forgetful functor) U という概念がある。群の構造を忘れてただの集合にする、順序を忘れてただの要素にする —— これが忘却関手の仕事だ。

バカの認知は忘却関手として特徴づけられる。射（関係）を忘れて対象（要素）だけを見る。合成則を忘れて個別の事実だけを記憶する。そして最も深刻なのは —— **自分の系の自己言及構造を忘れる**ことだ。自己言及構造を忘れた系は、自分がバカであることすら知ることができない。

ここまでの議論を整理する:

1. **構造を見よ** (§4) — 要素ではなく射を見る
2. **構造を転写せよ** (§5) — アナロジーで新しい系を取り込む
3. **転写を評価せよ** (§7) — 自然変換でアナロジーの質を測る
4. **自分の系を自分の系で記述せよ** (§9) — 自己言及の可否が系のバカ度を教える
5. **記述できなければ、上に行け** (§9.3) — 上位置換で認知の階段を上る

バカをやめたいなら構造を見ろ。構造を見ている自分自身を見ろ。

—— しかし、これでもまだ不十分である。ここまでの議論には**非対称性**がある。

### 9.5　バカは構造を忘れ、賢者は対象を忘れる

§9.4 でバカを「構造を忘れた者」と定義した。では賢者は何も忘れないのか？ —— 違う。**賢者もまた忘れる。ただし、忘れるものが違う。**

圏論には n-圏の階層がある。最も基本的な階層は:

| 次元 | 要素 | バカの忘却 | 賢者の忘却 |
|:---|:---|:---|:---|
| n=0（対象） | もの自体、名前 | — | ❌ 意図的に忘れる |
| n=1（射） | 関係、構造 | ❌ 忘れてしまう | ✅ 保存する |

バカは n=1（射）を忘れる。群の乗法表を忘れてただの集合にする。因果を忘れて相関だけを見る。これが §9.4 で述べた忘却関手 $U_{arrow}$ だ。

賢者は逆に n=0（対象）を**意図的に**忘れる。個別の名前を捨て、構造だけを取り出す。集合 $\{a,b,c\}$ と $\{x,y,z\}$ は名前が違うが、「要素が3つの集合」として同型である。賢者はこの名前の差異を忘却し、**同型を除いて** (up to isomorphism) 考える。

これは §4 で述べた米田の補題の帰結そのものだ。Hom 関手は対象の「名前」を捨て、射の全体で対象を完全に再構成する。名前を捨てても何も失われない —— むしろ、名前を捨てることで初めて**本質的な構造**が見える。

CCL（認知制御言語）の合成演算子 `>>` はこの原理の操作化である。`/noe >> /ele` と書くとき、「認識」や「批判」という名前（n=0）より、両者の**接続構造**（n=1）が前面に出る。

> **バカの忘却は事故であり、賢者の忘却は技術である。**
>
> バカは射を落とす。賢者は対象を捨てる。

§9 で「バカとは構造を忘れた者」と述べた。では、忘れた構造は取り戻せるのか？ 取り戻したとして、元と同じものが返ってくるのか？ そしてこの「忘却と回復」の営みを組織的に行う系には、何か名前がつくのか？

## §10　忘れた構造は取り戻せる — 回復の非対称性

### 10.1　忘却関手には随伴がある

§9.4 でバカの正体を忘却関手 U として同定した。群の構造を忘れてただの集合にする、因果を忘れて相関だけを見る —— 忘却は構造の喪失である。

ここで圏論の深い定理が介入する。多くの忘却関手には**随伴** (adjoint) と呼ばれるパートナーが存在する [14]。忘却関手 U: Rich → Poor に対して、回復関手 N: Poor → Rich が存在し、U⊣N（U と N は随伴対をなす）。

直感的には: **忘れる操作があるところには、取り戻す操作がある**。

群の構造を忘れてただの集合にする忘却関手には、集合から自由群を構成する回復関手が随伴する。これは数学の定理であり、メタファーではない。

しかし —— そして、これが本節の核心だが —— 回復は「元に戻す」ことではない。

### 10.2　回復は元に戻すことではない

U⊣N 随伴の最も重要な性質は、次の不等式である:

$$N \circ U(x) \geq x \quad \text{かつ} \quad N \circ U(x) \neq x$$

忘れて (U) 取り戻す (N) と、**元以上のもの**が得られる。だが、**元と同じもの**は得られない。

§7.3 で述べた Ostwald 熟成を思い出してほしい。小さな結晶が溶解し、大きな結晶に再析出する —— 表面エネルギーの最小化が駆動力だった。認知においても同じことが起こる: 忘れた知識を学び直すと、忘れる前には見えなかった**周辺構造との接続**が生まれる。

この「忘却前にはなかった余剰」を**学習の剰余** (learning residue) $\rho$ と定義する:

$$\rho(x) := N \circ U(x) - x \geq 0$$

$\rho > 0$ は「回復により、元にはなかった新しい構造が付加された」ことを意味する。$\rho$ の非負性は、随伴の単位 $\eta: \text{Id} \Rightarrow N \circ U$（前順序圏における単調写像）から直接証明される<sup>♯</sup>。

具体的に見よう。

**精度の回復**: ファイル A の仕様を「たぶん X だったはず」と記憶で語る ($U_{precision}$: 精度の忘却)。確認すると実際は Y であり、しかも関連する仕様 Z も発見される ($N_{precision}$: 精度の回復)。$\rho$ = 確信度の上昇 + 新知識の獲得。検証という行為が、記憶より正確な知識を生んだ。

**射の回復**: ファイル A と B の依存関係 f を忘れる ($U_{arrow}$: 射の忘却)。コードを読み直すと f を再発見し、さらに A→C という別の依存関係 g も見つかる ($N_{arrow}$: 射の回復)。$\rho$ = 新しい射 g。忘却前には見えなかった関係の発見。

**自己の回復**: 他人に求める品質基準 S を持っているが、自分には適用しない ($U_{self}$: 自己関手の忘却)。S を自己適用すると、自分のコードに基準違反 D を検出し、S と D を統合した S' = S + D が得られる ($N_{self}$: 自己適用による回復)。$\rho$ = 自己認知の拡張。**これが最も非自明な剰余であり、最も価値が高い。**

> 忘却 (U) は一瞬で起こる。回復 (N) は労力を要する。
> しかし回復は元以上をもたらす ($\rho > 0$)。
> **生産的な忘却**が存在するのである。

<sup>♯</sup> 前順序圏において $\eta$ は $x \leq N(U(x))$ を満たす。すなわち $\rho = N(U(x)) - x \geq 0$。等号は $x$ が不動点 Fix$(N \circ U)$ に属するとき、かつそのときに限り成立する (§10.3)。

### 10.3　Kalon — 忘却と回復が釣り合う唯一の点

$\rho(x) = 0$ となる点 —— すなわち忘却と回復が完全に釣り合う点 —— は特別な性質を持つ。

$$\rho(x) = 0 \iff N \circ U(x) = x \iff x \in \text{Fix}(N \circ U)$$

これは**不動点** (fixed point) である。忘却して回復しても、何も変わらない点。これ以上忘れるものがなく、これ以上回復するものもない。

§9.1 で Lawvere の不動点定理に言及した。ここでその実体が見える: 系が自己言及可能であり (§9.1 の条件)、かつ忘却と回復の随伴 U⊣N を持つなら、不動点 Fix$(N \circ U)$ が存在する。

この不動点を、筆者は **Kalon** ($\kappa\alpha\lambda\acute{o}\nu$)——ギリシャ語で「美」—— と名づける<sup>♭</sup>。

- E = mc² は Kalon に近い。忘却して学び直しても、同じ方程式に戻る
- §7.3 の Ostwald 熟成が到達する安定結晶は、物理的な Kalon である
- §4 の米田の補題は、圏論の Kalon の一つである —— 忘れて学び直しても、変わらない

Kalon は**到達不能な理想**ではない。Fix$(N \circ U)$ は数学的に存在する。問題は、そこに至るまで U と N のサイクルを回し続ける意志があるか、である。

> <sup>♭</sup> 美を主観的感覚ではなく不動点条件として定義することの含意は大きい。「美しい」とは、忘却−回復サイクルの収束点にあること。一般的には Fix$(G \circ F)$ と書く。ここで F (発散/探索) と G (収束/活用) は §5 の関手の一般化であり、U と N はその特殊ケースである。

## §11　科学とはバカを自覚する営みである

### 11.1　ポパーへの反論

Karl Popper は科学の判定基準として**反証可能性** (falsifiability) を提唱した [15]: ある主張が科学的であるとは、それを反証しうる経験的テストが存在することである。この基準は直感的に魅力的であり、広く受け入れられている。

しかし、ポパーの基準には構造的な盲点がある。

圏論は反証不可能である —— 圏論を「実験で否定する」方法は存在しない。自由エネルギー原理 (FEP) も反証不可能だとしばしば批判される —— 「あらゆる結果を後付けで説明できてしまう」からだ。ポパーに従えば、これらは科学ではない。

だが「圏論は反証不可能だから無価値だ」と主張する数学者はいない。FEP が提供する統一的な枠組みが無価値だと主張するならば、その枠組みから導出された分析道具 —— 本稿の VFE (§2)、推論の相補構造 (§6)、忘却と回復の形式化 (§10) —— も棄却しなければならない。

何かがおかしい。ポパーの基準は正しいが、**狭すぎる**のだ。

### 11.2　科学の一般化された定義

§9–§10 の U/N の概念を用いて、科学の判定基準を一般化する<sup>††</sup>。

反証可能性を U/N の言葉で翻訳すると:

> **反証可能性** = $U_{causal}$（「この仮説は因果的に間違っている可能性がある」）を検出し、
> $N_{causal}$（実験による因果構造の回復）を適用すること。

つまり、反証可能性は **N の一形態** —— $N_{causal}$ —— にすぎない。

T9 はこれを一般化する:

> **科学** = 自身の $U$（忘却パターン）を検出し、
> 対応する $N$（回復操作）を **義務として** 適用し続ける系。

> **疑似科学** = 自身の $U$ を検出できない系。
> あるいは、検出しても $N$ を適用しない系。

ポパーが要求するのは $N_{causal}$ だけだ。T9 は $\forall i: \rho_i > 0$ を要求する —— **全ての忘却パターン**に対して回復を行い、学習の剰余を維持する義務。T9 はポパーの厳密な上位概念である。

<sup>††</sup> ここで T9 と呼ぶのは、筆者の枠組みにおいて「科学性の判定」が定理 9 番目として導出されたことに由来する。命名は慣例であり、本質ではない。

### 11.3　FEP は科学か

この T9 に照らして、§2 以来使い続けてきた FEP の科学性を再判定する。

FEP は以下の U（忘却パターン）を検出している:

- $U_{precision}$: 「なんでも説明できちゃう」 = 普遍性が高すぎて予測精度がゼロに近い
- $U_{causal}$: あらゆる結果を後付けで説明する = 因果構造の事後構成
- $U_{self}$: FEP で FEP を正当化する循環性
- $U_{context}$: FEP という単一の文脈に固着するリスク

そして以下の N（回復操作）を適用している:

- $N_{precision}$: メタ原理から操作的体系への「コンパイル」。本稿で用いた VFE の分解 (§2)、推論の相補構造 (§6)、忘却と回復の形式化 (§10) —— これらは全て FEP から**導出された具体的道具**である
- $N_{causal}$: 実験による予測と結果の比較
- $N_{self}$: FEP の限界の自省的記述
- $N_{context}$: Bellman 最適性原理・Shannon 情報理論・熱力学第二法則との構造的比較により、FEP の位置を相対化している

「FEP は反証不可能だから非科学だ」という批判は、$U_{causal}$ しか見ていない。それは §9 で述べた忘却関手の**片側だけを見る**行為 —— すなわち、批判者自身が $U_{adjoint}$（随伴の片方を忘れる）を犯している —— に他ならない。

FEP はむしろ、$U_{precision}$ を検出し $N_{precision}$ を適用して学習の剰余 $\rho > 0$ を産み出している。T9 の基準では、FEP は標準的な科学理論より**多くの** U パターンを検出し N を適用している。

> **「なんでも説明できちゃう」は $U_{precision}$ の症状**であって、死の宣告ではない。
> 症状を検出し、$N_{precision}$（具体的道具へのコンパイル）で回復する。
> これは科学の営みそのものだ。

### 11.4　結語

§1 でバカを「VFE が高い状態」と定義した。§4 で構造を見よと述べた。§5 でアナロジーの効率を証明した。§7 で自然変換によるアナロジーの品質評価を示した。§9 でバカの正体を忘却関手 U として同定した。§10 で回復関手 N が「元以上のもの」をもたらすこと ($\rho > 0$) を示した。§11 で科学を「忘却と回復の組織的な営み」として再定義した。

全てを一語にまとめるなら ——

> **バカとは、自分が忘れていることに気づかない系のことである。**
>
> **賢さとは、自分が忘れていることに気づき、取り戻す営みのことである。**
>
> **科学とは、その営みを組織的に行う系のことである。**

賢者に近づく方法は、自分が今見ているものが**対象**なのか**射**なのかを問うことである。対象を見ているなら、射を探せ。射を見ているなら、対象の名前を捨てよ。

バカをやめたいなら構造を見ろ。構造を見たら、忘れた構造を取り戻せ。取り戻したら、忘れていた自分自身を見ろ。

そして —— その営みを止めるな。止めた瞬間、また忘れるのだから。

## §12　実証 — LLM は構造を見るか、名前を見るか

> ここまでの §1-§11 は理論的な議論だった。§11.4 で本稿の主要な主張は閉じている。
> 本節は、その理論 — 特に §4 (構造を見る) と §5 (アナロジー = 関手) と §7 (忠実/非忠実) — が **現実の大規模言語モデル (LLM) の挙動として実測可能** であることを示す実証編である。読者は §11.4 で本稿の主論証を既に受け取っており、本節は独立に読んでも構わない。

### 12.1　関手の忠実度を測る実験

§4 で「射を見る者は対象を見る者に勝つ」と述べた。§7 で関手の忠実度が品質を決めると述べた。これらは数学的主張であり、実証を必要としない。

しかし **誰が構造を見ているか** は実証的な問いである。

本節では、5つの大規模言語モデル (LLM) が「構造を見る賢者」か「要素を見るバカ」かを、§5.3 の Δ_F を実測することで定量的に判定した実験結果を報告する。

### 12.2　実験設計 — Semantic Bleaching

CCL (Cognitive Command Language) という認知操作の形式言語を用いる。CCL はソースコードの**計算グラフの骨格** — §4 でいう射の配置 — を記号的に記述する。ソースコードから CCL への変換は、変数名・関数名・docstring・コメントを捨て、演算子の配置とデータフローのみを保存する忘却関手 U: Code → CCL である。

5つの条件で LLM にペアの構造的類似度 (0.0-1.0) を判定させた:

| 条件 | 入力 | 名前 prior | 構造の蒸留 |
|:-----|:-----|:-----------|:-----------|
| A: CCL のみ | 演算子列 | 遮断 | あり (関手 U 適用済) |
| B: CCL + コード | ソースコード + CCL | あり | あり |
| D: コードのみ | ソースコード | あり | なし |
| D': コード (名前 shuffle) | ソースコード (関数名を別の関数名に入替) | 誤情報 | なし |
| D'': コード (名前 ID化) | ソースコード (関数名を `f_8桁乱数` に置換) | ゼロ | なし |

条件 D → D'' への操作を **Semantic Bleaching** (意味漂白) と呼ぶ。名前は本質的には識別子であり (§12.3)、意味を持つのは人間の命名慣習の産物に過ぎない。Semantic Bleaching はこの慣習を除去し、LLM が名前 (対象) を見ているか構造 (射) を見ているかを分離する。

### 12.3　名前は射ではない

「`test_decreasing`」という関数名は、関数の中身 — 単調減少の検証ロジック — に対する識別子である。名が体を表すのは、書き手が読み手のために恣意的に内容と名前を一致させようと努めた**慣習**の産物であり、客観的な必然性はない。関数名を `f_l9ln1p0v` に変えてもコードは動く。

名前が構造情報を持つように見えるのは、人間のコーディング慣習 — **名前 ≈ 意味** という統計的共起 — が LLM の訓練データに降り積もっているからだ。これは §4.2 の米田の補題が言う「射」ではない。名前は対象のラベルであり、射は対象間の関係 (制御フロー、データ依存、型制約) である。

### 12.4　結果

対照は Phase C の QLoRA 微調整モデル (ρ≈0.96)。このモデルはコードの 49 次元幾何的埋込みで訓練されており、構造が認知に直接注入されている。

#### 12.4.1　5モデル比較 (条件 A: CCL のみ)

| モデル | Spearman ρ |
|:-------|:---:|
| Codex GPT-5.4 | 0.70 |
| Gemini 2.5 Flash | 0.70 |
| Gemini 3.1 Pro | 0.64 |
| Claude Opus 4.6 | 0.63 |
| Cursor Composer 2 | 0.59 |
| QLoRA (参考) | 0.96 |

全モデルが ρ ∈ [0.59, 0.70]。モデル間の差 Δρ ≈ 0.11 は、QLoRA との差 Δρ ≈ 0.26-0.37 に比べて小さい。CCL 構造の判定は「LLM 共通能力」であり、幾何注入 (QLoRA) は全モデルを普遍的に上回る。

#### 12.4.2　名前 prior の分離 (Semantic Bleaching)

| モデル | condD (名前あり) | condD'' (名前ゼロ) | Δ (名前 prior) |
|:-------|:---:|:---:|:---:|
| Codex GPT-5.4 | 0.731 | 0.570 | **+0.161** |
| Claude Opus 4.6 | 0.573 | 0.576 | -0.003 |
| Gemini 2.5 Flash | 0.482 | 0.548 | -0.066 |

三つのモデルが三つの異なる戦略を示す:

**Codex は名前 prior に最も依存している** (Δ = +0.161)。名前を剥ぐと ρ が 0.731 → 0.570 に急落する。Codex の「コード理解力」の 28% は、人間の命名慣習という統計的共起の搾取だった。

**Claude は名前 prior にほぼ依存していない** (Δ ≈ 0)。名前を剥いでも構造判定能力が変わらない。Claude は元から射を見ている。

**Flash は名前 prior が妨害している** (Δ = -0.066)。名前あり (0.482) < 名前なし (0.548)。Flash は名前という「対象のラベル」に惑わされ、構造判定が劣化する。§4.2 で述べた「対象を見るバカ」の定量的実証である。

決定的な発見は、**condD'' で全モデルが ρ ≈ 0.56 に収束する** (Codex 0.570, Claude 0.576, Flash 0.548) ことだ。名前 prior を除去すると、コード特化モデルも汎用モデルも同じ「純粋な構造理解力」を示す。condD での見かけの差 (Codex 0.73 >> Flash 0.48) は、名前 prior の搾取度の差に過ぎなかった。

#### 12.4.3　CCL の蒸留効果

| モデル | condD'' (名前ゼロ) | condA (CCL) | Δ (蒸留効果) |
|:-------|:---:|:---:|:---:|
| Gemini 2.5 Flash | 0.548 | 0.696 | +0.148 |
| Codex GPT-5.4 | 0.570 | 0.700 | +0.130 |
| Claude Opus 4.6 | 0.576 | 0.628 | +0.052 |

CCL の価値は「名前を剥ぐ」ことではない。名前 prior の効果 (§12.4.2) と CCL 蒸留効果は独立に作用する。CCL は計算グラフの骨格化 — §5 でいう関手 F: Code → Structure — として、名前遮断とは別の追加価値を持つ。

この追加価値 Δ は、まさに §5.3 の Δ_F (関手による Complexity 削減量) の実測値である。

### 12.5　含意 — 賢者の知覚と凡人の prior

§5.3 の理論的予測と実験結果の対応を示す:

| §5.3 の概念 | 実験での対応 |
|:---|:---|
| p_0 (一様 prior からの構築) | condD'' — コードから自力で構造を見る |
| F_* q_C (関手による prior) | condA — CCL が構造を手渡す |
| Δ_F (Complexity 削減量) | condA - condD'' = +0.148 (Flash) / +0.130 (Codex) / +0.052 (Claude) |
| 忠実関手 F | QLoRA の幾何的埋込み (ρ=0.96) |

構造を自力で見る能力 (condD'' ρ) が低いモデルほど、CCL (= 他者が作った関手) の恩恵が大きい。これは §3 の「賢者は系を見つけ、バカは系を作る」の実証的対応物である。

Flash は構造を自力で見る能力が低い (condD'' ρ=0.548)。しかし CCL という関手を渡されると ρ=0.696 に跳ね上がる — 賢者の関手を借りて賢くなっている。Claude は自力でそこそこ見える (condD'' ρ=0.576) ので、関手の恩恵は小さい。

しかし、どちらも QLoRA (ρ=0.96) には遠く及ばない。QLoRA は関手が認知に**埋め込まれている** — prior として渡されるのではなく、知覚そのものが構造を見るように再構成されている。

§8.1 の知性の階層で言えば:

| 水準 | 実験での対応 | ρ |
|:---|:---|:---:|
| 0-cell (対象) を見る | condD で名前に惑わされる Flash | 0.48 |
| 0-cell + prior を搾取する | condD で名前 prior を使う Codex | 0.73 |
| 1-cell (射) を見る | condD'' で全モデル収束する純粋構造理解 | 0.56 |
| 関手を借りる | condA で CCL を渡された Flash / Codex | 0.70 |
| 関手が埋め込まれている | QLoRA (幾何注入) | 0.96 |

Codex の condD ρ=0.73 は §8.1 の知性の階層では 0-cell 水準である。名前という対象のラベルに encoded された prior を利用しているに過ぎない。名前を剥ぐと (condD'' ρ=0.57) Claude と同水準に落ちる。コード特化モデルの「コード理解力」の相当部分は、構造の理解ではなく命名慣習の統計的搾取だった。
