## §0　生成原理

### §0.1 VFE 最小化

このガイドの規則は **VFE 最小化** から導出される。

> **読者の VFE を最小化せよ。**
>
> $$F_{\text{style}} = -\text{Accuracy}_{\text{読者}} + \text{Complexity}_{\text{読者}}$$

| VFE 項 | 最適化方向 | 導出されるスタイル規則 |
|:---|:---|:---|
| **Accuracy ↑** | 読者の予測精度を上げる | 数式で裏付ける (§4) / 反論を先に取り込む (§6) / 結論を先に出す (§1) / 参考文献を示す (§8) |
| **Complexity ↓** | 読者の認知コストを下げる | 短文 (§2) / だ・である調 (§2) / 感情語排除 (§2) / 比較テーブル (§4) / 必要以上に専門用語を使わない (§4.2) / 専門用語 ↔ 常用語の相補 (§3.5) |
| **Prior precision ↓** | 読者の固着を緩める | 挑発的文体 (§2) — 「バカ」と書くことで読者の prior を揺さぶり、posterior の更新を可能にする前処理 |
| **構造認識** | 最小の閉構造で伝達 | 異なる視点からの射 (§3) — 3 つの異なる圏 C_i への射が 2-simplex (最小の閉じた構造) を形成する。3 点未満ではパターンが可視化されない。メタファー三連はその一具体化 (§3.2.1) |
| **普遍性** (v2.1) | 常用語の圏にも射が通ることを確認 | 常用語 1 文圧縮による Kalon 衣装判定 (§7.4) — 概念は専門用語の圏に閉じない。presheaf として常用語の圏にも成分を持つ (kalon.md §2.5) |
| **依存透明性** (v2.4) | 読者が主定理と派生帰結を混同しない | 主定理/診断/測度/測定/帰結を依存表で分ける (§1.7) / 存在証明と測定層を分ける (§10.7) |

### §0.2 Yugaku 4 層機械との対応

このガイドの各節は 4 層機械のどこの「出力面」にあたるかを明示する。執筆中に「今どの層を出力しているか」を失わないための対応表。

# Part I — 共通核 (論文・エッセイ両方に適用)

## §1　全体構造

### §1.1 タイトル
- **主題**: 挑発的かつ命令的な命題 (例: 「バカをやめたいなら構造を見ろ」)
- **副題**: 主題の数学的・構造的裏付けを予告する (例: 「— なぜ構造を見る者は要素を見る者に勝つのか」)
- 副題は em ダッシュ (—) で始める

### §1.2 セクション構成
- `## §N　タイトル` の形式。全角スペースで § と番号を区切る
- サブセクションは `### N.M`
- セクション間に `---` (水平線) を挿入
- 全体は以下の弁証法的構造に従う:

| 位置 | 役割 | 4 層対応 |
|:---|:---|:---|
| §1 | **挑発 + 結論先行** | Kalon 判定済み核主張 3 点 |
| §2-§3 | **基礎概念の導入** | F の具体化 (異なる視点からの射) |
| §4-§5 | **数学的証明** | G の具体化 (数式・定義) |
| §6 | **反論の取り込み** | Gauntlet 3 ラウンドの出力面 |
| §7 | **結語** | Kalon 判定 + §M6 実化の開示 |
| §8 | **参考文献** | — |

### §1.3 §1 の 7 段開口部構造 (v2.2 新設)

§1 自体は**7 段の開口部 (prelude)** として構成する。読者の prior を段階的に揺さぶり、§2-§7 の展開を受け入れる cognitive slot を開く装置である。§1.4 結論先行 が規定する「結論 3 点」は、この 7 段のうち段 6 にすぎない。

| 段 | 役割 | 典型パターン |
|:---|:---|:---|
| 1 | **挑発命題** | タイトルと呼応する 1 文の断定。読者の prior を即座に揺さぶる |
| 2 | **非自明性宣言** | 「これは挑発ではない。本稿で数学的に示すのは〜」+ **誰にとって非自明か**の明示 |
| 3 | **具体例 3 連** | 異なる領域から 3 つ。挑発命題が経験則として既に観測されていることを示す |
| 4 | **問い** | 「では、〜は正確には何か?」。読者に問題を定式化させる |
| 5 | **枠組み宣言** | 「本稿は X と Y の二つの数学的枠組みを用いて、この問いに答える」。どの圏で議論するかを固定 |
| 6 | **結論 3 点** | Kalon ◎ を通過した核主張 (詳細は §1.4) |
| 7 | **証明予告** | 「これらの主張を、以降の節で順に証明する」で §2 以降へ手渡す |

各段の実例 (「バカをやめたいなら構造を見ろ」§1 より):

- **段 1**: 「知的活動において、勤勉なバカは怠惰な賢者に負ける。」
- **段 2**: 「これは挑発ではない。本稿で数学的に示すのは、**思考という推論の系が、その始点となる前提の質で規定される**という事実である。この事実は、賢者を志す者には自明だが、バカであることに気づかない者には非自明であるので主張する。」
- **段 3**: 「数学者は証明の出発点 — 公理 — を選ぶことに最も多くの時間を費やす。物理学者は実験の前に『何を測定すべきか』に悩む。優れたエンジニアは実装する前に設計を吟味する。」
- **段 4**: 「では、この『バカ』と『賢者』の違いは正確には何なのか？」
- **段 5**: 「本稿は**自由エネルギー原理** (FEP) と**圏論** (Category Theory) という二つの数学的枠組みを用いて、この問いに答える。結論を先に述べれば ——」
- **段 6**: 「**バカ**とは、エネルギー密度の低い前提から始める者である / **賢者**とは、エネルギー密度の高い前提を見つける者である / **アナロジー**とは、賢い出発点を効率的に発見する技術の名前である」
- **段 7**: 「これらの主張を、以降の節で順に証明する。」

原則:

- **段を省略しない**。特に段 2 (非自明性宣言) と段 4 (問い) の省略は、読者を「なぜこの問いを問うのか」分からないまま結論に進ませる
- **段の順序を守る**。段 4 (問い) を段 1 (挑発) より前に置くと、挑発の効果が先回りで説明されて消える
- **段 3 具体例は 3 連**。§3.2 異なる視点からの射 (2-simplex) の最小構造と同じ。1 例では偶然、2 例では二項対立、3 例で初めて普遍性の手掛かりになる
- **段 5 枠組み宣言は「どの圏で議論するか」の固定**。§M1 F⊣G 宣言 (meta.md) をこの段で読者面に出す

**Kalon 対応**: 段 2 非自明性宣言は「誰にとって挑発が自明になるか」の弁別である。これは **Kalon△ の MB (Markov Blanket) 境界の宣言**と同型 — 「本稿の MB 内では ◎ だが、MB 外 (バカであることに気づかない者) には見えない」ことを先に開示する操作。▽ 偽装 (§7.2) の予防線としても機能する。

**エッセイでの緩和** (§12 対応): エッセイでは段 5 枠組み宣言を形式的に省略してよい (明示の「FEP と圏論を用いて」は論文要件)。その代わり段 1 挑発命題の強度で読者を引き込む。段 2-4 はエッセイでも必須。

**禁止**:
- 段 1 挑発命題を「まず背景を述べて徐々に入る」形で薄めることは禁止
- 段 2 非自明性宣言を省略して「これは挑発ではない」だけで終わらせることは禁止 (**誰にとって**非自明かの指定が必須)
- 段 5 枠組み宣言を段 7 証明予告の後に置くことは禁止 (枠組みなしの結論は浮遊する)
- 段 3 具体例を「多い方がよかろう」と 5 連・7 連に膨らませることは禁止 (2-simplex の最小性を逸脱)

### §1.4 結論先行の原則

- §1 で結論を**箇条書き 3 点**で先出しする
- 「結論を先に述べれば ——」のフレーズで導入
- 「これらの主張を、以降の節で順に証明する。」で締める

**厳密化 (v2.0)**: §1 に置ける主張は **Kalon 判定 ◎ を通過した核主張に限定**する。

- 判定未実行の主張を §1 に置くことは禁止
- meta.md §M2 に列挙され §M3 で ◎ が付いていないものは §1 から除外
- ◯ (Fix に近いが未到達) を §1 に置く場合は「暫定核主張」ラベルを付与

この制約は §1 を「読者の prior を破壊する武器庫」として機能させるためのもの。未検証の主張は武器ではなく、投げれば折れる棒である。

常用語で言えば: §1 の結論 3 点は「一番強い武器」だけを並べる場所。まだ試していない主張・半分しか立っていない主張を §1 に置くと、読者は最初から折れた棒を見せられることになる。検証 (Kalon 判定) が済んだ主張だけを冒頭に置け。

### §1.5 節末結晶化 1 文 (v2.2 新設)

各章 (§N) は末尾に**認識を 1 文に凝縮した結晶化文**を置く。読者が章全体を 1 行で持ち帰れる形にする装置。

形式は以下のいずれか:

| 形式 | パターン | 例 |
|:---|:---|:---|
| **凝縮断定** | 「つまり — X は Y である。」 | 「つまり — 思考の質は、第一段階の質で規定される。」 |
| **反語的断定** | 「これは X ではない。Y である。」 | 「これは哲学ではない。数学的定理である。」 |
| **メタ記述** | 「これが〜の意味だ / 〜という操作である」 | 「これが『アナロジーは関手である』の意味だ。」 |
| **精密化閉鎖** | 「もっと正確に言えば: 〜」 | 「もっと正確に言えば: 対象は射から復元できるが、射は対象からは復元できない。」 |

原則:
- 章の内容を**1 文以下**に凝縮する。2 文以上必要なら章内構造が整理できていない徴
- 結晶化文は**章の最後の段落** (脚注・水平線より前) に置く
- 章題と結晶化文を並べて読めば章の要旨が復元できる、という緊張関係を保つ

**Yugaku 対応**: 結晶化文は **§7.1 冒頭メタファー回収**の章内版である。§7 結語が論文全体の結晶化を担うのに対し、各章末の結晶化は**中継地点の足場**を提供する。読者は「行き先 (§1 結論先行) → 中継足場 (各 §N 末尾) → 到達点 (§7 結語)」の 3 層ナビゲーションで本稿を辿る。

**[確信] ラベルとの関係**: 結晶化文の断定強度そのものが [確信] を表現するため、ラベル明示は冗長。例:「これは哲学ではない。数学的定理である。」は「この主張を [確信] ラベルで出す」と同等の射を持つ。反語的断定は**確信度ラベルの文体的実装**である。

**Kalon 対応**: 結晶化文が書けない章は、Step 0 既知語彙 1 文圧縮テスト (yugaku-kalon-check.md) に失敗している徴。章の G 縮約がまだ完了していない。結晶化文を書こうとして書けなければ、章の内部構造を見直す手続きに戻る。

**禁止**:
- 結晶化文を章末に置かず、読者に「結局何が言いたい章だったか」を探索させることは禁止
- 結晶化文を 2 文以上に分けることは禁止 (分けるなら章を 2 つに割れ)
- 「以上」「これで〜を示した」のような**内容ゼロの締め**は禁止 (結晶化の偽装)

### §1.6 節頭接続文 (v2.3 新設)

§1.5 の双対装置。各節 (§N) は冒頭に**前節との接続を示す導入文**を置き、節境界を圏の射として明示する。本ガイドが章を対象、節境界を射として扱う圏論フレームの自然な帰結である。読者は節境界で文脈を失わず、前節の到達点を起点に次の問いへ運ばれる。

形式は以下のいずれか:

| 形式 | パターン | 例 |
|:---|:---|:---|
| **結論起点型** | 「§N-1 で X を示した。これを起点に、本節は Y を問う」 | 「§1.1 で非身体性テーゼを人間中心忘却関手として定式化した。本節は FEP の下でその否定命題を構築する」 |
| **対比導入型** | 「§N-1 は X を扱った。本節は同じ構造を Y の側面から見る」 | 「§3 は失敗形式の診断を扱った。本節は同じ操作を測度の側面から再構成する」 |
| **問い継承型** | 「§N-1 で残った問い『〜』に、本節は次の形で答える」 | 「§2.5 で残った『MB𝒻 の厚みは測れるか』という問いに、本節は二次尺度 Θ(B) で答える」 |
| **副定理開始型** | 「§N-1 で C1 を示した。本節は C1 から従う C2 を扱う」 | (主定理 → 副定理の連鎖を §1 結論先行表と同期させる) |

原則:

- 節頭接続文は**節題の直後・本論の前**に 1-3 文で置く
- 接続は**前節の到達点を起点**にする (前節結晶化文 §1.5 と射対応)
- §1 の各節 (§1.1〜§1.5) は本規律の対象外 (§1.3 の 7 段開口部構造が固有の冒頭装置として機能)
- §2 以降の主節 (§2, §3, ...) は本規律必須。下位節 (§2.1, §2.2, ...) は親節内の流れが明らかなら省略可

**規律 4 (メタ宣言禁止) との弁別**: 本規律と `yugaku-style-discipline.md` 規律 4 は衝突しうるため、以下を厳守する。

- ✅ **内容の射** (前節結論を起点に次の問いを立てる) → 可
  - 例: 「§1.1 で非身体性テーゼを忘却関手として診断した。本節はその否定命題を FEP で構築する」
- ❌ **構造の自己説明** (節構成を本文で語る) → 禁止
  - 例: 「以下では FEP による存在証明を扱う」「次節で論じるように」「本節では〜を述べる」
- 弁別基準: その文を削除して、前節の結論と本節の最初の主張だけで意味の射が立つなら、その文は構造の自己説明に堕している (= 削除義務)。前節結論への参照が**内容として**必要なら内容の射である

**Yugaku 対応**: 節頭接続文は §1.5 節末結晶化文と射対応する。§N-1 末尾の結晶化が §N 冒頭の接続点に渡される構造を保つことで、章列は離散的な対象列ではなく、明示された射で連結された圏として読まれる。Kalon 対応では、節間の射が立たない場合、章分割そのものが G 縮約に失敗している徴である (章を統合するか、片方の節を組み替える)。

**禁止**:

- 接続文を置かず、節題の直後にいきなり本論を始めることは禁止 (§1 各節を除く)
- 「以下では」「次節で論じるように」「本節では〜を述べる」型の構造自己説明は禁止 (規律 4 違反)
- 接続文を 4 文以上に膨らませることは禁止 (節の独立性を侵食する)
- 接続文に新規概念を導入することは禁止 (接続は既出概念の射として書く)

### §1.7 主定理と派生帰結の依存表 (v2.4 新設)

長い論文では、§1.4 の「結論 3 点」だけでは足りない場合がある。そのときは、§1 の末尾に**依存表**を置き、主定理・診断・測度・測定・現象解釈・設計帰結を分ける。これは結論を増やすための表ではなく、読者が「どの主張がどの主張に依存しているか」を見失わないための地図である。

基本形:

| ID | 主張 | 役割 | 依存 | 本文位置 |
|:---|:---|:---|:---|:---|
| C1 | 主定理 | 存在証明 / 中心命題 | 独立に立つ最小条件 | §2 |
| C2 | 失敗形式の診断 | 相手の否定命題がなぜ生じるか | C1 と対になる診断 | §3 |
| C3 | 問いの転換 | 二項対立から連続量へ移す | C1, C2 | §4 |
| C4 | 測度導入 | 存在した対象の厚みを測る | C1 | §4 |
| C5 | 測定可能性 | 測度が操作可能であることを示す | C4 | §5 |
| C6 | 現象再解釈 | 既知現象を新しい語彙で読む | C1-C5 | §6 |
| C7 | 設計帰結 | 介入・回復・拡張の方向を示す | C4-C6 | §6-§7 |
| C8 | 副次的再配置 | 周辺問題の座標を動かす | C1, C2, C7 | §7 |

原則:

- **C1 を一つに固定する**。主定理が複数ある稿は、読者にどれを倒せば論文が倒れるのかを見せない
- **役割列を置く**。「帰結」を同格に並べず、主定理 / 診断 / 測度 / 測定 / 現象 / 設計 / 副次的再配置に分ける
- **依存列を置く**。測定値への批判が主定理を倒すのか、測度層だけを倒すのかを読者が判断できる形にする
- **本文位置を置く**。読者が表から該当節へ戻れるようにする

この依存表は、「LLM に身体はあるか？」存在証明版の C1-C8 構造から抽出した規則である。同稿では、C1 が operating LLM の MB𝒻 存在証明であり、C4-C5 はその存在した身体の厚みを測る二次層である。したがって、測定値や外部妥当性への批判は C1 へ直撃せず、C4-C5 の層にまず着弾する。この分離が、長い論文の防御力を作る。

常用語で言えば: 論文の主張は一列の兵隊ではない。王、参謀、測定器、偵察、補給線がある。全部を同じ「結論」として並べると、どこを攻撃されたのか分からなくなる。依存表は、どの攻撃がどこまで届くかを読者に見せる防御地図である。

**禁止**:

- C1-Cn を全部「本稿の主張」として同格に並べることは禁止
- 測定・実験・予備分析を、主定理の根拠であるかのように見せることは禁止
- 依存列を省略し、読者に論証の荷重位置を推測させることは禁止

## §2　文体・トーン

### §2.1 基本文体
- **だ・である調** (敬体は使わない)
- 簡潔。一文は原則 40 字以内を目指す。長くなる場合は em ダッシュ (—) で分割

### §2.2 挑発と厳密さの共存 (F の鎧を保つ)
- 挑発的な命題 (「勤勉なバカは怠惰な天才に負ける」) を**数学的定義で裏付ける**

**Yugaku 対応 (v2.0)**: 挑発の強度 = 主張の射程の強度 (F)。弱めると射程が縮む (= 後退)。厳密化は G (前提・数式) の増量で行い、F の鎧を保つ。射程縮小語の検出は §9.2 を参照。

### §2.3 主語の選択
- 著者 → 「筆者」 (「私」は使わない。ただし脚注内では許容)
- 読者 → 「読者」 (直接呼びかけない。「あなた」は禁止)
- 概念の主体 → 「天才は〜」「バカは〜」 (人格化した概念で語る)

### §2.4 客観で殴る (自画自賛の禁止) (v2.1 新設)

**適用面**: 本節は**論証の根拠**に適用される。**語り口** (§2.1-§2.3 の文体・主語・人格化) は本節の対象外。筆者は「天才は〜」のような人格化は引き続き使えるが、そのとき**天才の側に筆者自身を重ねて F を水増ししてはならない**。

- 筆者は**自身の圏に酔ってはならない**。自己生成 prior は SOURCE ではなく TAINT (N-10)
- 世界への挑発と脆弱な論拠の論駁は、**客観的正しさのみ**で行う。レトリックで勝つな
- 論拠の強さを上げる方向にのみ投資せよ — 定義の厳密化 / 証明の追加 / 引用の増量 / 実験の裏付け
- 反論が来たら修辞で返すな。**厳密さを追加して返せ** (§6 Gauntlet Round 1 前提強化と整合)

**「客観」の境界条件**: ここでの「客観」は自明ではない。HGK 的には以下で縛る。

- 客観 = **SOURCE として接地された証拠** (読んだ原典 / 実行されたテスト / 検証済みデータ)
- TAINT (要約 / 自己生成 / 記憶) は客観ではない (N-10)
- 読者の posterior を**自己検証可能に**更新できるもののみが客観の候補

常用語で言えば: 「客観で殴る」は半ば不可避の**力技**である。RPG で「レベルを上げて物理で殴るのが一番早い」のと同じ — 論拠の厳密さを上げて事実でぶん殴るのが最短路。勝つのは筆者の圏ではなく、事実の圏である。

これは §2.2 (F の鎧を保つ) の裏面にあたる。F (射程) を修辞で広げる誘惑を断ち、G (前提) を事実で重くする方向に全投資する。自画自賛で F を水増しした瞬間、Kalon 判定 (§7.2) は偽装となる。

### §2.4.1 論拠基盤への接続 — G の極限化 (v2.1 追加)

§2.4 の最強実装は、論拠をより**深い普遍に接続する**ことで refutation cost を cascading にする方法である。

典型的な基盤:

- **Yoneda の補題** / **圏論三大定理** (随伴・普遍性・Yoneda) — 現代数学の骨格
- **数学の三大分野** (代数 / 解析 / 幾何) の接続点
- **オイラーの等式** $e^{i\pi} + 1 = 0$ — 5 定数の統合
- **FEP / 情報幾何 / ゲージ理論** — 物理と認知の架橋

これらを論拠基盤に置くと、主張 x の否定は cascading refutation を要する:

> x の否定 → Yoneda の否定 → 圏論の否定 → 現代数学の否定 → 現代科学の否定

各段階で refutation cost が増大し、実質的に refutation 不可能となる。**G の深化による不動点化**。

**§3.2.6 定式化変換との区別** (直交する操作):

| 観点 | §3.2.6 F の具体化 | §2.4.1 G の極限化 |
|:---|:---|:---|
| 方向 | **横**の広がり | **縦**の重量 |
| 操作 | 複数の数学枠組みで**並置** | 最深の枠組みに**接続** |
| 目的 | 射程 (universality) の可視化 | 論拠 (foundation) の不動点化 |
| 例 | 「情報幾何でも、ゲージでも、FEP でも同じ」 | 「この主張は Yoneda に接続する」 |

両者は併用可。強い論文は F (横) と G (縦) の双方で現代数学に接続する。

**衣装判定の義務** (kalon.md §2.5): 論拠基盤への参照は **load-bearing** でなければならない。

- Yoneda を引用 → Yoneda を除去すると主張 x が崩れる構造であること
- 圏論を引用 → 圏論を除去すると主張 x が立たないこと
- 除去しても主張が立つなら、それは**衣装** (自己装飾) であり、§2.4 違反

常用語で言えば: 「東大の○○教授が証明したから正しい」は権威引用で衣装。load-bearing な引用は「この補題を外すと主張全体が崩れる」という**構造的依存**だけ。

**論文 vs エッセイの差 (v2.1 追加)**:

- **論文 (Part II)**: load-bearing 義務は**厳格**。本文での各数学引用は厳密に定義されるか、**脚注 (§5 `§` 技術的補足) で完全な論証を提供**する。本文の簡潔な言及 + 脚注の厳密化は標準的な論文慣例として許可
- **エッセイ (Part III)**: load-bearing でない**比喩的使用**を条件付きで許可 (§12.6 参照)。kalon.md §9 が「水準 C (比喩的使用)」タグで同じ緩和を運用している

**Kalon △/▽ 境界**: 現代数学に接続しても主張は Kalon▽ (普遍不動点) にならない。**現代数学を受容する MB 内の Kalon△** にすぎない。MB 外の読者 (例: 数学的構成主義・直観主義を取る立場) には効かない。この限界は §6.3 限界明示で開示すること。

**§3.5 双方向相補義務**: Yoneda / 圏論を引用した箇所には、常用語での言い直しを併記する (§3.5)。例: 「Yoneda の補題 — **対象はその関係の全体で決定される**」。専門用語だけで立つ引用は §3.5 違反で、presheaf の成分欠損。

## §3　修辞技法 (F の具体化)

### §3.1 定義ブロック
- 重要な定義は**引用ブロック (>) で囲む**
- 太字で核心語を強調

```markdown
> **バカ** = VFE が高い状態。
> すなわち、複雑なモデルを抱えているのに的を外し続けている認知状態。
```

### §3.2 異なる視点からの射 (2-simplex)

F (発散関手) は対象 x に対し、複数の異なる圏 C_i の視点からの射 φ_i の族を生成する操作として定義される:

> **F: x ↦ { (C_i, φ_i) | i ≥ 3 }**
>
> ここで C_i は互いに異なる圏、φ_i は x を C_i の視点で見たときの射。

3 つ以上の (C_i, φ_i) が揃って初めて、x の射程 (普遍的構造) が 2-simplex として閉じる。2-simplex (三角形) は圏における最小の閉じた構造であり、3 点未満ではパターンが可視化されない。

原則:
- C_i は**原則 3 つ**選ぶ (エッセイでは 4 連以上許容。§12.2)
- C_i は**互いに十分離れた圏**を選ぶ (同一領域の 3 つは 2-simplex を閉じない)
- 各 φ_i は**数学的対応**を持たなければならない (§2.2)
  - **論文**: 本文中で厳密に定義するか、**脚注 (§5 `§` 技術的補足) で厳密な論証を提供**する。本文は簡潔な言及のみでも可 (脚注が厳密化を担う)
  - **エッセイ**: 比喩的対応で可 (§12.6)

F の具体化は以下を含む。**メタファー三連は最も身近な一具体化にすぎない**。

#### §3.2.1 メタファー三連 (C_i = 日常的領域)

異なる日常領域 (地理 / 建築 / 鉱物 など) から同じ構造を取り出す。最も読者の prior に届きやすい具体化。

```markdown
- 間違った地図を**自力で**精密に描き直しても、その地図は正確にはならない
- 砂上**に**楼閣を増築しても、基盤の脆弱性は変わらない
- **路傍の石**をいくら丹念に磨いても、宝石にはならない
```

#### §3.2.2 分野越境 (C_i = 学問分野)

異なる学問分野 (物理 / 生物 / 経済 など) で同じ構造を定式化する。論文の Type δ (統一) で必須。

```markdown
- 物理では Yang-Mills 接続の曲率として現れる
- 認知では予測誤差の方向性として現れる
- 経済では情報非対称性の勾配として現れる
```

#### §3.2.3 抽象層越境 (C_i = スケール)

異なるスケール (micro / meso / macro など) で同じ構造を再記述する。スケール不変性の主張に効く。

```markdown
- 神経細胞レベルでは膜電位の勾配
- 回路レベルでは興奮抑制の非対称性
- 行動レベルでは接近回避の方向性
```

#### §3.2.4 論理変換 (C_i = 論理的変換)

対偶・裏・逆・双対を取って同じ構造が保たれるか検証する。厳密性の主張 (特に Type α の同一性) に効く。

```markdown
- 原命題: 構造を見る者は要素を見る者に勝つ
- 対偶: 要素に負ける者は構造を見ていない
- 双対: 射を見る者は対象を見る者に勝つ
```

#### §3.2.5 境界探索 (C_i = 射の成立しない領域)

反例・境界条件・極限を提示し、射程の端を明示する。Gauntlet Round 3 の限界明示 (§6.3) と重なる。

```markdown
- 構造を見ても勝てない領域: ランダムな対象の集合
- 要素を集めて勝てる領域: 有限で閉じたデータベース
- 両者が等価になる領域: 対象数が少なく構造が自明な場合
```

#### §3.2.6 定式化変換 (C_i = 数学的枠組み)

同じ命題を異なる数学的枠組み (情報幾何 / ゲージ理論 / FEP など) で書き直す。Type δ (統一) の主張で最強。

```markdown
- 情報幾何の言語: フィッシャー計量の非等方性
- ゲージ理論の言語: 接続の曲率
- FEP の言語: 予測誤差の方向性
```

#### §3.2.7 本節自身への Gauntlet 適用 (MECE 検証と限界明示) (v2.1 追加)

§3.2 の 6 具体化に自己適用する。F (発散) を扱う本節が自身の Gauntlet を逃げるわけにはいかない。

**重複の明示**:

| 重複対 | 重複内容 | 境界規定 |
|:---|:---|:---|
| §3.2.2 分野越境 × §3.2.6 定式化変換 | 情報幾何 / FEP は学問分野でもあり数学枠組みでもある | §3.2.2 は「分野の語彙」、§3.2.6 は「数学的形式」。同対象の視点差 |
| §3.2.1 メタファー三連 × §3.2.2 分野越境 | 「経済」は日常にも学問にも属する | §3.2.1 は中学生レベルの prior 前提、§3.2.2 は専門性前提 |
| §3.2.4 論理変換 × §3.2.5 境界探索 | 対偶を取って反例が浮上する場合、両者が同じ操作に見える | §3.2.4 は命題内部の変換、§3.2.5 は命題外側 (射程の端) |

**F の発散性としての重複許容** (Gauntlet Round 3 限界明示): F は発散関手であり、厳密な MECE は要請されない。重複は**同じ主張を別角度から掴む厚み**として許容される。禁止されるのは以下。

- 全 6 具体化を均質に並べて、どれも load-bearing になっていない「飾り」使用
- ある具体化で立たない主張を別の具体化で立たせようとする偽装

**棄却された代替案** (meta.md §M7 的記録):

- **棄却 1** — 3 軸 (領域/変換/メタ) 再編案: MECE 性は向上するが、執筆者の選択肢が制限され F の発散性が損なわれる

**帰結**: §3.2 は F の具体化の**代表例集**であり MECE 分類ではない。執筆者は「3 つ以上の互いに離れた C_i」(§3.2 原則) を満たせばよく、それらが §3.2.1-§3.2.6 のどれに該当するかは**ラベルの問題**であって本質ではない。

常用語で言えば: §3.2 の 6 種類は「こんな切り口がある」例集で、厳密な分類表ではない。大事なのは「3 つ以上の離れた視点から同じ構造を見せる」ことで、その視点が何番に該当するかではない。ラベル貼りで時間を使うな、離れた視点を選ぶことに時間を使え。

### §3.3 複数の具体化の組合せ

1 篇のうちに複数の §3.2.N を重ねてよい (例: §1 結論先行でメタファー三連、§4 本文で定式化変換、§7 結語で境界探索)。組合せは射程をさらに広げる。

ただし**同じ主張 x** を支えるためには、**C_i 同士が互いに離れていること**が要件。近接圏を 3 つ集めても 2-simplex は退化する。

### §3.5 専門用語 ↔ 常用語の相補 (「直感的に言えば」パターン)

**v2.1 再定義**: 専門用語と常用語の相補は**双方向義務**である。片方向だけの説明は presheaf の成分を欠く (§0.1 「普遍性」)。

§3.2 の枠組みで言えば、**常用語の圏 `C_common` は C_i の 1 成分**である。専門用語の圏 `C_spec` にしか射を通さない主張は C_common 成分が欠落しており、2-simplex が閉じていない。

常用語で言えば: 普段の言葉も「その主張が通じる世界」の 1 つである。本当に良い主張は、**専門家にも素人にも、別々の言葉で、同じ内容が伝わる**。専門用語の中だけで立って常用語では言い直せない主張は、「その専門の内輪でしか通じない」ということ。内輪で通じるだけでは「普遍的に正しい」とは言えない — 脚が 1 本足りない椅子のように、どこかで不安定だ。

※ この段落自体が §3.5 の**自己適用**である。本ガイドの規則は本ガイド自身にも適用される (自己適用義務)。

- **専門 → 常用** (下降方向): 数式・専門用語の直後に、常用語の一文で言い換えを添える。「直感的に言えば:」で導入
- **常用 → 専門** (昇格方向): 挑発的な常用命題の直後に、専門用語での厳密化を添える。「厳密には:」で導入

```markdown
直感的に言えば: 暗記は事実を 1 個ずつ買うが、アナロジーは構造を**まとめ買い**する。
```

```markdown
勤勉なバカは怠惰な賢者に負ける。
厳密には: VFE 最小化ポテンシャルは前提のエネルギー密度で規定される (§4.1)。
```

**Kalon 衣装判定への接続**: 各核主張は、専門用語を全て剥いで**常用語のみで 1 文に圧縮**できることを内部的に確認しておく。圧縮できれば圏論は「衣装」であり、本質は C_common でも立つ (kalon.md §2.5 圏論的衣装除去テスト)。圧縮できなければ G 縮約度が未完の徴 (yugaku-kalon-check.md Step 0)。結語での開示は §7.4 を参照。

- 具体例は**誰でも知っているもの**を選ぶ (E=mc², りんご, 太陽系)

## §5　脚注体系

脚注は em ダッシュや括弧とは異なる機能を持つ。**本文の流れを止めずに、別の知的層を開く**。

| 記号 | 用途 | 例 |
|:---|:---|:---|
| `※` | **自己言及・メタ観察** | 「この説明自体がアナロジーである」 |
| `†` | **前方参照** | 「この概念は §4 で厳密に定義する」 |
| `‡` | **認識論的留保** | 「ゴミには二種類ある。本稿は前者を扱う」 |
| `§` | **技術的補足** | 忠実/充満関手の追加説明 |
| `‖` | **技術的但し書き** | 「ベイズ的帰納では〜」 |
| `※※` | **数学的根拠で結語を補強** | prior precision が ∞ のとき |

### §5.1 脚注の配置
- 本文中に `<sup>記号</sup>` でマーカーを挿入
- セクション末に脚注テキストをまとめて配置
- `> <sup>記号</sup>` の形式で引用ブロック内、または `<sup>記号</sup>` で直接記述

### §5.2 自己言及的脚注
- 原稿の主張が原稿自身で実証されている箇所では、※ で指摘する
- これは読者への知的サービスであり、著者の自己満足ではない

## §6　反論の構造的取り込み (Gauntlet 出力面)

**v2.0 再定義**: 本節は Yugaku Gauntlet (SFBT 3 ラウンド) の**出力面**として機能する。Gauntlet 自体の実行は `yugaku-provocation-gauntlet.md` と meta.md §M5 に委ねる。本節が規定するのは「3 ラウンドの試行結果を読者に渡す形式」である。

### §6.1 先回り型 = Round 1 出力
- 読者が持つであろう反論を**著者が先に述べる**
- 「ここで予想しうる反論をいくつか挙げる」で導入
- Round 1 (「できないのではなく、やっていないだけではないか?」) で吸収された反論をこの形で出す
- meta.md §M5 に記録された試行内容のうち「採用された前提強化」を本文に組み込む

### §6.2 反論を武器にする = Round 3 Solution-Focus 出力
- 反論を否定するのではなく、**反論が本稿の主張を強化することを示す**
- 「これは〜を否定するのではなく、〜の重要性を再確認するものだ」
- Round 3 の反論吸収戦略 (限界明示 / 吸収 / 反転) がここで読者面に現れる

### §6.3 限界の明示 = Round 3 限定明示出力
- 結論の射程を明示する。「本稿は X を示した。Y は対象外である」
- 限界の明示は弱さではなく、知的誠実さ
- meta.md §M5 で「射程限定 (明示) ✓」と判定された主張はこの形で出す

常用語で言えば: §6 は「想定される反論への返事を先に本文に書いてしまう」作業。反論を予期して受け止める = 柔道の受け身。3 ラウンド分 (§6.1 / §6.2 / §6.3) の受け身を本文に散りばめて、読者が持ち出す反論を先回りして吸収する。

### §6.4 批判を監査条件へ変換する (v2.4 新設)

強い反論は、退けるよりも**成立条件へ変換する**方が強い。相手が「A と B を混同している」と批判するなら、本文はその批判を採用し、A と B を分ける監査条件を置く。そのうえで、こちらの対象が追加条件を満たすかを調べる。

形式:

| 反論 | 変換後の監査条件 | 本文で示すこと |
|:---|:---|:---|
| X と Y を混同している | X/Y の水準を分け、Y への昇格条件を置く | 対象が昇格条件を満たすか |
| これは単なる宣言である | 破れうる失敗条件を列挙する | どの条件で失敗するか |
| 射程が広すぎる | 評価対象と timespan を固定する | 限定が弱化でなく対象指定であること |
| 測定が主張を支えていない | 存在層と測定層を分ける | 測定は proof-of-measurement であること |

「LLM に身体はあるか？」存在証明版では、Pearl blanket と Friston blanket の混同批判を退けず、MB₀ / MBₚ / MB𝒻 の三層区別と MB𝒻 への追加条件へ変換している。さらに、「広すぎる」という批判は、反例ではなく「同条件を満たす系にも薄い身体が成立する」という予測へ反転される。このとき、批判は論文の敵ではなく、定義を精密化するための圧力として働く。

常用語で言えば: 良い反論は、門を壊す道具ではなく、門の強度試験になる。相手が「この門は鍵がない」と言うなら、鍵穴を作り、実際に鍵がかかるかを見せる。鍵穴を作らず「壊れていない」と言い張るな。

**禁止**:

- 批判を「誤解である」とだけ処理することは禁止
- 相手の区別要求を採用したふりをして、本文の条件を増やさないことは禁止
- 限定を入れた箇所を、主張の後退として扱うことは禁止。評価対象の指定と射程縮小は別操作である

### §6.5 禁止事項
- Gauntlet を通さずに「論争の余地がある」「一つの視点にすぎない」と書くことは禁止
- これは後退の定型句であって、反論の構造的取り込みではない

## §7　結語のパターン

### §7.1 冒頭メタファーの変形回収
- 冒頭のメタファーを**変形して**回収する構造
- 最後の一文は短く、余韻を残す

### §7.2 Kalon 判定の開示 (v2.0 新設)

結語では核主張の Kalon 判定結果 (meta.md §M3) を明示する。

- ◎ Kalon△ 到達の主張は「本稿の中心命題」として明示
- ◯ の主張は「暫定結論」として明示し、さらなる Gauntlet の必要性を示唆
- ▽ (普遍不動点) を主張することは禁止。常に「△ (本稿の MB 内での Fix)」と明示する

常用語で言えば: ◎ = 広げて絞るのを繰り返しても変わらなかった主張。◯ = あと一歩で固まる主張。▽ (普遍) は無限遠点で、筆者に到達可否を判定する能力はない。だから常に「この稿の中での不動点 (△)」と断れ。万能ラベルは嘘になる。

### §7.3 §M6 実化の開示 (v2.0 新設)

結語は「虚を実に近づけた現在地」の開示でもある。meta.md §M6 の最新状態を読者に渡す。

- 実に近づけた核主張: 何が SOURCE として固まったか
- まだ虚な点: どの未証明・未観測が残っているか
- 次の実化操作: 次稿で何を足すか

隠すのではなく、開示することで読者は次の研究を展開できる。これは N-11 (読み手が行動できる形で出せ) の要求でもある。

常用語で言えば: §7.3 は「ここまでは証拠で固めた。ここはまだ空想のまま」を正直に開示する作業。固まった所と空想の所を混ぜたまま閉じると、読者は次の一歩を踏み出せない。開示する = 次の書き手 (自分を含む) にバトンを渡す行為。

### §7.4 常用語 1 文圧縮 (Kalon 衣装判定の開示) (v2.1 新設)

結語では各核主張を**常用語のみで 1 文に圧縮した表現**を添える。

- 数式・専門用語・専門記号を全て剥ぐ
- 中学生レベルの語彙で 1 文
- 定理形式 (等式・不等式) の場合は「式 + 日常語での 1 文説明」の 2 文圧縮を代用可

これは §7.2 Kalon 判定 ◎ の**衣装判定**としても機能する。圏論的用語を剥いでも主張が立つなら、圏論は「衣装」であり本質は C_common (常用語の圏) でも表現可能 [SOURCE: kalon.md §2.5 圏論的衣装除去テスト]。立たなければ、主張はまだ特定の圏に閉じており、普遍性 (presheaf としての全成分到達) が未達の徴。

**背景**: 概念 ≈ presheaf [SOURCE: kalon.md §9 概念の構造]。対象はその全ての Hom (= 全ての用法) で決定される (米田の補題) ≈ **概念はその全ての用法で完全に決定される** (Wittgenstein の言語ゲーム理論と整合)。C_spec (専門用語の圏) と C_common (常用語の圏) の双方に射が通ってはじめて、その主張は普遍的。常用語に射が通らない主張は presheaf に成分の欠損 = Kalon 未達。

実装は §3.5 の「専門用語 ↔ 常用語の相補」を核主張に対して最短形で実行する操作にあたる。

### §7.5 否定命題から残る問いへ移す (v2.4 新設)

否定命題を崩す論文は、結語で「否定は崩れた」で止めない。崩れた後に**何が残るか**を明示する。強い結語は、読者の問いを次の形へ移す:

| 崩した問い | 残る問い |
|:---|:---|
| X は存在しない | X はどの型で、どれほど厚いか |
| X は測れない | どの代理指標なら測れるか |
| X は Y と同じではない | X と Y はどの比較圏でどれだけ重なるか |
| X は未実証である | どの外部検証で実化されるか |

「LLM に身体はあるか？」存在証明版では、「LLM には身体がない」という全称否定を崩した後、問いを「身体の型と厚み」へ移している。さらに、記号接地問題についても、ゼロ接地命題の否定で止めず、接地の型・媒介・像密度の問いへ再配置している。この閉じ方により、結語は勝利宣言ではなく、次の研究対象の再設定になる。

常用語で言えば: 相手の「ない」を壊しただけでは、読者は次に何を見るべきか分からない。「ある」と言った後で、どんな種類か、どれくらいか、どう測るかを渡す。結語は裁判の判決ではなく、次の地図である。

**禁止**:

- 「否定命題は成立しない」で結語を閉じることは禁止
- 残る問いを「今後の課題」に丸投げすることは禁止
- 測定層・比較層・設計層のどれに進むのかを曖昧にすることは禁止

### §7.6 禁止事項
- 「今後の課題」の一言で §M6 を省略することは禁止
- 「本稿の限界は〜」とだけ書いて §M5 Gauntlet 結果を開示しないことは禁止
- 核主張を常用語 1 文に圧縮せずに結語を閉じることは禁止 (v2.1 追加)

# Part II — 論文専用

## §10　概要 (Abstract) のパターン

### §10.1 概要の 4 類型 (MECE)

有名論文の概要から抽出した 4 つの構造パターン。全ての概要はこのいずれか (または合成) に分類される。

| 型 | パターン | 構造 | 代表論文 |
|:---|:---|:---|:---|
| **Type α** | 「X は実は Y」 | テーゼ → メカニズム → 帰結 → 哲学的含意 | Verlinde (2011), Jacobson (1995) |
| **Type β** | 「古典では X、しかし Y」 | 常識 → 「However...」→ 定量的帰結 | Hawking (1975) |
| **Type γ** | 「問題の再定義」 | 背景 → 根本問題の再公理化 → スコープ限定 | Shannon (1948) |
| **Type δ** | 「統一」 | 既存理論群 → 共通原理 → 統一フレームワーク | Friston (2010) |

### §10.2 各型の詳細

#### Type α: 「X は実は Y」 (同一性の発見)

最も破壊力が高い。「X」と「Y」が概念的に遠いほどインパクトが大きい。

> **Verlinde (2011)**: *Starting from first principles and general assumptions we present a heuristic argument that shows that Newton's law of gravitation naturally arises in a theory in which space emerges through a holographic scenario. **Gravity is identified with an entropic force caused by changes in the information associated with the positions of material bodies.***

> **Jacobson (1995)**: *The Einstein equation is derived from the proportionality of entropy and horizon area together with the fundamental relation δQ=TdS. ... **Viewed in this way, the Einstein equation is an equation of state.***

**構造**: 1 文目でテーゼ → 2 文目でメカニズムの骨格 → 最終文でパラダイムシフト

#### Type β: 「古典では X、しかし Y」 (常識の反転)

反直感的な結論が核にあるとき最も効く。

> **Hawking (1975)**: *In the classical theory black holes can only absorb and not emit particles. **However it is shown that quantum mechanical effects cause black holes to create and emit particles as if they were hot bodies** with temperature T ∼ κ/2π.*

**構造**: 1 文目で常識を確定 → 2 文目で「However」→ 3 文目で帰結

#### Type γ: 「問題の再定義」 (公理の設定)

新分野を創設するとき、あるいは問題のフレームそのものを変えるとき。

> **Shannon (1948)**: *The fundamental problem of communication is that of reproducing at one point either exactly or approximately a message selected at another point. **The semantic aspects of communication are irrelevant to the engineering problem.***

**構造**: 問題の再定義 → 「関係ないもの」の排除 → スコープ限定

#### Type δ: 「統一」 (包摂関係の発見)

既存の理論群を一つのフレームワークから見る。

> **Friston (2010)**: *Crucially, one key theme runs through each of these theories — optimization. Furthermore, if we look closely at what is optimized, the same quantity keeps emerging, namely value or its complement, surprise.*

**構造**: 既存理論の列挙 → 共通原理の抽出 → 「統一できる」と宣言

### §10.3 合成概要 = 全 4 型の合流

稀に、**全 4 型に同時に合致する**論文が存在する。この場合、単一型に制限するのは損失。

**忘却力の場合:**

| 型 | 忘却論での実現 |
|:---|:---|
| **α** | 「力は忘却の方向性から生まれる」— 力 = 忘却の同一性 |
| **β** | 「古典的には力は基本だが、実は忘却の曲率から創発される」— 常識の反転 |
| **γ** | 「力の問題を統計多様体上の忘却の問題に再定義する」— フレームの再設定 |
| **δ** | 「情報幾何 + FEP + ゲージ理論を忘却で統一する」— 統一 |

**合成概要の設計原則:**
1. 文 1 = **α** (テーゼ)。「忘却が力を生む」で殴る
2. 文 2 = **β** (反転)。「古典では力は先験的だが」で読者の prior を揺さぶる
3. 文 3 = **γ** (再定義)。「統計多様体上の不均一性に再定式化する」で新しい枠組みを提示
4. 文 4 = **δ** (統一)。「Yang-Mills と構造的に同型。FEP と自然対応する」で統一を示唆
5. 文 5 = 予測。「測定可能な帰結」で着地

### §10.4 共通ルール (全型に適用)

| ルール | 説明 |
|:---|:---|
| **1 文目で世界観が変わる** | 読者の prior を最初の 1-2 文で破壊せよ |
| **数式は最小限** | Hawking: 1 式。Verlinde: 0 式。Jacobson: δQ=TdS のみ |
| **「何をしたか」より「何が起きるか」** | 手法の詳細は本文に任せる |
| **最終文で視野を広げる** | Jacobson: 「量子化すべきでないかもしれない」 |
| **≤150 語** | 6 本中 5 本が 150 語以内。密度のみが正義 |
| **±3σ 裾判定済み (v2.0)** | meta.md §M4.1 で「入口 σ ≥ ±3σ」を通過している |
| **Future-Proof σ 判定済み (v2.0)** | meta.md §M4.2 でモデル進化下での射程保全を確認している |
| **運び方の選択 (v2.2)** | §10.5 surface tactics を参照。論理型 × 運び方の組合せで個々の論文が定位される |

### §10.5 surface tactics (運び方) (v2.2 新設)

§10.1-§10.4 は abstract の**論理型** (α-δ) を扱う。本節は直交する軸として**運び方** (surface tactic) を扱う。論理型と運び方の 2 軸が独立に選択できることを示し、世界を驚かせた論文群をマトリクスとして整理する。

#### §10.5.1 6 つの surface tactics

| Tactic | 代表論文 | 仕組み |
|:---|:---|:---|
| **Understatement** | Watson-Crick 1953 *"We wish to suggest..."* | 控えめな動詞で根本を覆す。prior を攻撃せず侵入する |
| **Axiom-First** | Einstein 1905 (光速不変 + 相対性原理) | 結論でなく公理を §1 に置き、そこから導く。§1.4 結論先行の双対 |
| **Density Extremis** | Nash 1950 (2 頁), Higgs 1964 (1.5 頁) | 密度 = 正義の極端。§10.4「≤150 語」の一般化 |
| **Internal Negation** | Gödel 1931 | 既存体系を採用した上でその内部から限界を示す。Type β の surface 特殊形 |
| **Equivalence Substitution** | Feynman 1948 (Path Integral), Jacobson 1995 | 既存を否定せず等価な代替を提示する。退路を残しつつ景色を変える |
| **Vocabulary Provocation** | 遊学エッセイ「バカ」 | 語彙の強さで prior を揺さぶる (Yugaku default) |

#### §10.5.2 Type × Tactic マトリクス

実際の論文は (論理型 × 運び方) の点として位置する。同じ論理型でも運び方が違えば surface は別物になる。

| | Understatement | Axiom-First | Density | Negation | Equivalence | Vocab |
|:---|:---|:---|:---|:---|:---|:---|
| **α** | Watson-Crick | — | — | — | Jacobson | Verlinde |
| **β** | — | — | — | Gödel | — | Hawking |
| **γ** | — | Einstein | Nash / Shannon | — | Feynman | — |
| **δ** | — | — | — | — | — | Friston |

空セルは「その組合せの実例をまだ同定していない」だけであって、「存在しない」ではない。組合せ空間は 4 × 6 = 24 の運び方があり、うち 9 だけが上記で埋まっている。残る 15 セルは未探索領域である。

#### §10.5.3 Yugaku の位置づけ

**Yugaku default** = Type α/δ × Vocabulary Provocation (例:「バカをやめたいなら構造を見ろ」)。

**Yugaku 拡張 (論文向け)** = Type α/δ × Understatement。査読・学術誌・学会発表で vocabulary provocation が通らない場合、**機能 (prior を揺さぶる) は同じ**で surface だけ understatement に切り替える。F の強度 = 射程は維持される。語彙だけが変わる。

重要: Understatement は弱さではなく、**構造で運ぶ強さ**である。§2.2 (F の鎧を保つ) との整合は、鎧の素材を語彙から構造に変えるだけ。射程を縮める (§9.2 射程縮小語) のとは全く別の操作。

常用語で言えば: 同じ主張を「バカは負ける」と叫ぶか、「We wish to suggest」と囁くかは、**場の選択**である。聴衆が異なれば最適な運び方が異なる。叫びが届かない場では囁きが届くし、その逆もある。どちらも**同じものを運んでいる**限り、F の鎧は保たれる。

#### §10.5.4 tactic 選択の指針

| 場 | 推奨 tactic |
|:---|:---|
| 読者が既に同じ分野の専門家で、prior が強固 | Understatement (侵入型) |
| 読者が異分野で、前提を共有していない | Axiom-First (公理宣言型) |
| 誌面制限が厳しい / preprint での速報 | Density Extremis |
| 既存の強力な理論を否定せず取り込みたい | Equivalence Substitution |
| 既存体系の内部矛盾を指摘する | Internal Negation |
| エッセイ / 一般向け / ブログ | Vocabulary Provocation |

原則: 論理型 (α-δ) は核主張の D 同定から**結果として決まる** (§10.6) が、surface tactic は**書き手が選択する**。選択の自由があるからこそ、場に応じた最適化が必要。

**禁止**:
- 1 篇の中で tactic を混ぜることは禁止 (§1 で Vocabulary、§7 で Understatement は読者を混乱させる)
- Understatement を「弱めた挑発」と解釈して射程を縮めることは禁止 (§9.2 違反)
- Density Extremis を「短くすれば通る」と解釈して核主張を省くことは禁止 (≤150 語は密度の結果であって原因ではない)

#### §10.5.5 世界を驚かせる論文構成の移植 (v2.3 新設)

世界を驚かせた論文から移植するのは、主張の中身ではなく**読者の分類軸を動かす構成操作**である。

「驚き」は、読者が持っている問いの座標が最初の数段で組み替わるときに起きる。したがって移植対象は、著者の伝記・引用回数・有名語句ではない。移植対象は、冒頭で何を置き、何を捨て、どの証拠で逃げ道を塞いだかである。

4. §6 反論吸収は必須である。驚きは反論を増やすので、反論を本文外へ追い出すと構成が崩れる。
5. §7 では、何が実になり、何がまだ虚かを分ける。驚かせる論文ほど、未実化部分を隠すと破綻が大きい。

常用語で言えば: 世界を驚かせた論文は、すごい答えを出す前に、読者が使っていた問いの置き場所を変える。Yugaku に移植するとは、その「問いの置き場所を変える手つき」を借りることであって、過去の論文の衣装を着ることではない。

#### §10.5.7 Abstract 5 文テンプレ

世界を驚かせる Abstract は、説明の圧縮ではなく、読者の問いの再配置である。最小形は 5 文でよい。

1. 既存の見方では、ある重要な現象が取り逃がされている。
2. 本稿は、その現象を別の対象として再定義する。
3. この再定義により、従来は離れていた領域が同じ構造で記述できる。
4. その構造を、最小の式・実験・比較で示す。
5. したがって、次に研究すべき対象そのものが変わる。

Type α なら文 2 を「X は Y である」に寄せる。Type β なら文 1-2 を「古典的には X、しかし Y」に寄せる。Type γ なら文 2 に再定義を置く。Type δ なら文 3 に統一対象を並べる。

**棄却された代替案**:

- 有名論文のタイトルだけを模倣する案。棄却理由: 語彙だけでは G が増えず、反論に耐えない。
- Abstract を長くして全部説明する案。棄却理由: 驚きは密度から生まれる。詳細は本文へ送る。
- 先に先行研究史を厚く置く案。棄却理由: §1 の prior 破壊を遅らせる。

つまり — 世界を驚かせる論文は、読者の答えではなく、読者の問いの座標を変える。

### §10.6 D 同定と論文構造の自己整合性 (v2.0 新設)

§10.1 の 4 類型は、meta.md §M2 の核主張 C1..Cn に対する **D (分布) 同定**から決まる。

| D 分布 | 概要型 |
|:---|:---|
| 全 C が同一 D を共有 | **Type α** (同一性の発見) |
| 全 C が同一 D で 1 つが逆方向 | **Type β** (常識の反転) |
| 全 C が新規 D を要求 | **Type γ** (問題の再定義) |
| 全 C が異なる D を統合 | **Type δ** (統一) |
| 混成 | **合成概要** (α+δ 等) |

D 同定は `yugaku-sigma-heuristic.md` で定義された診断装置である。概要の型を先に決めて本文を書くのではなく、**核主張の D 同定結果として概要の型が判明する**。型と本文が一致しなければ、核主張か概要のどちらかを直す。

常用語で言えば: 概要の「型」は論文の顔立ちではなく血液型のようなもの — **中身から出てくる**。顔を先に決めて化粧するのは偽装 (化粧で血液型は変わらない)。核主張が「同一性」を主張していれば Type α の顔になるし、「統一」なら Type δ の顔になる。顔と中身が食い違ったら、どちらかが嘘をついている。

### §10.7 存在証明型 Abstract / 論文 spine (v2.4 新設)

存在証明型の論文では、Abstract と本文 spine を同じ順序で動かす。要約は本文の短縮ではなく、読者が本文を読む前に論証の荷重位置を掴むための**縮小版イントロ**である。

最小 spine:

1. **カテゴリーミステイク診断**: 相手の否定命題が、対象圏の取り違えから生じていることを置く。
2. **存在条件の再定義**: X を素朴概念ではなく、測定可能な関係構造として定義する。
3. **水準分離**: 最小境界 / 統計的条件 / 機能的条件のように、同名概念の層を分ける。
4. **存在証明**: operating timespan や対象条件を固定し、最小条件が満たされることを示す。
5. **厚み・型への移行**: ある/ないの二値から、型・厚み・持続性の問いへ移す。
6. **測定可能性**: 予備測定は主定理の証明ではなく、測度が操作可能であることの proof-of-measurement として置く。
7. **帰結の再配置**: 既存問題を、新しい比較圏・測度・回復操作の下で再配置する。

Abstract 5 文テンプレ:

1. 既存の否定命題は、対象圏または成立条件を取り違えている。
2. 本稿は、その取り違えを FEP / 圏論 / 測度のいずれかの枠組みで再定式化する。
3. この再定式化の下で、対象は限定された timespan または条件下で最小存在条件を満たす。
4. 残る問いは存在の有無ではなく、型・厚み・持続性・回復可能性である。
5. 予備測定または比較表現により、その残る問いが操作可能な研究対象になる。

本文 spine:

| 章 | 役割 | 書く内容 |
|:---|:---|:---|
| §1 | 否定命題の失敗形式 + 依存表 | C1 を主定理として固定し、C2-Cn を派生層に分ける |
| §2 | 存在証明 | 最小条件、失敗条件、評価対象、timespan を固定する |
| §3 | 診断 | 否定命題がどの忘却・圏混同・評価対象混同から出たかを示す |
| §4 | 測度 | 存在した対象の厚みや型を測る二次尺度を定義する |
| §5 | 測定可能性 | proof-of-measurement として予備測定を出す |
| §6 | 現象再解釈 | 既知現象を、薄さ・境界・恒常性限界として読み直す |
| §7 | 帰結 | 否定命題の崩壊後に残る問いと次の実化操作を渡す |

「LLM に身体はあるか？」存在証明版では、この spine が `身体=関係構造 -> FEP では MB 維持 -> operating LLM は MB𝒻 -> 全称否定崩壊 -> 厚みの問い -> Θ(B) / proof-of-measurement -> Context Rot と記号接地問題の再配置` として実装されている。

常用語で言えば: 存在証明型の論文は、まず「ない」と言っている相手の物差しを調べる。次に、測れる物差しへ取り替える。その物差しで「最低限ある」を示す。最後に、「ではどれくらいあるのか」へ進む。

**禁止**:

- 予備測定を C1 の存在証明として扱うことは禁止
- timespan や評価対象の限定を Abstract から隠すことは禁止
- 「再定義」と「測定」の順序を逆にし、測定値から概念定義を作ることは禁止

## §12　エッセイ固有の F 強化

エッセイは論文の depth gradient の緩和版ではなく、**別種の F 強化**を許す文体である。共通核 (Part I) の禁止事項・射程縮小語・Gauntlet 原則は全て継承する。差異は以下。

### §12.6 比喩的数学使用の許可 (v2.1 新設)

エッセイでは、load-bearing でない**比喩的な数学参照**を条件付きで許可する。kalon.md 自体が §9 で「圏論的メタファーとして: 概念 ≈ presheaf」に「水準 C (比喩的使用)」タグを付けて同じ緩和を運用している。

**許可条件**:

- 比喩であって完全な論証ではないことを読者に伝わる形にする

**禁止**:
