### R3: T3 機能消化 — 実装可能な新機能

#### Feature 1: Precision-Gated Depth (PGD)

> **概要**: BC-14 発動条件に定量的閾値を導入

**現状**: BC-14 は「ハイリスク判断」という主観的条件で発動
**CATTS の知見**: エントロピー H と マージン Δ が不確実性の実用的信号
**提案**: WM に `$uncertainty` 変数を追加し、定量的閾値で BC-14 / 深度レベルを自動切替

```
if H > 0.5 or Δ < 0.3:
    depth = L3  # 深い反省
    bc14 = True
elif H > 0.3:
    depth = L2  # 標準
else:
    depth = L1  # 軽量
```

**消化レベル**: 🟡 Absorbed — BC-14 の発動条件を洗練する。新構造は不要。

#### Feature 2: PKS Multi-view Reranking

> **概要**: PKS の検索結果に attention-based reranking を追加

**現状**: PKS は embedding similarity のみで reranking
**AttentionRetriever の知見**: Attention scoring と Embedding scoring は相補的
**提案**: PKS に第二チャネル (attention/keyword-based) を追加しスコアを融合

**消化レベル**: 🟡 Absorbed — PKS の既存コードに reranker を追加。アーキテクチャ変更なし。

#### Feature 3: Sequential 優先ルール (CCL)

> **概要**: CCL 実行で Sequential (`>>`) をデフォルトに

**現状**: CCL の `>>` と `,` (並列) は使い分けが曖昧
**UniT/CATTS の知見**: Sequential が Parallel を一貫して上回る
**提案**: CCL パーサーで `>>` のない並列列挙に警告を出す (lint-like)

**消化レベル**: 🟢 Observation — ルールとして記録。コード変更なし。
