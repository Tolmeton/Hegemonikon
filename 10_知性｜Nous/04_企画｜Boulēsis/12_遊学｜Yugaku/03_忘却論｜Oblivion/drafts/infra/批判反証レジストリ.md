## §0. 運用規約

### §0.1 批判の状態分類

| 記号 | 状態 | 意味 |
|:--|:--|:--|
| 🔴 | 未対応 | 批判は記録されたが応答未 |
| 🟡 | 応答方針確定 | 応答方針はあるが実装未 |
| 🟢 | 応答済み | 論文本体/補遺で応答完了 |
| ⚪ | 却下 | 批判は成立しないと判断 (根拠明示必須) |
| 🔵 | 保留 | 判断未定、情報待ち |

### §0.2 エントリの標準項目

各批判エントリは以下の 8 項目を持つ:

1. **批判 ID**: `[論文番号]-[連番]` (例: `I-01`, `XII-03`)。横断批判は `Z-[連番]`
2. **出典**: 外部 (GPT/Gemini/人間名) / 自己想定 (/ele) / 他論文との衝突 / 内部整合チェック
3. **批判本文**: 1-3 文の要約。必要なら原文を引用ブロックで記録
4. **標的**: 論文の具体的節・定理・主張 (可能な限り行番号付き)
5. **応答方針**: 却下 / 保留 / 受け入れ / 反論 の 4 状態と根拠
6. **状態**: 🔴 / 🟡 / 🟢 / ⚪ / 🔵
7. **関連 OP**: Open Problem へのリンク (あれば)
8. **更新履歴**: 追加日、状態変更日

### §0.3 運用原則

1. **批判の追加は自由、応答の追加には SOURCE 必須** — 批判は思いつきでも記録可 (🔴 未対応で登録)。応答には論文本体 or 外部文書の根拠が必要
2. **却下は慎重に** — ⚪ 却下は「批判が成立しない」ことを明示する必要がある。「無視する」ではない
3. **状態の時系列追跡** — 各エントリの更新履歴を保持。状態変化 (🔴 → 🟡 → 🟢) が見える
4. **横断的批判は必ず §1 に** — 複数論文にまたがる批判は個別論文の §2.x に書かず §1 横断に集約。これが重複検出の鍵
5. **外部レビューは高優先度** — GPT/Gemini/人間からの批評は自己想定より優先的に処理 (自分の盲点を最も多く突いてくる可能性が高い)

### §0.4 本レジストリがしないこと

- 論文本体の改訂 (応答方針までに留め、実装は各論文セッションで行う)
- 批判の「解決」を勝手に宣言 (論文本体で応答完了してから 🟢 に更新)
- 却下理由の省略 (⚪ には必ず根拠を書く)
- 横断批判と個別批判の混在 (§1 と §2 は厳密に分離)
