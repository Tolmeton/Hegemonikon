## §1 問題: 49d は「分子式」に過ぎない

49d 特徴量ベクトル (`code_ccl_features.pkl`) は CCL トークンの**カウント**である。
これは化学における**分子式** (H₂O, C₂H₆O) に相当する。

分子式の本質的限界: **構造異性体が区別できない。**

```
エタノール: CH₃-CH₂-OH    → C₂H₆O
ジメチルエーテル: CH₃-O-CH₃  → C₂H₆O
分子式は同一。構造式で初めて区別できる。
```

CCL 特徴量における構造異性体:
```
/noe >> /bou _ /ene   (理解→意志→実行 — 順次パイプライン)
/noe * /bou >> /ene   (理解∧意志→実行 — 融合後変換)
```
両者は同じ CCL トークンカウント (noe=1, bou=1, ene=1) を持つが、
演算子構造 (`>>`, `_`, `*`) が異なる。49d では区別できない。

### 3.2 テンソル積は「分子式」— 結合の種類がない

テンソル積 (U_i × U_j) は原子の「共起」を捉えるが、「どう結合しているか」を捉えない。
Z-score 前処理で 3次項が復活する (var: 0→974) が、天井は構造式問題。

| 条件 | d | R@1 | R@10 | 解釈 |

| T4: Z-first 2nd+3rd | 453 | 2.4% | 17.2% | best テンソル積。なお 49d 未達 |

**tensor3 実験 (2026-04-05) の結論**: テンソル積は分子式の複雑化であって構造式ではない。
22d→453d に次元を 20 倍にしても 49d baseline の R@1=6.2% に届かない。

### 3.3 アドホック 27 次元 = 暗黙の「化合物」

49d のうち演繹核でない 27 次元 (特にアドホック 8 次元) が高い弁別力を持つ理由:

| アドホック次元 | 暗黙にエンコードする化合物 |
|:---------------|:-------------------------|
| `n_builtin` | U_arrow >> U_context (組込み関数 = 矢印と文脈の結合) |
| `n_method` | U_compose >> U_arrow (メソッド呼出 = 合成と矢印の連鎖) |
| `n_subscript` | U_arrow >> U_precision (添字アクセス = 矢印と精度の合成) |

各アドホック次元は **CCL 演算子の結合パターンを 1つの数値で暗黙に表現** していた。
「間違った方法で正しいことをしていた」— Python AST の特徴が偶然 CCL 化合物をエンコード。

### 道 A: ❌ 実験的に棄却

### 道 B: ⭐ 構造式の明示的特徴量化 — 未踏の中間道

**アイデア**: CCL 演算子 (>>, ~, *, _, |, &, ^, \) の結合パターンを
「官能基」として特徴量化する。テンソル積 (盲目的組合せ) ではなく、
**演算子構造を明示的にエンコード**する。

候補特徴量:

| 特徴量 | 化学的位置 | 定義 |
|:-------|:-----------|:-----|
| **演算子 n-gram** | 結合パターン | CCL テキスト中の演算子 bigram/trigram カウント (>>_, _*, ~>>, etc.) |
| **パイプライン深度** | 鎖長 | >> 演算子の最長連鎖 |
| **振動度** | 環構造 | ~, ~*, ~! 演算子の出現数と位置 |
| **融合度** | 架橋度 | * 演算子の出現位置 (中間=架橋, 末端=装飾) |
| **分岐度** | 分岐構造 | \|, & の出現による経路の分岐数 |
| **次元操作** | 酸化還元 | ^(上昇), √(下降) の出現回数と位置 |
| **制御構造** | 高分子骨格 | F:[], I:[], C:{} 等の制御構文のネスト深度 |
| **随伴参照** | キラル中心 | .d, .h, .x サフィックスの出現 (構造の対称性指標) |

**実装方針**:
1. CCL テキストをトークン化 (operators.md の演算子テーブルを辞書として使用)
2. 演算子 n-gram を特徴量ベクトルに変換
3. 既存 49d と結合して拡張ベクトルを構築
4. 自己検索 Recall@k で 49d baseline との比較

### 道 C: ✅ Phase C で実行中

Phase C v3 (GCE L4, PID 30886) が CCL テキストを直接 QLoRA で学習している。
transformer の BPE トークナイザが CCL テキストを処理するため、構造式→高分子レベルの情報が
暗黙的にエンコードされる。ただし BPE の分割が CCL 演算子の意味構造と一致する保証はない。

**道 B と道 C の関係**: 道 B は道 C の「解釈可能版」。

## §5 Z-score 前処理の技術的知見 (tensor3 実験より)

Z-score 正規化を交差項計算の **前** に適用すると:
- 2次交差項: zero_var が 33/231 → 21/231 に改善
- 3次交差項: top-5 の分散が 0.0000 → 974.1 に劇的改善
- R@10 が T1(11.2%) → T3(16.0%) に +4.8pp

**メカニズム**: raw 特徴量はスケールが不均一 (var: 0.0000~0.0036)。
乗算 (テンソル積) で小さい値同士が掛け合わされると機械精度の 0 に沈む。
Z-score で全次元を unit variance に揃えると、乗算結果が有意な値を持つ。

**適用範囲**: 道 B (構造式特徴量) を構築する際にも Z-score 前処理は必須。

### 6.3 根本原因の診断

**1. >> 演算子の支配**: bigram top-5 が全て `(>>, struct)` 系。CCL テキスト中の `>>` は
43549 回出現し、他の演算子は 1桁少ない。`>>` は「任意のステップ間接続」であり、

次元が増えるほど劣化 (S5=171d < S3=149d < S0=49d)。

**3. code_to_ccl 変換器の情報損失**: 現在の CCL テキストは Python AST からの機械的変換。
CCL 演算子の本来の意味構造 (>>, ~, *, % 等の使い分け) ではなく、
AST のノード接続を全て `>>` で表現している。**構造異性体を保存する精度で変換していない**。

## §8 化学同型のフル展開 — CCL の周期表・結合・官能基・特性

化学との同型を「たとえ話」ではなく「完全な同型」として展開する。
化学の各概念に CCL の対応物を定義し、特徴量設計への含意を導出する。

### ~~8.1 素粒子 → トークン分類~~ (v2 で削除: カテゴリ誤り)

> **初版の誤り**: 素粒子 (陽子/中性子/電子) → トークン種別の対応を定義していた。
> しかし化学の素粒子は原子を「構成」する — 陽子と中性子から核ができ、電子が周りを回る。
> CCL のトークンは互いを構成しない — `>>` は `fn` と `¥` から作られない。
> **全てのトークンは同じ抽象レベルの「原子」であり、互いを「結合」で繋いでいる。**
>
> 正しい分類: **原子 = 意味トークン** (§8.2)、**結合 = 演算子** (§8.3)。

### 8.2 周期表 — CCL 原子の分類と特性 (v2: 修正)

> **v2 修正 (2026-04-06)**: 初版は演算子 (>>, *, |) を「原子」と定義していた。**これは誤り。**
> 化学の原子は「それ自体で安定に存在できる最小単位」。
> `>>` は原子と原子を繋ぐ結合であって、それ自体では意味を持たない。

**原子 = 意味トークン** (それ自体で意味を持ち、独立に存在できる):
- `fn` (関数呼出), `#` (内部状態), `¥` (外部入力), `.method` (メソッド),
  `pred` (述語), CTRL (`F:`, `I:`, `V:`), `str_` (リテラル), `.attr` (属性), `!err` (例外)

**結合 = 演算子** (原子間を接続し構造を作る。→ §8.3 で詳述):
- `>>` (合成), `*` (融合), `|` (選択), `&` (条件接続), `_` (シーケンス)

CCL 周期表 — 意味トークンの 3×3 分類:

```
              情報源              情報変換            情報制御
         ─────────────────  ─────────────────  ─────────────────
  外部  │  ¥ (外部入力)    │  fn (関数呼出)   │  !err (例外)    │
  内部  │  # (内部状態)    │  .method (メソッド)│  pred (述語)   │
  構造  │  str_ (リテラル) │  .attr (属性)    │  CTRL (F:/I:/V:)│
         ─────────────────  ─────────────────  ─────────────────
```

**原子の特性テーブル:**

| CCL 原子 | 結合価 | 電気陰性度 (情報吸引力) | 情報保存性 | 副作用性 |
|:---------|:------:|:---------------------:|:---------:|:-------:|
| `¥` | 1 | — (情報源) | 外部依存 | 入力 |
| `#` | 多 | — (内部蓄積) | 内部完結 | なし |
| `fn` | 1-多 | 中 (変換) | 変換 | 関数依存 |
| `.method` | 1 | 低 (属性参照) | 保存 | オブジェクト依存 |
| `pred` | 1 | 高 (判定) | bool 化 | なし |
| `str_` | 0 | — (定数) | 完全保存 | なし |
| `.attr` | 1 | 低 (読取) | 保存 | なし |
| `!err` | 1 | 高 (例外捕捉) | 例外情報 | 制御フロー変更 |
| `F:` | 1+ | 中 (蓄積) | 蓄積 | 状態変更 |
| `I:` | 1-2 | 高 (分岐判断) | 経路依存 | なし |
| `V:` | 1 | 高 (通過/棄却) | 通過なら保存 | なし |

### 8.3 化学結合 — CCL における演算子の結合論 (v2: 拡張)

> **v2 拡張 (2026-04-06)**: 初版は演算子を「原子」側に配置していた。
> 正しくは**演算子 = 結合** (§8.2)。結合エネルギー・配位数・結合角の概念を追加。

化学では結合の種類が分子の性質を決定する。
CCL でも**演算子 (結合) の種類**が構造異性体を区別する鍵。

| 化学結合 | CCL 演算子 | 意味 | 結合エネルギー | 配位数 |
|:---------|:-----------|:-----|:--------------|:------|
| **共有結合 (単)** | `>>` | 順次パイプ。A の出力が B の入力 | 低 (透明、可逆) | 2 (1:1) |
| **共有結合 (二重)** | `*` | 融合。A と B が 1 つになる | 高 (lossy、不可逆) | 2+ (多:1) |
| **選択結合** | `\|` | A または B のいずれか | 中 (情報棄却) | 2 (1:1) |
| **条件結合** | `&` | A 成立の上で B | 中 (条件付き) | 2 (1:1) |
| **イオン結合** | `¥ >> #` パターン | MB 境界を超える情報移動 | 高 (境界越え) | 2 |
| **水素結合** | `.attr`, `.method` | 弱い参照。所有者に依存 | 低 (借用) | 1 |
| **金属結合** | `F:[each]{...}` | 多数の要素が均等に結合 | 中 (反復) | N |
| **ファンデルワールス** | 隣接するが演算子なし | 近接効果。弱い文脈共有 | 最弱 | 0 |

**結合エネルギー** (情報変換の不可逆度):
- `>>` は情報を透過的に渡す (可逆) → 低エネルギー
- `*` は 2 入力を 1 出力に融合する (lossy) → 高エネルギー
- `|` は選択後に非選択枝の情報を捨てる → 中エネルギー
- 特徴量化: 高エネルギー結合の比率 = 情報損失の度合い

**配位数** (1 つの原子が形成できる結合の数):
- `fn` は引数の数だけ結合を持てる → 可変配位
- `¥` は 1 結合 (1 入力源) → 配位数 1
- `#` は参照される回数だけ結合を持てる → 高配位
- 特徴量化: 平均配位数 = コードの「結合密度」

**結合角** (3 原子間の空間的配置 — トポロジー情報):
- `¥ >> fn >> #`: 直線型 (180°) — 単純パイプライン
- `(¥ * #) >> fn`: T 字型 — 融合入力
- `I:[cond]{A} | E:{B}`: V 字型 — 分岐構造
- 特徴量化: 直線/T 字/V 字の比率 = 構造の幾何学的指紋

### 8.5 分子の特性 — コード全体の性質

化学の分子には沸点、溶解度、反応性などの **巨視的特性** がある。
コード (CCL テキスト全体) にも同様の巨視的特性がある。

| 化学的特性 | CCL 特性 | 定義 | 計算方法 |
|:----------|:---------|:-----|:---------|
| **分子量** | トークン総数 | コードの規模 | `len(tokens)` |
| **沸点** (複雑さ閾値) | 認知的複雑度 | 理解に必要な作業記憶量 | ネスト深度 × 分岐数 |
| **極性** (情報の偏り) | 情報フロー方向性 | 入力偏重か出力偏重か | `(n_¥ - n_output) / n_tokens` |
| **溶解度** (再利用性) | 構成的独立性 | 他の文脈に溶け込めるか | 外部依存 (¥) の比率の低さ |
| **反応性** | 副作用密度 | 環境を変更する度合い | `.method` + `!err` + 出力操作の比率 |
| **安定性** | べき等性 | 繰り返し実行で結果が変わらないか | `V:` の比率。ループ内状態変更の有無 |
| **対称性** | 構造的対称性 | 入出力の型的対称性 | `¥ >> ... >> ¥` パターンの有無 |
| **密度** | 演算子密度 | トークンあたりの構造的操作量 | `n_ops / n_tokens` |
| **異性体指数** | 構造異性体の数 | 同じ原子組成で何通りの構造があるか | (理論的概念 — 直接計算困難) |

### 8.6 化学反応 — コード間の関係

| 化学反応 | CCL 操作 | 意味 |
|:---------|:---------|:-----|
| **合成** (A + B → AB) | `>> _` 連結 | 2 つの処理を直列化 |
| **分解** (AB → A + B) | リファクタリング | 1 関数を 2 関数に分割 |
| **置換** (AB + C → AC + B) | 引数変更 | 入力の差し替え (¥ の変更) |
| **触媒** | 高階関数 | 自身は変化せず反応を促進 (fn を引数に取る fn) |
| **酸化還元** | 抽象度変更 (^, √) | 情報の抽象レベルを上げ下げ |

## §9 結晶構造予測問題との同型 — v3 ネガティブ結果の化学的説明 (2026-04-06)

### 9.1 なぜ追加特徴量は 49d を希釈するのか

v1~v3 の全実験で、理論的に導出した追加特徴量 (topo, fg, n-gram) は 49d baseline を劣化させた。
これは「道 B が無効」なのではなく、**化学が予測する失敗パターン**と同型の困難に直面している。

**結晶構造予測問題 (Crystal Structure Prediction, CSP)**:
化学では分子式・官能基・結合角を完全に知っていても、結晶のパッキング (分子の空間配置)
を予測するのは極めて困難である。2007年の Blind Test (Lommerse et al.) では、
計算化学の最先端手法ですら正解率が低かった。

理由: **分子「内」の情報と分子「間」の配置は異なるスケールの問題**である。

| レベル | 化学 | CCL 特徴量 |
|:-------|:-----|:-----------|
| 局所 (分子内) | 原子・結合・官能基 | 49d / topo / fg |
| 大域 (分子間) | 結晶パッキング | cosine similarity |
| ギャップ | 分子間力のモデリング | **メトリクスの選択** |

### 9.2 cosine similarity は暗黙の忘却関手

cosine similarity: sim(a, b) = a·b / (|a|·|b|)

この操作は以下を暗黙的に忘却する:
- **ベクトルの大きさ** (L2 正規化で消去) — 分子量に相当する情報
- **次元間の相関構造** (内積は独立性を仮定) — 結合角に相当する情報
- **非線形な関係** (cos は線形射影) — 化学結合の非線形性に相当

つまり cosine は **暗黙の忘却関手 U_cos** であり、その核 (ker) は:
```
ker(U_cos) ⊇ {大きさの情報, 次元間相関, 非線形構造}
```

これは定理 3 (集約射影の核) の別インスタンスである:
- 定理 3: ker(count) = {順序情報} → 分子式が構造異性体を潰す
- §9 拡張: ker(cosine) = {大きさ, 相関, 非線形性} → メトリクスが構造情報を潰す

### 9.3 49d が最強である理由の仮説

49d baseline が全追加特徴量より強い理由として、3つの仮説が考えられる:

**H1 (偶然整合仮説)**: 49d の情報構造が cosine の ker と「偶然」整合している。
Python AST のアドホック特徴量が、cosine が保存する部分空間に情報を集中的にエンコード
していた。理論的特徴量は cosine の ker に落ちる次元を追加するため希釈が起きる。

**H2 (情報飽和仮説)**: 49d が cosine で抽出可能な情報の上限に既に到達している。
追加次元は cosine の保存部分空間に新しい情報を追加できない。

**H3 (次元の呪い仮説)**: 追加次元は有意な情報を持つが、高次元空間での
距離集中現象 (concentration of measure) により弁別力が希釈される。

判別方法:
- H1 vs H2: Mahalanobis 距離や learned metric で 49d+topo が改善すれば H1
- H2 vs H3: fg-only (49d なし) の実験で fg が独立に弁別力を持てば H3
- H1+H3: 次元削減 (PCA/UMAP) 後に topo/fg を追加して改善すれば両方

### 9.4 忘却論による解決の方向

忘却論の中心テーゼ: **忘却は制御可能な設計パラメータ**である (Paper I)。

CSP 同型を忘却論で読み替えると:

1. **cosine は「何を忘れるか」を制御していない** — 暗黙の U_cos が構造情報を潰す
2. **49d は「何を忘れるか」が偶然適切だった** — AST の抽象化が良い忘却関手
3. **解決 = 忘却関手を「選ぶ」のではなく「学ぶ」** — Phase C の本質

| 忘却の種類 | 何が潰されるか | 対応する操作 |
|:----------|:-------------|:-----------|
| U_count (分子式) | 順序 → 構造異性体 | 49d ベクトル化 |
| U_cos (cosine) | 大きさ・相関・非線形 | 距離計算 |
| U_embed (embedding) | モデル依存 | Phase C (transformer) |
| **U_learn (learned)** | **データから学習** | **metric learning** |

Phase C (transformer 直接学習) が有望なのは、U_learn を暗黙的に学習するから。
しかし Phase C の ker は不透明 (BPE トークナイゼーションの影響)。
道 B が提供すべき価値は「Phase C の ker を**解釈可能にする**」こと。

### 9.5 「知性は溶媒である」との接続

「知性は溶媒である」(マクロスケール) では:
- 溶媒 = embedding 空間
- 溶解 = 情報のベクトル化
- **溶媒効果 = embedding が距離空間の曲率を変える**

49d + cosine = **真空中の分子間力**。分子間力は真空では正確だが、
溶媒中 (embedding 空間内) では溶媒効果により修正される。

Phase C = **溶媒中の距離計算**。transformer の hidden state が溶媒であり、
attention が溶媒効果 (相互作用の選択的増幅/減衰) を実現する。

道 B の位置づけの修正: 道 B は「より良い分子式」を作る試みだった。
しかし CSP 問題が示すように、分子式の精緻化では結晶パッキングは予測できない。
道 B の真の価値は「Phase C の結果を**解釈する道具**」であり、
Phase C に代わる手法ではない。

## §10 定理 2 の精密化 — ゴールディロックス帯 (2026-04-06)

### 10.1 問題

定理 2 (multi_scale_chemistry_isomorphism.md §6) は p(x)→1 で enrichment→1 を示すが、
enrichment が最大になる coverage の範囲 (ゴールディロックス帯) を特定していない。

### 10.2 enrichment の上界

enrichment の定義 (定理 2 より):
```
enrich(g) = P(同カテゴリ | 両方 ∈ G_g) / P(同カテゴリ | ランダム)
```

G_g = パターン g を含む関数集合。p = |G_g| / N (coverage)。

**分子** P(同カテゴリ | 両方 ∈ G_g) は G_g 内のペアの類似度に依存。
G_g が小さいほど (coverage が低いほど)、G_g 内の関数は g という共通構造を持つため、
類似度が高くなる**傾向**がある。

ただし、P(同カテゴリ | 両方 ∈ G_g) は最大で 1 (G_g 内の全関数が同カテゴリ)。

**分母** P(同カテゴリ | ランダム) = 定数 c (コーパス全体の基底類似度)。

したがって: enrich(g) ≤ 1/c (上界)

### 10.3 ゴールディロックス帯の導出

enrichment の期待値を coverage p の関数として考える:

```
E[enrich(p)] ≈ α(p) / c
```

ここで α(p) = G_g 内の同カテゴリ率。

**大きい p (高 coverage)**: G_g ≈ 全集合 → α(p) ≈ c → enrich ≈ 1 (弁別力ゼロ)

**小さい p (低 coverage)**: G_g は少数の関数。2つの効果が競合する:
- 構造的効果: 共通パターン g を持つ → α(p) > c → enrich > 1
- 統計的効果: |G_g| が小さい → 推定の分散が大きい → 信頼区間が広い

統計的に有意な enrichment を検出するには、|G_g| = pN のペア数が十分必要:
```
ペア数 = C(pN, 2) = pN(pN-1)/2 ≥ K_min  (最小ペア数)
```

K_min ≈ 50 (統計的閾値) とすると:
```
p_min ≈ √(2K_min) / N ≈ √100 / 11768 ≈ 10/11768 ≈ 0.085%
```

**ゴールディロックス帯**: p_min < p < p_transition

ここで p_transition は α(p) が c に近づき始める閾値。

### 10.6 定理 2' (精密化版)

**定理 2' (ゴールディロックス定理)**: 離散確率空間 (Ω, p) 上の部分構造 g について、
enrichment の期待値 E[enrich(g)] は coverage p = |G_g|/N の関数として:

1. **飽和帯** (p > p_sat): E[enrich] → 1 (定理 2 の再掲)
2. **ゴールディロックス帯** (p_min < p < p_sat): E[enrich] は最大値を取りうる
3. **統計不足帯** (p < p_min): E[enrich] の分散が期待値を上回る (信頼不能)

p_min ≈ √(2K_min) / N, p_sat は構造的共有度が基底率に接近する閾値。

**系**: 最適な部分構造は「珍しすぎず、ありふれすぎない」coverage を持つ。
これは化学の元素存在比 (弁別的元素 = 中頻度) と同じ法則の同一インスタンスであり、
Shannon のエントロピー不等式 I(X;Y) ≤ H(X) の操作的帰結である。

## §12 U_cos ボトルネック — 距離関数が真の壁

### 12.1 2段階忘却の構造

```
CCL テキスト →[U_count]→ Nd ベクトル →[U_cos]→ 類似度スカラー
               特徴量設計              距離関数
               (改善した)              (ボトルネック)
```

v1-v3 の全実験は U_count (特徴量設計) を改善しようとしていた。
しかし真のボトルネックは U_cos (cosine similarity) にある。

### 12.2 証拠: 独自情報の 94% が ker(U_cos) に落ちる

圏論的 18d は 49d の 45% を説明する独自情報を持つ (被覆 R²=0.451)。
しかし R@10 は 18.6% → 19.0% の +0.4pp しか改善しない。

18d の独自情報のうち cosine が保存するのは約 6% (+0.4pp / 6.6pp baseline)。
残り ~94% は ker(U_cos) に落ちる。

### 12.3 なぜ cosine が射の構造を殺すか

cosine = 線形内積 = 「次元ごとの独立な寄与の合計」。
しかし射の構造的情報は**次元間の関係**が意味を持つ:

- 「線形度が高い ∧ テンソルランクが低い」= 純粋パイプライン → この「∧」を cosine は捕捉できない
- 「反復パターンが同じ」は cosine で拾えるが、「反復パターンが相補的」(A が B の逆) は負の類似度になる
- 構造的類似 ≠ ベクトル空間の角度

### 12.4 Whitened 悪化の再解釈

Whitened (Mahalanobis) が -3.0pp に悪化した理由:

49d の次元間相関が「暗黙の構造情報」をエンコードしていた。
dim[n_seq] と dim[n_call] の共分散は「パイプライン型関数は呼出が多い」という構造情報。
Whitened がこの相関を除去 → 暗黙の構造情報が消失。

**「cosine + 49d の相関構造」= 1 つの最適化された忘却関手。**
個別に改善しようとすると壊れる局所最適。

### 12.7 フィルトレーション距離 d_lēthē — 忘却論から距離関数を演繹する

#### 12.7.1 cosine に「必然性」がない問題

cosine が検索距離に使われているのは慣習であって演繹ではない。
cosine が最善でないことは演繹的に示せる:
cosine は n=1 レベルの忘却像の内積 = フィルトレーション距離の 1 層の 1 測り方。
独自情報の 94% が ker(U_cos) に落ちる (§12.2)。

#### 12.7.2 距離の演繹的定義

Aletheia フィルトレーション (VISION §2.1):

```
n=0: 名前     → U₀: Code → Names
n=1: 射       → U₁: Code → CCL
n=2: 自然変換  → U₂: Code → TypeSeq
n=3: 豊穣     → U₃: Code → Precision
```

定義: d_aletheia(A, B) = min{ n | U_n(A) = U_n(B) }

連続化 — 多重距離:

```
d_lēthē(A, B) = Σ_n  w_n · d_n(A, B)
  d_n(A, B) = 1 - sim_n(U_n(A), U_n(B))
  w_n = 検索意図で動的決定
```

| レベル | 忘却関手 | 距離 | 測り方 |
|:-------|:--------|:-----|:------|

| n=∞ | U_count (混合) | cos(49d_A, 49d_B) | 現行 49d + cosine |

cosine は d_lēthē の n=2 と n=∞ でのみ適切。n=1 は Jaccard、n=1.5 は tree edit が自然。

## §13 Face Lemma による CSP 再定式化 — 1-skeleton から 2-skeleton へ

§9 で CSP を2次忘却として定式化し、§12.7 で d_lēthē を多層距離として導出した。
本節はこれらを Paper II §3.4 Face Lemma で統合し、**なぜ現行パイプラインが構造的に不十分か**と**何が必要十分か**を演繹的に導出する。

### 13.1 現行パイプラインの構造的診断 — 1-skeleton の限界

現行: f = U_count, g = U_cos の2射。
Nerve の 1-skeleton (辺のみ) に対応:

```
Code ──f──→ 49d ──g──→ cosine
```

Face Lemma (Paper II §3.4): 概念安定性に必要な**独立な生成射**の最低数は **3**。

```
stable(x) ⟺ ∃ composable triple (f, g, h) with g ∘ f = h containing x
```

dim Ξ = |独立射| - 1 = 2 - 1 = **1** (スカラー)。
dim Ξ = 1 では忘却の不均一の「形」が測定不能 — 大小のみ、方向なし。
**CCL 検索の失敗 = 2-skeleton の問題を 1-skeleton の道具で解いている。**

Stability Simplex Theorem (Paper II §3.5): 3射で 2-simplex が出現し、
面 (g∘f = h の検証可能性) = **冗長性** = **自由度**。
自由度があって初めて「つなぎ」が見え、概念が安定する。

### 13.2 なぜ「3」か — 3つの根拠の構造的同型

| 根拠 | 名前 | 「3」を要求する理由 |
|:-----|:-----|:--------------------|
| A. 合成の検証可能性 | Nerve 定理 | 2-simplex (面) = g∘f = h の検証に3射必要 |
| B. 忘却の不均一の測定 | §4.6e Ξ | dim Ξ ≥ 2 でスペクトラムの形状が測定可能。d=1 では大小のみ |
| C. 自己参照性 | Kalon 三属性 | Fix(G∘F) + Generative + Self-referential の3つで再帰が閉じる |

小圏の nerve は **2-coskeletal** (Paper II §3.4 (iv)) → 4射目は新次元を追加しない。
3は必要かつ十分。

### 13.3 Cell Hierarchy に整合した3パス — 正三角形の演繹

Cell hierarchy (Paper VIII §6.2):

```
0-cell: 原子 (トークン)     — 離散的存在
1-cell: 結合 (演算子構造)   — 原子間の関係
2-cell: 自然変換 (型フロー) — 結合間の関係
```

各レベルに自然な忘却関手と距離を対応:

| Cell | 忘却関手 | 距離 | 測り方 | §12.7 対応 |
|:-----|:---------|:-----|:------|:-----------|
| **0-cell** | U_count (原子カウント) | d_0 | 49d cosine | Stage 1 (d_∞) |
| **1-cell** | U_struct (構造パターン) | d_1 | FG Jaccard | Stage 1.5 (d_1) |
| **2-cell** | U_type (型フロー) | d_2 | TypeSeq cosine | Stage 3 (d_2) |

**正三角形の根拠** (Stability Simplex Theorem):

Δ_F = {F(η), ε_F, id_F} の固有値 = {+λ, -λ, 0} → V ≅ (ℝ, +) 一意 (Hölder)。
最大対称性実現 = 正三角形。

Cell レベルは定義的に独立 (米田: 上位→下位は決定するが逆は不成立)。
各ペアの ker 交差は残り1 cell レベルを residual として残す → **等辺**:
- ker(d_0) ∩ ker(d_1) → 2-cell のみが弁別可能
- ker(d_0) ∩ ker(d_2) → 1-cell のみが弁別可能
- ker(d_1) ∩ ker(d_2) → 0-cell のみが弁別可能

### 13.4 d_face — 2-skeleton 統合距離

§12.7 の d_lēthē は1次元 (線形結合) = 1-skeleton 距離。
2-skeleton に拡張:

```
d_face(A,B) = d_lēthē(A,B) · (1 + γ · Area(Δ(A,B)))
```

- **d_lēthē** = Σ w_n · d_n (線形結合 = 1-skeleton 距離)
- **Area(Δ)** = 3パス間の不一致度 (三角形の面積)
- **γ** = 面積重み (ハイパーパラメータ)

| Area | 意味 | 行動 |
|:-----|:-----|:-----|
| ≈ 0 | 3パス合意 → 高信頼 | d_lēthē で十分 |
| ≫ 0 | パス不一致 → CSP-hard | Phase C へ委任 |

Area ≈ 0 の領域 = Face Lemma が安定を保証する領域。
Area ≫ 0 の領域 = 2-skeleton でもなお不安定 → Phase C (G4) が必要。

d_face は §9.12 のロードマップの**Stage 2 と Stage 3 の間**に位置する:
Stage 1 は d_0 単独、Stage 1.5 は d_0 + d_1、
Stage 2 は d_0 + d_1 + d_2 (= d_lēthē)、**Stage 2.5 は d_face** (d_lēthē + 面積補正)。

### 13.5 2→3 窓実験 — 最優先予測

| 窓数 | dim Ξ | 予測 | 根拠 |
|:-----|:------|:-----|:-----|
| 1 (49d cosine) | 0 | R@10 = 25.6% (実測) | — |
| 2 (+ Jaccard re-rank) | 1 | +2.0pp (実測) | 量的改善のみ |
| **3 (+ TypeSeq)** | **2** | **質的跳躍** | Face Lemma 生成閾値: dim Ξ = 1→2 |
| 4 (+ AST edit) | 2 | c⁴/4! 階乗減衰 | 2-coskeletal 飽和 (Paper VII §7.4) |

**核心予測**: 2→3 の改善幅 > 1→2 の改善幅。
これは 1→2 が「量的」(dim Ξ 不変) で 2→3 が「質的」(dim Ξ 跳躍) であるため。

1. TypeSeq 特徴量抽出 (U_type: Code → 型シーケンス → ベクトル)

成功基準:
- **P38**: Δ(2→3) > Δ(1→2) (質的跳躍)
- **P39**: Area(Δ) と再現率の負の相関 (面積仮説)
- **P40**: Δ(3→4) / Δ(2→3) ≈ c/4 (2-coskeletal 飽和)

### 13.6 CSP 不良設定の構造的証明 — 「観測が解を殺す」

CSP under U = U_cos ∘ U_norm ∘ U_count は Hadamard の意味で不良設定:
- ✅ 存在: α=0 で解存在 (Paper VIII F1)
- ❌ 一意性: ker(U) 無限・構造なし (L3) — ~94% in ker(U_cos)
- ❌ 連続依存性: ker が 94% をカバー

しかし Face Lemma は「解けない」のではなく **「観測が解を殺す」** ことを示す:
1-skeleton の観測で解空間が退化 → 1射の追加 (2→3) で ker の交差が劇的に縮小 → 解空間が復元。

化学 CSP と完全に同型: 粉末X線回折 (1次元) では多形体が縮退。
単結晶X線回折 (3次元) で初めて解が一意に決まる。
**次元の追加が質的跳躍を生む構造は、忘却論の Face Lemma の普遍的帰結。**

## §15 制約-符号化 分離仮説との接続 — Paper XI Exp0 からの帰還 (2026-04-07)

### 15.2 U_cos = E 軸の忘却関手

§12 で定式化した U_cos (cosine が方向情報のみ保存し大きさを忘却する) は、Paper XI の E 成分の忘却関手として再解釈できる:

- U_cos: ℝⁿ → S^{n-1} (正規化。大きさを忘却)
- これは E 軸の選択 (cosine を距離関数として選ぶ) がもたらす忘却操作
- 49d adhoc 特徴量が cosine で勝つのは、49d が U_cos の ker に入る情報を最初から持たない (= U_cos に適応した E を持つ) ため

PINAKAS Q-008 への回答の精密化:

> 「メトリクス問題か情報問題か」は偽の二択。真の問題は **E-metric 整合性** (特徴量の符号化がメトリクスの幾何学的前提を満たすか)。49d は cosine の E に適応しており、理論的特徴量は適応していない。

### 15.3 H₃' → ゴールディロックス帯の FEP 的説明

§10 のゴールディロックス帯 (最適次元数が存在する) は H₃' で説明できる:

- 次元が少なすぎる → C (情報内容) の損失 → E[Q] が低下
- 次元が多すぎる → E (cosine との幾何学的整合性) の崩壊 → Var[Q] が爆発 (高次元集中現象)
- ゴールディロックス帯 = C と E の両方が十分に機能する次元範囲

これは §10 の経験的観察を「C が期待値を、E が精度を制御する」という H₃/H₃' の枠組みで**演繹的に**説明する。

### 15.4 次実験への含意

1. **メトリクス変更実験**: L2 距離、マハラノビス距離、情報幾何学的距離で 49d vs 理論的特徴量を比較。H₃ が正しければ、E を変えると順位が逆転しうる。
