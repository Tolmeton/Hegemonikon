### Dokimasia 修飾子 — 座標間の直積パラメータ (60個)

6修飾座標から2つ選び (₆C₂=15)、各2極の組合せ (2×2=4) = **60 修飾パラメータ**。
1つの動詞に別の座標の「色」を着色する。

**構文**: `/verb[座標略:極]` — 例: `/sap[Sc:Mi]`

| 座標略 | 座標 | 極1 | 極2 |
|:-------|:-----|:----|:----|
| `Va` | Value | `E` (External) | `P` (Internal) |
| `Fn` | Function | `Ex` (Explore) | `Ep` (Exploit) |
| `Pr` | Precision | `C` (Certain) | `U` (Uncertain) |
| `Sc` | Scale | `Mi` (Micro) | `Ma` (Macro) |
| `Vl` | Valence | `+` (Positive) | `-` (Negative) |
| `Tm` | Temporality | `Pa` (Past) | `Fu` (Future) |

#### いつ使うか: 修飾子 vs 合成

| 状況 | 方法 | 例 | 意味 |
|:-----|:-----|:---|:-----|
| **1つの行為の多面性** | 修飾子 | `/sap[Sc:Mi]` | 精読しつつ微細部に注目 (1行為) |
| **2つの独立した行為** | 並列合成 | `/sap * /prs` | 精読と注視を同時に (2行為) |
| **順序ある2行為** | 順次合成 | `/sap >> /prs` | まず精読、次に注視 (2段階) |

**主軸の違い**: 同じ座標の組合せでも主軸の選択で認知的態度が変わる:
- `/sap[Sc:Mi]` = 「正確に確定したい。対象はミクロ部分」→ **精度が目的**
- `/prs[Pr:C]` = 「細部に注目したい。精度は高く」→ **粒度が目的**

#### 実用例

| 修飾 CCL | 意味 | ユースケース |
|:---------|:-----|:-------------|
| `/sap[Sc:Mi]+` | ミクロ部分を高精度で精読 | 特定関数のシグネチャ確認 |
| `/prs[Pr:C,Vl:-]` | 高精度・批判的に細部を注視 | バグの局所調査 |
| `/the[Pr:U,Sc:Ma]` | 広域を低精度で観照 | 初見のコードベース概観 |
| `/his[Sc:Mi]` | 過去の特定行の変更履歴 | git blame の精密版 |
| `/ere[Tm:Fu]` | 未来の兆候を探索的に探知 | 技術トレンドの走査 |
| `/exe[Sc:Ma]` | 全体的に弱点を吟味 | アーキテクチャレベルのリスク検出 |
