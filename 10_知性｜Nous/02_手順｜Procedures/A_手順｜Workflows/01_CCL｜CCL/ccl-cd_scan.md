# @cd_scan: 認知歪みスキャン マクロ (v1.1)

> **CCL**: `@cd_scan = /kat-_/ele-_/dio-`
> **用途**: 18種の認知歪みを体系的に検出し修正する
> **旧体系**: dia.cd_scan (旧 A2 Krisis WF) を消化
> **座標触接**: Precision(C↔U) + Valence(+↔-) = 2軸, 2族
> **認知骨格**: 把握 → 論駁 → 修正 (CBT Thought Record の自動化)

## 発動条件

| トリガー | 説明 |
|:---------|:-----|
| `@cd_scan [対象思考]` | 特定の思考パターンを認知歪みスキャン |
| 「歪んでいないか」 | 自然言語トリガー |
| 判断の確信度が極端に高い/低い時 | 暗黙発動 (BC-6 連携) |
| `/fit` で 🔴 判定が出た時 | 暗黙発動 (歪みが原因か検査) |

### ✓ 使うべきとき

- 自分 (Claude) の推論が確証バイアスに汚染されていないか検証
- Creator の仮説に同意した根拠が INPUT TAINT でないか確認
- 設計判断の前提に認知歪みが混入していないか検査
- ハイリスク判断前の BC-14 (Thought Record) と連携

### ✗ 使うべきでないとき

- 品質が劣化しているか検査 (→ `@degradation`)
- リスク許容度を評価 (→ `@risk`)
- 全体の整合性確認 (→ `/ops`)

## 具体例

### 判断: 「このリファクタリングは絶対に失敗する」

```
1. /kat-: 把握
   ✅ #1 全か無か思考: 「絶対に」
   ✅ #5 結論の飛躍: 証拠なしに最悪を想定
   ✅ #7 感情的決めつけ: 不安が根拠
   検出: 3/18, 深刻度: MEDIUM

2. /ele-: 論駁
   | 歪み | 支持証拠 | 反論証拠 | 再解釈 |
   | 全か無か | 過去に1度失敗した | 過去5回中4回は成功 | 成功確率80%。リスクはあるが「絶対」ではない |
   | 結論の飛躍 | 複雑なコード | テストカバレッジ85% | テストが安全網になる |
   | 感情的決めつけ | 不安を感じる | 不安≠事実 | 不安はリスク認識のシグナル、判断の根拠ではない |

3. /dio-: 修正
   元: 「絶対に失敗する」
   修正: 「リスクはあるが、テストとロールバック計画があれば管理可能」
   確信度: 95%(失敗確信) → 40%(懸念あり) + 60%(管理可能)
```
