# /ccl-query: 問い生成特化 (Φ1-Φ4)

> **CCL**: `@query = /pro_/noe_F:[×3]{/zet~(/noe*/dia)}_/kho_/dia`
> **用途**: 検索実行なし、問い生成のみに集中。「計画 8 割」の具現化
> **認知骨格**: 問い→拡散→収束の 3 回反復で最良の問いを研ぎ澄ます

## 展開

| 相 | ステップ | Φ段階 | 意味 |
|:---|:---------|:------|:-----|
| Pre | `/pro` | Φ0 | 初期直感 — 何に引っかかるか |
| 認識 | `/noe` | Φ1 | 深い認識 — 何を知らないか |
| 反復 | `F:[×3]{/zet~(/noe*/dia)}` | Φ2-4 | 問いの拡散×収束を 3 回反復 |
| 配置 | `/ops` | Φ3 | 文脈配置 — どこで問うべきか |
| 判定 | `/dia` | Φ4 | 最終判定 — 最良の問いを確定 |

## /ccl-search との違い

| 項目 | /ccl-query | /ccl-search |
|:-----|:-----------|:-----------|
| 検索実行 (Φ5-6) | ❌ なし | ✅ あり |
| 出力 | 精錬されたクエリ | 検索結果 + 統合レポート |
| 用途 | 問いの質を高めたいとき | 実際に調査したいとき |
| 深度 | L2 (標準) | L1-L3 |

## 使用例

```ccl
@query                     # 問い生成 → 手動で検索
@query _ @search           # 問い生成 → 検索フロー
@query+                    # 深い問い生成（/noe+で展開）
```

> **FEP 根拠 (§1.3)**: 問い生成フェーズでは ε (epistemic value) >> π (pragmatic value)。
> EFE 最小化はほぼ全て ε の最大化に帰着する。良い問い = 正しい attractor basin の特定。
