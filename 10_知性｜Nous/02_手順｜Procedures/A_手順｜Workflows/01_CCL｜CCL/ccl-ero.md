# /ccl-ero (Erōtēsis) — 問いのマクロ

> **目的**: 認識 (`/noe`) → 意志・目的 (`/bou`) → 問いの発見 (`/zet`) という探求のパイプラインを回す。
> **分類**: 探索的マクロ (複合 CCL)
> **代替対象**: `@ooda` (行動で閉じるためのフレームワークからの脱却)

## CCL 定義

```ccl
@ero = /noe|>/bou|>/zet
```

## 処理フロー

各フェーズは `|>` (パイプ) で結合され、**前段の出力が後段の入力コンテキスト**となる。

1. **PHASE 1: /noe (認識)**
   - 対象の本質、構造、現在地を深く認識する。
   - 例: 「このシステムはどう動いているか？」「何が起きたのか？」
2. **PHASE 2: /bou (意志・目的)**
   - `PHASE 1` で得た認識を入力とし、「我々は何を成し遂げたいのか？」を再定義する。
   - 例: 「この構造において、本質的に望ましい姿は何か？」
3. **PHASE 3: /zet (探求・問い)**
   - `PHASE 2` の意志を入力とし、それを実現するため、あるいはさらに進むための「核心的な問い」を発見する。
   - 例: 「この意志を実現するために、まだ問われていないことは何か？」

## 応用構文: 末尾パイプの開放化 (案A 適用時)

`@ero` 自体は問い (`/zet`) で終わる「開かれたマクロ」である。
必要に応じて、後続のアクションを接続できる:

- `@ero|>/ene` — 問いを見つけてから、そのまま実装/検証コードを書く
- `@ero|>/dia` — 問いに基づく判断・評価を行う

## 出力形式

このマクロを実行した場合、必ず以下のセクションを出力すること。

```markdown
## Erōtēsis (@ero) — 問いのパイプライン

### 1. Noēsis (認識)
- {対象の本質的な構造と現在地の要約}

### 2. Boulēsis (意志)
- {Noēsis を踏まえた上で、我々が目指すべき核となる状態}

### 3. Zētēsis (問い)
- {Boulēsis に至るために解かれるべき 3 つの問い}
  1.
  2.
  3.

### 4. Next (末尾が開いているため)
- {上記の問いをどう扱うか。例: -> /ccl-search, -> /ene+ などの提案}
```
