# @loss: 損失回避分析マクロ (v1.1)

> **CCL**: `@loss = /ele-_/prm-_/dio-`
> **用途**: 損失回避バイアスを検出し、合理的な損失評価と修正行動を導く
> **旧体系**: ore.loss_aversion を消化
> **座標触接**: Valence(-) + Temporality(Future) = 2軸, 2族
> **認知骨格**: 論駁 → 先見 → 修正

---

## 発動条件

| トリガー | 説明 |
|:---------|:-----|
| `@loss [対象]` | 特定の損失シナリオの分析 |

| 撤退/中止の判断時 | 暗黙発動 |

## 損失回避バイアスの種類

| バイアス | 定義 | 検出パターン | 座標 |
|:---------|:-----|:------------|:-----|
| **損失回避** | 同額の損失は利得の2倍痛い | 「失うのが怖い」> 「得るのが嬉しい」 | (-)/Asymmetry |
| **サンクコスト** | 既投資を理由に継続 | 「ここまでやったのだから」 | Past/Sunk |
| **現状維持** | 変化のコストを過大評価 | 「今のままでいい」 | (-)/Status |
| **保有効果** | 保有物の価値を過大評価 | 「手放したくない」 | (+)/Endowment |

## 具体例

### 「使われていないコードの削除を躊躇」

```
1. /ele-:
   サンクコスト: ✅ 「3日かけて書いたから」→ 過去の投資は削除判断に無関係
   保有効果: ✅ 「いつか使うかも」→ 6ヶ月使われていない = 市場価値ゼロ
   合理的評価: 削除の実際のコスト = 0 (git で復元可能)

2. /prm-:
   回避: コードベース肥大化 → 認知負荷↑ → メンテコスト↑
   受容: 削除 → クリーンな状態 → 新機能追加が容易に
   EV: 回避=-500, 受容=+200

3. /dio-: 即削除。git log で復元可能。
```
